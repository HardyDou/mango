# 支付投产阻塞与用户确认清单

## 1. 目标

记录当前支付中心距离完整投产仍未关闭的交付台账项，明确每项缺少的外部资料、用户决策或最终门禁，防止在未联调、未确认或未验收时把台账伪标为完成。

## 2. 范围

- 外部真实通道：通联、华夏银行、微信、支付宝和其他已知通道清单。
- 用户确认项：线下转账范围、其他通道清单和优先级、支付成功后开票事件、租户定义。
- 最终门禁：交付前台账检查。

| 未注册适配器行为 | `PaymentChannelAdapterRegistry.requireAdapter` 对未注册通道按业务错误拒绝 | 当前不会伪走外部通道固定成功，符合“未接真实接口不得伪装完成”红线 |
| 外部通道边界契约测试 | `PaymentExternalChannelReadinessContractTest` 断言通联、华夏、微信、支付宝仍由注册表拒绝；生产适配器只有芒果支付；未声明微信/支付宝 SDK；外部通道台账和阻塞证据仍保持未完成/阻塞状态 | 防止未接真实通道时误走芒果支付或伪标完成 |

## 6. 本轮结论

- 当前支付中心本地闭环、芒果支付内置虚拟通道、后台页面、OpenAPI、通知、对账、结算、E2E 和集成测试已按台账分项关闭。
- 当前不具备把外部真实通道标记完成的证据。
- 当前不具备把 4 个用户确认项标记完成或例外的用户确认依据。
- 当前不能关闭 `PAY-TEST-004`，因为交付台账仍有未完成项。
