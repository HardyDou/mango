# 网址导航执行计划

文档状态：草案
日期：2026-06-30
需求文档：`mango-docs/designs/2026-06-30-url-navigation-requirements.md`
设计文档：`mango-docs/designs/2026-06-30-url-navigation-design.md`

## 1. 背景

用户需要一个公司常用网址入口，同时支持登录用户收藏网址和登记个人网址。后台能力放在 `平台能力 / 网址管理` 下，维护 `网址分类` 和 `网址列表`。

## 2. 目标

完成网址导航第一版，交付可用、可验收的用户侧和后台侧能力：

- 用户侧可以查看公司网址、收藏网址、维护我的网址。
- 后台侧可以维护网址分类和网址列表。
- 网址按公开、公司内、指定部门、指定用户、个人范围控制可见性。
- 分类为一级分类，不做 tree。
- Open API 可以在未登录状态下只读访问公开网址。

## 3. 范围

本次包含：

- 后端模块：网址分类、网址、可见目标、收藏记录。
- 后端 API：用户侧查询/收藏/个人网址，后台分类/网址管理。
- Open API：公开网址只读查询。
- 前端页面：公司网址、我的收藏、我的网址、网址分类、网址列表。
- 前端公开接口包：`@mango/link-openapi`，供非管理后台调用公开网址 Open API。
- 菜单资源：用户侧 `网址导航`，后台侧 `平台能力 / 网址管理`。
- 自动化测试：覆盖核心可见范围、收藏、个人网址、后台维护。

本次不包含：

- 小组网址。
- 多级分类 tree。
- 访问统计。
- 网址健康检测。
- 浏览器插件。
- 后台查看用户个人网址。

## 4. Sprint 拆分

### Sprint 1：后端基础对象和后台管理

目标：完成网址分类、网址列表和可见范围数据基础。

改动项：

- 新增后端模块 `mango/mango-platform/mango-link`，包含 `mango-link-api`、`mango-link-core`、`mango-link-starter`、`mango-link-starter-remote`。
- 新增数据表：网址分类、网址、可见目标、收藏记录。
- 实现后台网址分类 API。
- 实现后台网址列表 API。
- 实现公开网址 Open API。
- 实现公开、公司内、指定部门、指定用户、个人可见范围校验服务。
- 新增菜单资源声明。

完成标准：

- 后台可以创建分类。
- 后台可以创建公开、公司内、指定部门、指定用户网址。
- 服务端可按用户和登录态判断网址是否可见。
- Open API 只返回公开、启用分类、启用状态的网址。
- API 测试覆盖可见范围判断。

### Sprint 2：用户侧网址导航

目标：完成用户侧公司网址、我的收藏、我的网址。

改动项：

- 新增前端用户侧页面：公司网址。
- 新增前端用户侧页面：我的收藏。
- 新增前端用户侧页面：我的网址。
- 接入用户侧查询公司网址 API。
- 接入收藏、取消收藏 API。
- 接入个人网址新增、编辑、删除 API。
- 增加页面空态、加载态、失败态、无权限态。

完成标准：

- 未登录用户只能看到公开网址。
- 登录用户能看到公开、公司内、指定部门、指定用户范围内的网址。
- 用户可以收藏和取消收藏。
- 用户可以维护个人网址，且他人不可见。

### Sprint 3：后台页面和回归验收

目标：完成后台页面和端到端验收。

改动项：

- 新增后台网址分类页面。
- 新增后台网址列表页面。
- 支持分类搜索、状态筛选、启用、停用、删除受限。
- 支持网址搜索、分类筛选、可见范围筛选、状态筛选。
- 支持公开、公司内、指定部门、指定用户配置表单。
- 新增 E2E 测试和验收证据。

完成标准：

- 后台创建的网址能在用户侧按可见范围展示。
- 停用分类或网址后，用户侧不再展示。
- 指定部门和指定用户范围隔离正确。
- E2E 覆盖 P0 场景并形成验收证据。

## 6. 验证方式

最低验证：

- 后端编译。
- 受影响模块单元/API 测试。
- 前端类型检查和构建。
- 用户侧页面 E2E。
- 后台页面 E2E。
- 权限和数据范围 E2E/API 测试。

核心验收场景：

| 验收项 | 验证方式 |
|---|---|
| AC-001 未登录只能看公开网址 | E2E + API |
| AC-002 登录用户可看公司内网址 | E2E + API |
| AC-003 指定部门隔离 | API + E2E |
| AC-005 收藏网址 | E2E |
| AC-009 新增个人网址 | E2E |
| AC-012 个人网址隔离 | API + E2E |
| AC-014 停用分类后用户侧不展示 | E2E |
| AC-018 指定部门网址展示 | API + E2E |
| AC-019 指定用户网址展示 | API + E2E |
| AC-020 停用网址后用户侧不展示 | E2E |
| AC-021 Open API 返回公开网址 | API |
| AC-022 Open API 不返回公司内网址 | API |
| AC-023 Open API 租户上下文缺失失败 | API |

## 7. 完成标准

- PRD、设计文档和执行计划已确认。
- 后端真实接口和数据库持久化完成。
- Open API 只读返回公开网址，不要求登录，不泄露非公开网址。
- `@mango/link-openapi` 可独立构建，只调用公开接口，不依赖管理后台包。
- 前端页面接入真实接口，不使用演示数据。
- 菜单资源真实入库或通过现有资源机制初始化。
- 权限、租户、可见范围判断在服务端生效。
- 用户侧和后台侧 P0 验收项通过。
- 验收证据记录测试命令、环境、账号/租户标识、截图或报告路径。

## 8. 遗留问题

| 问题 | 是否阻断本次 | 处理方式 |
|---|---|---|
| 是否增加小组网址 | 否 | 不纳入本次；需要独立小组模型后再设计 |
| 是否支持分类 tree | 否 | 本次固定一级分类 |
| 是否允许后台审计个人网址 | 否 | 本次后台不展示个人网址 |
| 是否支持网址有效性检测 | 否 | 不纳入本次 |
