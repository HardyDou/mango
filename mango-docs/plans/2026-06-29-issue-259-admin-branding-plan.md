# Issue 259 Admin 后台品牌配置开发计划

## 1. 目标

完成 Admin 独立后台品牌配置能力，支持系统管理中维护品牌信息，并在登录页、后台框架、浏览器标题、favicon 和页脚中生效。

## 2. 开发任务

| ID | 任务 | 文件范围 | 完成标准 |
|---|---|---|---|
| DEV-001 | 后端品牌配置契约 | `mango-system-api` | 有保存命令和展示 VO |
| DEV-002 | 后端品牌配置服务 | `mango-system-core` | 复用 `sys_config` 完成读取和保存 |
| DEV-003 | 后端 REST 与资源 | `mango-system-starter` | 新增 public/query/save 接口、菜单和默认配置 |
| DEV-004 | 前端配置页 | `mango-ui/packages/system` | 可读取、编辑、上传文件 token、保存 |
| DEV-005 | 页面注册 | `mango-ui/packages/admin-pages` | 菜单 component 能匹配前端页面 |
| DEV-006 | 运行时品牌应用 | `admin-shell`、`auth`、`mango-admin` | 登录页、Logo、标题、favicon、页脚生效 |
| DEV-007 | 文档与验证 | `README`、测试、PMO 检查 | 构建/测试/文档检查结果明确 |

## 3. 验收清单

| 验收项 | 验证方式 |
|---|---|
| 未配置时展示默认品牌 | 后端单测、页面走查 |
| 保存后配置页能回显 | 页面走查 |
| 登录页标题/副标题/图片生效 | 页面走查 |
| 后台 Logo、浏览器标题、favicon、页脚生效 | 页面走查 |
| 文件字段不保存访问 URL | 单测和代码检查 |
| 菜单资源可同步 | 后端资源文件检查 |

## 4. 不处理范围

- 不新增站点管理、CMS 或广告位联动。
- 不改登录策略、权限策略、默认首页。
- 不新增独立数据表。
- 不把文件预览地址持久化到配置中。

## 5. 风险

- 运行时读取 public 接口失败时必须静默降级，不能影响登录。
- favicon 变更依赖浏览器缓存，页面验证时需要确认 link 标签已更新。
- 配置菜单需要资源同步后才会在真实菜单中出现。
