# Migration 数据运行验证说明

## 目标

验证模块级 `V1` migration 在本地 MySQL `mango` 库中初始化后，菜单、功能按钮、角色授权、用户和基础业务数据是否可用，并确认单体服务在关闭本机 office 组件依赖后能启动。

## 范围

- 本地 MySQL `mango` 数据库。
- 单体应用 `mango-app/monolith/mango-monolith-app`。
- Flyway history 表、授权菜单/按钮/角色关联、身份用户、系统字典、模块基础数据。
- 服务健康接口和公开 OpenAPI 文档接口。

## 不做什么

- 不修改业务代码。
- 不发布 jar。
- 不验证前端浏览器页面交互。
- 不依赖共享环境或生产数据。

## 设计输入

- 已确认项目尚未正式发布，migration 无共享环境依赖。
- migration 已按模块压缩为模块级 `V1`。
- 单体默认启动依赖本机 office 组件；本次验证通过启动参数关闭 office 插件。

## 数据变化

本次验证允许重建本地 `mango` 测试库，并由 Flyway 重新初始化数据。

## 菜单、页面、权限变化

本次不新增菜单、页面或权限，只验证 migration 初始化后的菜单、按钮和角色授权数据完整性。

## 测试范围

- 空库 Flyway 执行。
- 模块独立 Flyway history 表。
- 菜单/按钮数量、父子关系、按钮权限码。
- 角色菜单授权引用完整性。
- 用户、租户成员、字典、模块基础数据。
- 单体应用启动和基础 HTTP 访问。
