# 审批中心发起流程 logo 文件 ID 修复设计说明

## 目标

修复审批中心“发起流程”页面在流程 logo 未配置或保存为非文件 ID 文本时仍请求文件预览接口，导致后端返回 `参数类型错误: id` 的问题。

## 范围

- 修正前端文件引用归一化逻辑，只把后端可解析的正整数 ID 视为文件 ID。
- 发起流程页继续使用文件包归一化结果；空 logo、图标名或普通文本不触发 file 预览接口。
- 补充文件 ID 归一化回归测试，覆盖 `file`、Element 图标名、空值和 `mango-file:{id}` token。

## 不做什么

- 不修改工作流后端接口和数据库结构。
- 不调整流程 logo 保存字段协议之外的业务字段。
- 不处理历史脏数据清理；前端对非文件 ID 做兼容展示。

## 设计输入

- 用户反馈：审批中心发起流程中，流程 logo 未配置时仍会加载 file，并提示 `参数类型错误：id`。
- 当前实现：`start-process/index.vue` 会对 `item.icon` 调用 `normalizeFileId`，再请求 `fileApi.preview(id)`。
- 当前缺陷：`normalizeFileId` 只做字符串归一化，`file`、`CollectionTag` 等非 Long ID 也会被当成文件 ID。

## 设计说明

### 影响模块

- `mango-ui/packages/file`：文件引用归一化边界。
- `mango-ui/packages/workflow`：发起流程页依赖该边界回显流程 logo。

### 接口变化

- 无后端接口变化。
- 前端 `normalizeFileId` 行为收紧：仅正整数和 `mango-file:{正整数}` 返回文件 ID，URL、空值、图标名、普通文本返回空字符串。

### 数据变化

- 无数据库变更。

### 菜单/页面/权限变化

- 无菜单和权限变化。
- 页面表现：未配置 logo 或非文件 ID logo 时展示默认流程图标，不再请求文件预览接口。

### 测试范围

- 单元测试覆盖文件 ID 归一化边界。
- 构建验证覆盖工作流包和文件包类型编译链路。

## 风险与限制

- 该修复会阻止非数字字符串作为文件 ID 使用；当前后端文件接口参数为 `Long id`，因此这是符合后端协议的收紧。
