# Sprint：numgen 版本-片段模型重构

更新时间：2026-05-23

## 1. 目标

把 `numgen` 从“规则表达式配置页面”重构为“编号格式搭建器”。

最终用户按编号结构配置：

```text
生成器
  └─ 版本
       └─ 多个有序片段
```

页面不再要求用户理解或书写表达式。表达式只作为系统派生结果或内部实现，不作为主要配置入口。

## 2. 范围

本 Sprint 一次完成：

- 管理端页面推翻重做。
- 前台领域语言统一为“生成器 / 版本 / 片段”。
- 版本编辑采用横向片段条，参考用户草图。
- 片段支持固定文本、时间、业务参数、自增流水。
- 生成器详情内管理版本，不再单独暴露“规则台账”。
- 发布后版本不可编辑。
- 同一生成器只能有一个生效版本。
- 支持预览、发布、发号、发号记录查看。
- 补齐后端规则校验测试和前端浏览器 E2E。

## 3. 不做事项

- 不做编号回收。
- 不做已颁发编号重新生成。
- 不把内置变量作为普通用户主配置项。
- 不引入任意脚本或通用表达式执行引擎。
- 不在业务模块中直接访问 numgen 表。
- 不为了命名立即强制迁移历史表名；若现有表结构可承载版本模型，本 Sprint 复用现有表。

## 4. 验收标准

- 管理员能创建生成器。
- 管理员能在生成器详情内创建版本。
- 管理员能按横向片段条配置编号格式。
- 管理员无需编写表达式即可完成配置。
- 片段能新增、编辑、删除、排序。
- 能用模拟参数预览编号。
- 能发布版本，发布后成为当前生效版本。
- 发布后的版本不可编辑。
- 业务发号接口按当前生效版本生成唯一编号。
- 发号记录可查询。
- 后端测试覆盖版本发布、片段校验、参数缺失、流水隔离。
- 前端 Playwright E2E 覆盖创建生成器、创建版本、配置片段、预览、发布、发号、查看记录。

## 6. 领域模型

### 6.1 生成器

生成器是某类业务编号的入口，例如合同编号、订单编号、项目编号。

生成器保存：

- ID
- 编号键 `genKey`
- 名称
- 描述
- 启停状态
- 当前生效版本

生成器本身不描述编号格式，编号格式由版本承载。

### 6.2 版本

版本是一套完整编号生成方案。

版本保存：

- ID
- 所属生成器
- 名称
- 版本号
- 审核状态
- 发布状态
- 作用域策略
- 重置周期
- 起始值
- 步长
- 描述
- 多个有序片段

规则在用户语言中改称“版本”。后端可短期复用 `rule` 命名和表结构，但页面、文案、测试用例统一使用“版本”。

### 6.3 片段

片段是编号的一部分。一个版本由多个片段按顺序拼接。

片段类型：

| 类型 | 含义 | 示例 |
|---|---|---|
| 固定文本 | 固定输出指定字符 | `HT`、`-` |
| 时间 | 按格式输出日期时间 | `yyyyMMdd` |
| 业务参数 | 使用调用方传入的参数 | `type`、`projectId` |
| 自增流水 | 生成递增数字，可补零 | `000001` |

### 6.4 运行态

运行态包括：

- `Sequence`：按生成器、版本、作用域、周期维护流水。
- `History`：记录生成结果、版本、耗时、状态和失败原因。

## 7. UI 设计

### 7.1 页面结构

```text
┌────────────────┬──────────────────────────────────────────────┐
│ 生成器列表      │ 生成器详情                                    │
│ 搜索/新增       │ 基础信息 / 当前版本 / 版本列表 / 发号记录       │
│ 生成器卡片      │                                              │
└────────────────┴──────────────────────────────────────────────┘
```

详情区内按工作流组织：

1. 生成器基础信息
2. 版本列表
3. 当前选中版本的片段编辑器
4. 预览与发号
5. 发号记录

### 7.2 片段编辑器

核心交互采用横向片段条：

```text
[片段1 固定文本 HT] [片段2 时间 yyyyMMdd] [片段3 业务参数 type] [片段4 流水 000001] [+ 添加片段]
```

交互规则：

- 点击片段，在右侧抽屉或侧栏编辑该片段。
- 片段可上移、下移调整顺序。
- 新增片段时先选择类型，再填写该类型需要的字段。
- 已发布版本不允许修改片段。
- 页面实时展示预览结构。
- 表达式不作为输入项展示。

### 7.3 版本操作

- 新建版本：在当前生成器内创建草稿版本。
- 编辑版本：只允许草稿、审核未通过版本。
- 发布版本：发布后锁定，并替换当前生效版本。
- 删除版本：只允许未发布版本。

## 8. 后端设计

### 8.1 API 语义

优先保证用户侧语义正确：

- 前端类型使用 `NumgenVersion` 映射后端 `Rule`。
- 前端接口方法使用 `pageVersions`、`createVersion`、`publishVersion`。
- 后端可保留 `NumgenRuleApi`，本 Sprint 不强制重命名表和 Controller。

后端新增或强化约束：

- 发布入口校验版本存在。
- 发布入口校验版本至少包含一个片段。
- 发布入口校验自增流水片段数量符合要求。
- 发布后版本不可编辑。
- 发布新版本时，同生成器其他版本自动变为未生效。
- 发号只使用生效版本。

### 8.2 数据结构

本 Sprint 默认不新增 DDL。

沿用：

- `numgen_generator`
- `numgen_rule`
- `numgen_rule_segment`
- `numgen_sequence`
- `numgen_history`

若实施时发现当前表缺少必要字段，只新增 Flyway migration，不修改历史 migration。
