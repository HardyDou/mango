# 统一支付平台 Sprint 01 计划

## 1. 背景

统一支付平台需要按 `mango-docs/designs/统一支付系统设计说明书.md` 重构为可真实投产的支付能力。历史支付代码只能作为通道思路参考，不继承历史包袱。本 Sprint 不允许以合并页面、演示链路、伪实现或未验证主流程替代设计要求。

## 2. 目标

交付支付平台第一阶段“支付收单闭环”的生产级基础能力：支付域数据模型、支付中心后台菜单、应用接入配置、支付通道配置、开放接口、芒果支付内置虚拟通道、通联支付通道、华夏银行通道接入基础、收银台组件、管理后台示例应用、业务订单到支付成功通知闭环、基础账单导入和金额核对。

## 3. 范围

### 3.1 本 Sprint 必做

1. 建立支付模块边界、代码结构、依赖方向和公开 API。
2. 建立支付核心表、唯一约束、租户隔离字段、审计字段、金额规则。
2A. 建立标准支付方式、通道能力、签约能力和支付方式路由策略模型，支付方式不得直接绑定通道或通道签约配置。
3. 后端入库支付中心菜单及权限点，菜单按“应用接入、支付通道、交易订单、对账结算”二级目录分组。
4. 交付支付中心后台页面：应用管理、收银台配置、企业主体管理、通道签约配置、支付通道管理、支付方式管理、业务订单、支付订单、交易流水、通知记录、操作审计。
5. 交付退款订单、异常订单、对账管理、差异处理、结算汇总的基础列表和查询能力；涉及资金变更的处理动作按受控接口实现，不能留空壳。
6. 交付一套 Web/H5 `PaymentCashier` 可复用组件和非菜单收银台页面。
7. 交付开放接口：创建业务订单、查询业务订单、获取收银台地址、发起支付、查询支付订单、发起退款、查询退款、获取支付凭证。
8. 交付芒果支付内置虚拟通道，支持全部标准支付方式，并具备支付下单、支付回调、主动查单、关单、退款、退款查询、账单生成、对账、返回码映射和异常场景控制能力。
9. 交付通联支付通道适配基础，支持配置、支付、回调验签、查单、退款、账单解析。
10. 交付华夏银行通道适配基础；若官方或内部对接资料不足，停止在该项并登记阻塞，不得用其他通道替代。
11. 微信和支付宝适配可参考用户确认的 SDK 依赖：`IJPay-WxPay`、`IJPay-AliPay`、`weixin-java-mp`、`wechatpay-java`、`alipay-sdk-java`；SDK 只允许在通道适配层使用。
12. 交付状态机、幂等、重复支付唯一成功约束、通知重试、审计记录。
13. 交付管理后台示例应用，作为 Demo 跑通应用接入、支付通道、收银台配置、芒果支付内置虚拟通道付款、通知、订单、流水、审计和对账全流程。
14. 交付芒果支付内置虚拟通道 E2E：后台配置收银台，点击“收银台”，通过芒果支付内置虚拟通道完成支付，展示支付结果，后台可查询订单、流水、通知和审计。

### 3.2 本 Sprint 不做

1. 不建设钱包、余额、充值、提现、内部转账。
2. 不建设财务总账、会计凭证、自动付款。
3. 不把保函申请、项目、出函、拒保、撤单等业务对象放入支付模块。
4. 不设置独立“租户收银台”菜单。
5. 不把支付结果页登记为后台菜单。
6. 不使用未交付占位实现、演示替代或伪实现替代内置虚拟通道或真实能力。

## 4. 改动项

| 序号 | 改动项 | 交付内容 | 验收口径 |
|---|---|---|---|
| 1 | 模块边界 | 支付模块代码结构、公开 API、依赖声明 | 支付模块不直接依赖其他模块内部实现 |
| 2 | 数据模型 | 支付核心表、索引、状态字段、审计字段 | 数据迁移可执行，唯一约束生效 |
| 3 | 菜单权限 | 支付中心菜单、二级目录、页面权限、操作权限 | 后台菜单由后端入库，按应用接入、支付通道、交易订单、对账结算分组，权限控制生效 |
| 4 | 应用接入后台 | 应用管理、收银台配置、通信安全配置、示例应用 | 列表、详情、编辑、启停、默认收银台、示例应用 E2E 可用；管理后台不提供重置密钥按钮 |
| 4A | 支付通道后台 | 企业主体、通道签约配置、支付通道、支付方式、通道签约字段模板、通道能力、签约能力、支付方式路由策略 | 主体与签约配置关系清晰；支付通道管理维护签约字段模板和通道能力；通道签约配置按模板登记真实值并开通签约能力；支付方式管理维护三级标准方式和路由策略；商户号、AppId、证书和密钥只存在通道签约配置 |
| 5 | 订单后台 | 业务订单、支付订单、退款订单、流水、通知、审计页面 | 查询、详情、状态流转记录可用 |
| 6 | 异常与对账 | 异常订单、对账管理、差异处理、结算汇总页面 | 基础查询和受控处理闭环可验证 |
| 7 | 开放接口 | 订单、收银台、支付、退款、凭证接口 | 签名、防重放、幂等、多租户隔离通过测试 |
| 8 | 收银台组件 | 一套 Web/H5 `PaymentCashier`、支付方式选择、二维码、网银/银联跳转、线下转账说明、H5 跳转、结果页 | 组件复用，按收银台配置渲染页面，结果页不在后台菜单；App/小程序页面不纳入本期 |
| 9 | 芒果支付内置虚拟通道 | 全部标准支付方式、支付下单、支付回调、主动查单、关单、退款、退款查询、账单生成、对账、返回码映射、异常场景控制 | 芒果支付 E2E、退款 E2E、查单/关单测试、账单核对测试、异常场景测试通过 |
| 10 | 通联通道 | 配置、支付、回调验签、查单、退款、账单 | 联调环境测试或阻塞登记 |
| 11 | 华夏银行通道 | 配置、支付、回调验签、查单、退款、账单 | 联调环境测试或阻塞登记 |
| 12 | 微信/支付宝 SDK | 用户确认的 SDK 依赖只在通道适配层使用 | 依赖检查、代码审查、适配层测试 |
| 13 | 质量门禁 | 单测、集成测试、E2E、台账检查、敏感词扫描 | 全部通过才能提交 PR |

## 5. 阶段拆分

### 5.1 阶段 A：交付台账确认

完成标准：

1. `mango-docs/plans/2026-05-25-payment-delivery-ledger.md` 覆盖菜单、页面、API、数据、权限、E2E、通道、投产检查。
2. 用户确认阻塞项处理方式。
3. `delivery-contract-check` 在 plan 模式通过。

停止条件：

1. 用户未确认“其他已知支付通道清单”。
2. 用户未提供华夏银行官方或内部对接文档且要求必须完成真实联调。

### 5.2 阶段 B：基础域和菜单

完成标准：

1. 支付核心表迁移完成。
2. 支付中心菜单后端入库。
3. 应用管理、收银台配置、企业主体、通道签约配置、支付通道、支付方式基础管理可用。
4. 支付方式管理可维护“对公/对私 -> 网银/线下转账/支付宝/微信 -> 扫码/H5/App/小程序/网银跳转/账号转账”的三级标准方式。
5. 支付通道管理可维护不同通道的签约字段模板和通道能力，通道签约配置可按模板渲染单行、多行、密钥、文件上传、枚举、JSON 等控件并保存真实配置值。
6. 通道签约配置可从通道能力中开通签约能力，路由策略可将标准支付方式路由到签约能力。
7. 权限和租户隔离测试通过。

### 5.3 阶段 C：收单闭环和内置虚拟通道

完成标准：

1. 业务系统可创建订单并获取收银台。
2. 收银台只展示聚合后的标准支付方式，并通过路由策略命中芒果支付内置虚拟通道签约能力完成支付。
3. 支付成功后推进业务订单、支付订单、流水、通知和审计。
4. 重复创建、重复回调、重复通知均幂等。

### 5.4 阶段 D：外部通道基础接入

完成标准：

1. 通联支付具备正式适配器和联调验证记录。
2. 华夏银行通道具备正式适配器和联调验证记录；资料不足时登记阻塞并等待确认。
3. 通道密钥和证书不明文入库，不在后台明文展示。

### 5.5 阶段 E：后台验收和 E2E

完成标准：

1. 支付中心所有菜单可见且权限受控。
2. 收银台配置行操作进入非菜单收银台页面。
3. 芒果支付内置虚拟通道 E2E 全流程通过。
4. 对账导入、差异识别、结算汇总基础验证通过。
5. 台账所有本 Sprint 项为 `DONE` 或有用户确认的 `EXCEPTION`。

## 6. 验证方式

| 类型 | 命令或方式 |
|---|---|
| PMO 台账计划检查 | `node mango-pmo/tools/delivery-contract-check.mjs --design mango-docs/designs/统一支付系统设计说明书.md --ledger mango-docs/plans/2026-05-25-payment-delivery-ledger.md --mode plan` |
| PMO 台账交付检查 | `node mango-pmo/tools/delivery-contract-check.mjs --design mango-docs/designs/统一支付系统设计说明书.md --ledger mango-docs/plans/2026-05-25-payment-delivery-ledger.md --mode verify --scan <支付改动路径>` |
| 后端测试 | 执行支付模块单元测试和集成测试 |
| 前端测试 | 执行支付中心页面和 `PaymentCashier` 组件测试 |
| E2E | 执行支付中心菜单、配置、收银台、芒果支付内置虚拟通道付款、结果页、后台查询 Playwright 测试 |
| 安全检查 | 扫描敏感词、密钥明文、未完成占位、临时替代和伪实现等交付风险标记 |

## 7. 完成标准

1. 交付台账无未完成项，或未完成项均有用户确认的 `EXCEPTION`。
2. 支付中心后台菜单全部后端入库，不存在前端长期静态菜单替代。
3. 芒果支付内置虚拟通道独立可用，能完成全部标准支付方式、支付下单、支付回调、主动查单、关单、退款、退款查询、账单生成、对账、返回码映射和异常场景控制验证。
4. 通联支付和华夏银行通道按确认资料完成正式适配或登记阻塞。
5. 所有核心状态机、幂等、权限、多租户隔离、审计和 E2E 验证通过。
6. 代码中不存在未完成占位、临时替代和伪实现等未交付标记。

## 8. 遗留问题

| ID | 问题 | 当前处理 |
|---|---|---|
| Q-001 | 线下收款是否纳入第一阶段，是否需要独立通道能力 | 用户已确认线下收款作为独立完整支付通道；已转入 `PAY-OFFLINE-001` 至 `PAY-OFFLINE-007` 研发项 |
| Q-002 | 除芒果支付、通联、华夏银行外，其他已知支付通道清单和优先级 | 等待用户确认；不得自行删减 |
| Q-003 | 支付成功后是否需要通知业务系统触发开票 | 等待用户确认；支付系统默认只发支付结果通知 |
| Q-004 | 租户是公司内部事业部、外部客户，还是两者都支持 | 等待用户确认；当前模型保留通用租户隔离 |
| Q-005 | 华夏银行通道官方或内部对接文档和联调环境 | 等待资料；资料不足时停止该项，不允许替代交付 |
