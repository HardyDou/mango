# 工作流业务 UI 组件交付计划

## 目标

把工作流详情页中已经存在但分散的 UI 能力沉淀为 `@mango/workflow` 公共组件，支持其它业务模块直接复用。

## 范围

- 新增工作流业务布局组件。
- 新增工作流侧栏组件。
- 新增流程实例概要组件。
- 新增当前流程实例进度封装。
- 新增流程定义图语义组件和弹窗。
- 新增历史申请组件和弹窗。
- 更新 `@mango/workflow` README 和组件 README。
- 任务详情页尽量复用新组件，减少重复实现。

## 不处理范围

- 不调整 workflow 后端接口。
- 不调整流程设计器节点协议。
- 不新增业务模块后端表。
- 不发布 npm 包。

## 实施步骤

1. 登记 Mango Issue。
2. 新增公共组件和类型。
3. 更新包入口导出。
4. 将任务详情页右侧区域改为复用公共组件。
5. 更新 README 使用说明。
6. 补充组件测试。
7. 执行 `@mango/workflow` 相关验证。

## 验证计划

- 运行 `@mango/workflow` 包测试。
- 运行 `@mango/workflow` 类型检查或构建。
- 检查新增导出可从包入口引用。
- 检查任务详情页仍保留业务审批组件、自定义记录面板和审批动作能力。
