# 统一支付平台投产就绪计划

## 1. 目标

为统一支付平台建立投产前硬门禁，确保交付内容不是演示版、压缩版或不可用主链路，而是具备真实运行、可审计、可回滚、可观测、可验证的生产级能力。

## 2. 投产原则

1. 未经用户确认，禁止缩小设计范围。
2. 未完成交付台账，禁止声明完成。
3. 未通过 E2E，禁止提交 PR。
4. 未完成权限、租户隔离、幂等、状态机、审计、安全检查，禁止投产。
5. 通道配置、证书、密钥不得进入仓库，不得后台明文展示。
6. 自建沙箱通道是正式沙箱环境，不得使用未交付占位实现或演示替代。

## 4. 必跑验证命令

```bash
node mango-pmo/tools/delivery-contract-check.mjs \
  --design mango-docs/designs/统一支付系统设计说明书.md \
  --ledger mango-docs/plans/2026-05-25-payment-delivery-ledger.md \
  --mode plan
```

交付前必须执行 verify 模式：

```bash
node mango-pmo/tools/delivery-contract-check.mjs \
  --design mango-docs/designs/统一支付系统设计说明书.md \
  --ledger mango-docs/plans/2026-05-25-payment-delivery-ledger.md \
  --mode verify \
  --scan <支付相关改动路径>
```

研发完成后还必须执行实际工程命令：

```bash
pnpm lint
pnpm build
pnpm test
pnpm playwright test
```

后端测试命令以仓库支付模块实际构建工具为准，执行结果必须记录到 PR 描述和交付报告。

## 5. 阻塞登记规则

遇到以下情况必须停止对应交付项并登记，不得自行降级：

1. 华夏银行通道缺少官方或内部对接文档。
2. 外部通道缺少联调环境、商户参数、证书或回调验签规则。
3. 用户未确认其他已知通道清单。
4. 线下转账自动认款范围不明确。
5. 租户定义影响权限或数据隔离模型。
6. 业务系统开票触发要求影响通知协议。

登记格式：

| 日期 | 阻塞项 | 影响台账 ID | 需要用户确认的问题 | 当前不能做的事 |
|---|---|---|---|---|

## 6. PR 合并门禁

1. 交付台账必须更新。
2. 投产清单必须更新证据。
3. 所有测试命令必须有执行记录。
4. 未完成项必须为 0，或全部为用户确认的 `EXCEPTION`。
5. PR 描述必须列出台账总项、完成项、例外项、未完成项。
6. 禁止在 PR 中出现未完成占位、临时替代、伪实现、延期补齐等未交付标记。
