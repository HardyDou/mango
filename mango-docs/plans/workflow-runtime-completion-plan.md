## 目标

把 Mango 工作流从“能配置、能发布、能发起”的半闭环，补齐为“表单设计、流程发起、待办审批、流转记录、任务中心、运行监控”的可用闭环。

本计划只规划真实页面、真实 API、真实 E2E。没有真实运行能力的菜单不落库，不提前造入口。

## 术语边界

| 概念 | 说明 |
|---|---|
| 流程定义 | 可配置和发布的流程主档 |
| 流程版本 | 每次发布形成的不可变版本，绑定 Flowable processDefinition |
| 流程实例 | 用户发起后生成的运行实例 |

| 自定义表单 | 绑定业务页面或业务组件的表单 |

## P0 最小审批闭环

目标：一个“请假”流程可以从流程管理配置，进入发起流程，生成我的待办，被审批通过或拒绝，并在我的发起/我的已办看到状态和记录。

## P1 审批人与组织权限打通

目标：审批节点可以配置和解析 Mango 组织、岗位、成员、角色。

### 办理人类型

| 类型 | 说明 |
|---|---|
| 发起人 | 当前流程发起人 |
| 指定成员 | 从成员管理选择用户 |
| 指定角色 | 当前机构内某角色的成员 |
| 指定岗位 | 当前机构内某岗位成员 |
| 指定组织 | 当前机构内某组织成员 |
| 组织负责人 | 发起人所在组织或指定组织负责人 |
| 表单字段 | 从表单变量中读取用户/组织/角色 |

### 后端

新增审批人解析服务：

- `WorkflowAssigneeResolver`
- 输入：节点属性、流程变量、发起人、机构上下文。
- 输出：assignee、candidateUsers、candidateGroups。

待办查询规则：

- 当前用户是 assignee。
- 当前用户在 candidateUsers。
- 当前用户属于 candidateGroups 对应角色/岗位/组织。

E2E：

- 指定成员审批。
- 指定角色审批。
- 指定岗位审批。
- 发起人审批。
- 非候选用户看不到待办。

## P2 表单运行时正式化

目标：动态表单和自定义表单都成为可维护、可版本化的运行时模型。

### 数据模型

| 表 | 说明 |
|---|---|
| `workflow_form_definition` | 表单主档，保存表单编码、名称、类型 |
| `workflow_form_version` | 表单版本，保存 form-create rules 或自定义表单配置 |
| `workflow_definition_form` | 流程定义与表单版本绑定关系 |

### 动态表单

- 支持基础组件运行时渲染。
- 支持业务组件只读/选择器接入：成员、组织、岗位、角色、字典、地区。
- 支持发起态、审批态、详情态字段权限。
- 支持字段 required、readonly、hidden。

### 自定义表单

- 绑定前端组件 key，不直接绑定任意 URL。
- 组件必须在 `componentsMap` 或工作流表单 registry 注册。
- 发起、审批、详情都必须实现约定接口：
  - `load`
  - `validate`
  - `submit`
  - `readonly`

E2E：

- 动态表单版本发布后，旧流程实例仍按旧版本展示。
- 自定义表单发起并进入审批。
- 字段权限：发起可编辑，审批只读或部分可编辑。

## P3 审批动作增强

目标：补齐常见 OA 工作流动作。

优先级：

1. 驳回到发起人。
2. 撤回，限发起人且下一节点未处理。
3. 转办，当前办理人转给另一个成员。
4. 委托，按时间段自动转交。
5. 加签，前加签/后加签。
6. 抄送，任务完成或节点到达时生成抄送记录。
7. 作废，发起人或管理员终止流程。

E2E：

- 每个动作至少一条独立 E2E。
- 每个动作必须验证权限边界：非办理人不可操作。

## P4 流程监控与运维

目标：平台或机构管理员能查看和处理异常流程。

能力：

- 流程实例列表。
- 流程实例详情。
- 流程图高亮当前节点。
- 审批轨迹。
- 挂起/激活实例。
- 终止实例。
- 重试失败服务任务。
- 管理员操作审计。

E2E：

- 管理员查看实例详情。
- 管理员终止流程。
- 普通用户无法终止不属于自己的流程。

## P5 跨机构协同

目标：支持保函业务等跨机构流程，但不打破机构隔离。

原则：

- 不允许直接穿透对方机构数据。
- 通过业务参与方授权生成跨机构可见范围。
- 跨机构任务进入对应机构的待办。
- 平台代维必须记录被代维机构、理由和操作链路。

E2E：

- A 机构发起，B 机构处理。
- 未参与机构不可见流程。
- 平台代维操作有审计。
