# 角色授权按钮节点展示交付契约

## 1. 目标

角色管理「分配权限」弹框正常展示后端菜单数据中的按钮节点，使角色可以直接勾选已有的按钮权限。

## 2. 范围

- 前端角色管理页面的分配权限弹框。
- 后端 `getAssignableMenus` 返回的菜单树在前端完整展示。
- 原有勾选、保存、回显链路保持不变。

## 3. 不做什么

- 不新增“当前页面”伪节点。
- 不新增页面访问与按钮权限联动规则。
- 不调整后端接口、数据库结构、登录权限返回格式。
- 不实现按钮数据展示规则。

## 4. 设计输入

- 用户确认采用方案一：不需要“当前页面”，正常渲染菜单数据中的按钮节点。
- 菜单数据中已经配置查询列表等按钮节点，前端不再过滤。

## 5. 设计说明

### 5.1 影响模块

- `mango-ui/packages/rbac/src/views/role/index.vue`

### 5.2 接口变化

无接口变化。

### 5.3 数据变化

无数据库结构和初始化数据变化。

### 5.4 菜单/页面/权限变化

- 角色授权弹框展示范围变化：由过滤按钮节点改为展示完整菜单树。
- 权限保存仍提交 Element Plus Tree 的已选和半选节点 ID。

### 5.5 测试范围

- `@mango/rbac` 包构建。
- 角色授权弹框按钮节点展示、勾选保存和回显由页面验收确认。

## 6. 风险与限制

- 按钮节点展示依赖后端菜单树已返回按钮节点。
- 页面级验收需要使用具备角色管理权限的账号操作确认。
