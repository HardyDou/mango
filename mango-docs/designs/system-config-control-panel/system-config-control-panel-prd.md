## 1. 需求范围

本次需求将 Mango 后台的“系统配置”和“参数配置”统一为一套“系统配置”能力，并提供一个可复用的前端通用配置面板组件。

本次包含：

- 合并用户心智：后台只保留“系统配置”概念，不再让运营用户区分“系统配置”和“系统参数”。
- 系统配置支持按业务域组织。调用方可以传入一个或多个业务域，页面按业务域展示配置。
- 新增前端通用配置面板组件。组件接收业务域列表，按业务域生成 Tab，每个 Tab 展示该业务域下的配置卡片。
- 配置卡片展示配置名称、当前状态或当前值、可操作项、配置介绍。
- 简单配置可在卡片上直接操作。
- 复杂配置可点击卡片后进入弹窗查看详情并编辑。
- 一期支持配置展示类型：开关、文本、数字、单选、下拉、多选、日期、日期区间。
- 支持业务模块在自己的配置入口复用该配置面板，在一个地方管理本业务域的运行时配置。
- 后台系统配置管理页使用统一后的系统配置体验，不再展示“系统参数 / 系统配置”两个重复 Tab。

## 2. 不处理范围

本次不处理：

- 不建设 CMS、通知、工作流等具体业务模块自己的配置业务规则。
- 不一次性完成所有高级配置类型，例如日期时间、JSON/Object、敏感配置、组合配置。
- 不设计门户站点或 C 端配置页面。
- 不改变业务域本身的创建、启停、层级维护规则。
- 不要求所有历史配置立即补齐完整说明和控件元数据；历史配置允许按兼容规则展示。
- 不处理配置变更审计、审批、发布版本、灰度生效和回滚能力。
- 不处理按租户差异化配置的产品规则；若现有系统已有租户隔离，则沿用现状。

## 7. 关键业务流程

### 7.1 BF-001 流程：按业务域查看系统配置

#### 流程目标

管理员在统一页面或业务模块配置入口中，按业务域查看该业务域下所有系统配置。

#### 适用场景

- 平台管理员进入系统配置页面。
- 业务模块嵌入配置面板，只展示指定业务域配置。

#### 前置条件

- 用户已进入有权访问的后台页面。
- 页面或组件已确定需要展示的业务域范围。

#### 操作步骤

1. 用户进入系统配置页面或业务模块配置区域。
2. 页面展示业务域 Tab。
3. 用户点击某个业务域 Tab。
4. 页面展示该业务域下的配置卡片。
5. 用户根据卡片上的名称、当前值和配置介绍判断配置状态。

#### 业务规则清单

##### 规则 BR-001：业务域范围规则

- 定义：配置面板只展示调用方传入业务域范围内的配置。
- 适用业务用户类型：平台管理员、业务管理员。
- 触发条件：配置面板加载或业务域 Tab 切换。
- 判断口径：业务域编码在调用方传入范围内，才展示对应配置。
- 处理方式：按业务域生成 Tab，未传入的业务域不展示。
- 用户反馈：用户只看到可管理业务域。
- 状态变化：无。
- 影响范围：配置面板展示范围。
- 例外情况：未传入业务域时，系统配置管理页可按管理页默认范围展示。
- 验收标准：见 AC-001、AC-002。

##### 规则 BR-002：业务域空态规则

- 定义：业务域下没有配置时，页面展示空态而不是空白或错误。
- 适用业务用户类型：平台管理员、业务管理员。
- 触发条件：切换到无配置项的业务域。
- 判断口径：当前业务域配置数量为零。
- 处理方式：展示空态文案和刷新入口。
- 用户反馈：用户看到“当前业务域暂无系统配置”。
- 状态变化：配置面板进入空态。
- 影响范围：当前业务域 Tab。
- 例外情况：加载失败时展示失败态，不展示空态。
- 验收标准：见 AC-005。

#### 状态变化

| 操作前 | 操作 | 操作后 |
|---|---|---|
| 未加载 | 进入页面 | 正常 / 空态 / 失败态 |
| 正常 | 切换业务域 | 正常 / 空态 / 失败态 |
| 失败态 | 点击重试 | 正常 / 空态 / 失败态 |

#### 异常与限制

- 业务域加载失败时，必须展示失败提示和重试入口。
- 配置介绍为空时，卡片仍需正常展示配置名称和当前值。

#### 验收标准

见 AC-001 至 AC-006。

### 7.2 BF-002 流程：编辑简单配置

#### 流程目标

管理员在配置卡片上直接修改简单配置值。

#### 适用场景

- 开关类配置，例如是否启用内容审批。
- 文本、数字、单选、日期、日期区间等可直接表达的配置。

#### 前置条件

- 配置项处于启用状态。
- 配置项允许编辑。
- 用户具备编辑该配置的业务条件。

#### 操作步骤

1. 用户在配置卡片上查看当前值。
2. 用户直接操作卡片上的开关、输入框、选择器或日期控件。
3. 用户提交或完成修改。
4. 页面反馈保存结果。
5. 卡片更新为最新状态。

#### 业务规则清单

##### 规则 BR-003：展示类型编辑规则

- 定义：配置项按展示类型决定卡片上的编辑控件。
- 适用业务用户类型：平台管理员、业务管理员。
- 触发条件：配置卡片渲染。
- 判断口径：展示类型为开关、文本、数字、单选、日期、日期区间。
- 处理方式：按类型展示对应控件；未知类型按文本和详情入口兼容。
- 用户反馈：用户看到与配置含义匹配的控件。
- 状态变化：无。
- 影响范围：配置卡片展示和编辑体验。
- 例外情况：配置不可编辑时，控件置灰或只读。
- 验收标准：见 AC-007、AC-008、AC-009。

##### 规则 BR-004：配置值保存反馈规则

- 定义：用户修改配置后必须看到明确成功或失败反馈。
- 适用业务用户类型：平台管理员、业务管理员。
- 触发条件：用户保存配置值。
- 判断口径：保存结果成功或失败。
- 处理方式：成功后刷新当前卡片值；失败时保留原值并展示失败原因。
- 用户反馈：成功显示保存成功，失败显示保存失败。
- 状态变化：成功后配置值更新。
- 影响范围：当前配置卡片。
- 例外情况：网络或服务不可用时展示失败态，不应假装保存成功。
- 验收标准：见 AC-010、AC-011。

##### 规则 BR-005：不可编辑配置规则

- 定义：不可编辑配置允许查看但不允许修改。
- 适用业务用户类型：平台管理员、业务管理员。
- 触发条件：配置项声明为不可编辑或用户不满足编辑条件。
- 判断口径：配置项不可编辑。
- 处理方式：卡片控件置灰，详情中展示不可编辑原因。
- 用户反馈：用户看到配置不可编辑。
- 状态变化：无。
- 影响范围：当前配置卡片。
- 例外情况：无。
- 验收标准：见 AC-012。

#### 状态变化

| 操作前 | 操作 | 操作后 |
|---|---|---|
| 启用 | 保存成功 | 启用且值更新 |
| 启用 | 保存失败 | 启用且保留原值 |
| 不可编辑 | 尝试修改 | 不可编辑 |

#### 异常与限制

- 数字配置输入非数字时，页面应提示输入不符合要求。
- 日期区间结束日期早于开始日期时，页面应提示输入不符合要求。
- 单选配置没有可选项时，应展示不可配置或缺少选项提示。

#### 验收标准

见 AC-007 至 AC-012。

### 7.3 BF-003 流程：查看和编辑配置详情

#### 流程目标

用户点击配置卡片后查看完整配置信息，并处理不适合卡片直接编辑的配置。

#### 适用场景

- 需要查看配置键、业务域、类型、说明、当前值、状态。
- 配置值较长或需要更多上下文说明。
- 后续扩展 JSON、多选、敏感配置等复杂类型。

#### 前置条件

- 配置卡片已展示。

#### 操作步骤

1. 用户点击配置卡片或“详情”入口。
2. 页面打开详情弹窗。
3. 用户查看完整配置信息和当前值。
4. 用户修改可编辑配置。
5. 用户保存或取消。

#### 业务规则清单

##### 规则 BR-006：详情完整性规则

- 定义：详情必须展示卡片未完整展示的信息。
- 适用业务用户类型：平台管理员、业务管理员。
- 触发条件：用户打开配置详情。
- 判断口径：详情至少展示配置名称、配置键、业务域、展示类型、当前值、配置介绍、状态。
- 处理方式：弹窗展示完整信息。
- 用户反馈：用户能判断配置用途和影响。
- 状态变化：无。
- 影响范围：配置详情。
- 例外情况：缺失说明时展示为空值含义。
- 验收标准：见 AC-013。

#### 状态变化

| 操作前 | 操作 | 操作后 |
|---|---|---|
| 卡片展示 | 打开详情 | 详情打开 |
| 详情打开 | 保存成功 | 详情关闭或保持打开，卡片刷新 |
| 详情打开 | 取消 | 回到卡片展示 |

#### 异常与限制

- 关闭未保存详情时，应保留原值。
- 保存失败时，详情不应展示成功状态。

#### 验收标准

见 AC-013、AC-014。
