# Issue 66 Verification Report

## Objective

Fix #66: enterprise projects generated by Mango CLI should not show 404 when users click a generated business directory menu.

## Change

The business module `resource-manifest.json` template now gives the generated directory menu an explicit redirect to its first runnable child menu.

Generated example:

- Directory path: `/procurement-order`
- Directory redirect: `/procurement-order/procurements`
- Child route: `/procurement-order/procurements`
- Child component: `procurement-order/procurement/index`

Both template sources were updated:

- `mango-business-starter/.../resource-manifest.json`
- `mango-ui/packages/mango-cli/templates/business-module/.../resource-manifest.json`

The CLI check script now asserts that generated business directory menus redirect to their first runnable child menu.

## Notes

This fix intentionally changes the generated manifest instead of adding a shell fallback for all component-less menus. That keeps missing component mistakes visible unless a directory explicitly declares its intended redirect.
