# 前端 Monorepo 架构规范

## 1. 分层

- `apps/*`：应用基座和部署单元。
- `packages/common`：公共能力。
- `packages/api-schema`：共享契约。
- `packages/*` 业务包：按领域拆分业务能力。

## 2. 依赖规则

- `apps` 可以依赖 `packages/*`。
- 业务包可以依赖 `common` 和 `api-schema`。
- 公共包禁止依赖 `apps`。
- 业务包之间禁止随意互相依赖。
- 禁止循环依赖。

## 3. 宿主职责

- 宿主只负责布局、路由聚合、权限拦截、全局初始化。
- 宿主不承载公共组件第二实现。
- 宿主不长期保留业务包已下沉的代码。

## 4. 公共能力规则

- 可复用组件进入对应 `packages/*`。
- 公共组件需要统一导出。
- 公共组件禁止直接依赖宿主 store、router、api、i18n。
- 公共组件依赖的 API 必须放在对应公共位置。

## 5. 路由规则

- 路由按模块组织。
- 动态路由集中处理。
- 跨包页面必须显式注册或有统一映射。

## 6. Mock 与真实接口

- Mock 和真实接口保持同一契约。
- 不允许开发态和构建态走两套不一致协议。

## 7. 迁移规则

- 下沉代码时同时迁移类型、样式、测试、导出。
- 迁移完成后删除原重复实现。
- 不保留“先复制，后面再删”的长期重复代码。

## 8. 交付要求

- `pnpm build` 通过
- `pnpm dev` 可启动
- 关键页面可访问
- 关键测试可执行

## 9. 禁止事项

- `packages/*` 反向依赖 `apps/*`
- 同一能力在宿主和公共包各维护一份
- 公共包直接引用宿主别名路径
