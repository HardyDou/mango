Admin UI 通用规范 ​

1. 目标 ​

Mango Admin UI 以系统级一致性为最高优先级。

相同语义、相同层级、相同功能的界面元素，必须在布局、尺寸、间距、颜色、文案、图标、命名和交互上保持一致。

2. 必须遵守 ​

  1. 同类页面必须使用统一结构，禁止每个页面自定义一套布局。
  2. 页面内容区不展示面包屑、页签标题、路由层级；这些属于框架能力。
  3. 非必要不覆盖 Element Plus 基础控件样式。
  4. 输入框、下拉、按钮、表格、弹框等基础控件尺寸由统一样式表控制。
  5. 页面私有样式只处理业务布局和特殊展示，不定义新的控件规范。
  6. 系统主色、语义色、圆角、阴影、间距必须来自统一变量或统一样式。
  7. 相同语义的操作必须使用相同图标，图标来自 Element Plus Icons 或项目统一图标库。
  8. 删除、移除、清空等危险操作统一使用 danger。
  9. 页面必须覆盖 loading、empty、error 和无权限状态。
  10. 特殊页面可以例外，但必须说明原因，基础控件仍尽量遵守统一样式。

3. 命名约定 ​

页面变量、方法、权限标识和按钮语义优先使用统一动作词：

中文英文
新增add
编辑edit
删除delete
查看view
查询search
重置reset
提交submit
保存save
取消cancel
返回back
启用enable
停用disable
导入import
导出export
刷新refresh

禁止同一语义混用多套命名，例如 add/create/new、view/detail/info、delete/remove/del。

4. 禁止事项 ​

  1. 禁止页面内硬编码新的颜色、控件高度、圆角、阴影、间距规则。
  2. 禁止使用 emoji、字符箭头、临时 SVG 替代统一图标。
  3. 禁止为了接口字段顺序或开发便利性牺牲业务阅读顺序。
  4. 禁止把页面能打开当作 UI 完成，必须检查布局、状态、交互和异常场景。
