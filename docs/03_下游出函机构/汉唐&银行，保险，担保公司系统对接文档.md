# <a id="_Toc32401"></a><a id="_Toc14693"></a>引言

本文档的目的是为了形成我方与各银行，保险，担保公司系统的接口规范，来指导各银行，保险，担保公司系统的程序开发、联调等工作。

本文档的主要内容是描述汉唐银行，保险，担保公司系统对接汉唐电子保函业务的接口规范

## <a id="_Toc8997"></a><a id="_Toc15741"></a>文档说明

本文档接口字段上传说明：对于汉唐银行，保险，担保公司系统校验错误返回的，我方会重新核实比对，并再次上传给平台，直至成功。

文档中接口字段必传表示：Y表示“必传”，CY表示“条件必传”，空白表示非必传。

## <a id="_Toc22560"></a><a id="_Toc15139"></a>关键名词解释

__订单号：前端渠道传入购买本次产品所产生的订单号。也是本次交易的唯一标识。__

__投保函号：投保成功后由汉唐银行，保险，担保公司系统生成的当前订单唯一对应的投保函，具有	法律效益的号码。__

__起保日期：被保人的起保时间，指在某一指定时间进行履行产品相关条款以及责任，即为起保日期。__

__终保日期：被保人的终保时间，指在某一指定时间进行结束终止产品相关条款以及责任，即为终保日期。__

__区间：被保人的被天数、月数、年数。__

__区间单位：D 天数，M月数，Y年数__

__份数：投保人购买的当前产品的份数。__

__保额：当前购买产品的总保额，总保额=基础保额\*份数\*人数\*（可选：保额因子）__

__保费：当前购买产品的总保费，总保费=基础保费\*份数\*人数\*（可选：保费因子）__

__投保人：投标企业名称__

__被保人：招标人__

__电子保函：对接渠道在汉唐银行，保险，担保公司系统投保成功后，汉唐银行，保险，担保公司系统会出具生成具有法律效益的电子版的单，简称：电子保函。__

__电子保函：对接渠道在汉唐银行，保险，担保公司系统投保成功后，汉唐银行，保险，担保公司系统会出具生成具有法律效益的电子版的保函，简称：电子保函。__

__注销时间：保函注销请求时间__

### <a id="_Toc25383"></a>关于公私钥：

保司会生成对应的公私钥信息，公钥会线下颁发给渠道接入方，用于业务请求加密。

渠道接入方可自己生成密钥对，将公钥线下提供给保司，接口请求时，渠道接入方使用生成的私钥使用SHA1withRSA算法对数据签名，保司以渠道方提供的公钥对数据进行验签。
