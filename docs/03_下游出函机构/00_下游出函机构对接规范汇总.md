# 下游出函机构对接规范汇总

## 一、对接模式总览

```
┌─────────────────────────────────────────────────────────────────┐
│                       电子保函服务平台                           │
│                    (需适配多种出函机构)                          │
└───────────────────────────┬─────────────────────────────────────┘
                            │
              ┌─────────────┼─────────────┐
              ▼             ▼             ▼
           银行A        保险公司       担保公司
          (直开/分离式)   (保证保险)    (担保保函)
```

---

## 二、出函机构类型

### 1. 机构分类

| 机构类型 | 产品 | 特点 |
|----------|------|------|
| **银行** | 银行保函(直开/分离式) | 银行直接开具或委托担保公司 |
| **保险公司** | 保证保险保单 | 保险产品，需银保监会批复 |
| **担保公司** | 担保保函 | 融资性/非融资性担保资质 |

### 2. 业务模式

```
银行保函模式:
┌─────────────────────────────────────────────────────────────┐
│                    分离式银行保函                            │
│  投标人 → 担保公司 → 银行 → 公共资源交易中心               │
│         (担保公司提供担保，银行出具保函)                     │
└─────────────────────────────────────────────────────────────┘

保证保险模式:
┌─────────────────────────────────────────────────────────────┐
│                    保证保险保单                               │
│  投标人 → 保险公司 → 公共资源交易中心                       │
│         (保险公司出具保险单)                                  │
└─────────────────────────────────────────────────────────────┘

直开银行保函模式:
┌─────────────────────────────────────────────────────────────┐
│                    直开式银行保函                            │
│  投标人 → 银行 → 公共资源交易中心                           │
│         (银行直接授信并开具保函)                             │
└─────────────────────────────────────────────────────────────┘
```

## 三、数据交互方式

### 1. 通讯协议

| 协议 | 说明 |
|------|------|
| HTTPS + JSON | 主流方式 |
| HTTPS + XML | 部分银行使用 |
| SFTP文件交换 | 异步批量处理 |
| 专线VPN | 高安全要求 |

### 2. 常见银行对接方式

| 银行 | 对接方式 | 特点 |
|------|----------|------|
| 中国光大银行 | SDK/直连接口 | 报文规范完整 |
| 招商银行 | API接口 | 支持电子签章 |
| 浦发银行 | 银企直连 | 需网银开通 |
| 建设银行 | API接口 | 支持分离式 |
| 农业银行 | 内部系统 | 需申请 |

### 3. 常见保险公司对接方式

| 保险公司 | 产品 | 特点 |
|----------|------|------|
| 太平洋保险 | 投标保证保险 | 需授信 |
| 平安保险 | 保证保险 | 支持电子保单 |
| 中华联合 | 建设工程保证保险 | 专注建工 |
| 大地保险 | 投标保证保险 | 多省对接 |
| 大家财产 | 投标保证保险 | 新进入者 |

## 四、核心接口字段汇总

### 1. 保函开立申请 (光大银行 LGA003)

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ChnlNo | string(10) | 是 | 业务渠道 |
| CustomerNo | string(10) | 是 | 客户号(我行) |
| TransCode | string(6) | 是 | 交易码 LGA003 |
| UnitCode | string(18) | 是 | 申请人统一社会信用代码 |
| BchCod | string(10) | 是 | 业务机构(银行分配) |
| BatchID | string(50) | 是 | 交易流水号 |
| GaurType | string(2) | 是 | 保函类型 01-投标 02-履约 |
| GaurAmt | decimal(18,2) | 是 | 保函金额 |
| GaurBeginDate | date | 是 | 保函生效日期 |
| GaurEndDate | date | 是 | 保函到期日期 |
| ProjNo | string(50) | 是 | 项目编号 |
| TenderNo | string(50) | 是 | 招标编号 |
| BidBondNo | string(50) | 是 | 标段编号 |
| ApplyName | string(200) | 是 | 申请人名称 |
| BenName | string(200) | 是 | 受益人名称 |
| BenCode | string(18) | 是 | 受益人代码 |
| GaurContent | string(2000) | 是 | 保函内容 |

### 2. 保函查询接口 (光大银行 LGA008)

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| TransCode | string(6) | 是 | 交易码 LGA008 |
| BatchID | string(50) | 是 | 原交易流水号 |
| GaurNo | string(50) | 否 | 保函编号 |
| UnitCode | string(18) | 否 | 申请人代码 |

### 3. 保函注销申请 (光大银行 LGA010)

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| TransCode | string(6) | 是 | 交易码 LGA010 |
| BatchID | string(50) | 是 | 交易流水号 |
| GaurNo | string(50) | 是 | 保函编号 |
| CancelType | string(2) | 是 | 注销类型 1-正常注销 2-提前注销 |
| CancelReason | string(500) | 是 | 注销原因 |

### 4. 银行通用响应字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| TransCode | string(6) | 交易码 |
| RtnCode | string(10) | 返回码 0000-成功 |
| RtnMsg | string(200) | 返回信息 |
| BatchID | string(50) | 流水号 |
| GaurNo | string(50) | 保函编号(开立成功后返回) |
| GaurState | string(2) | 保函状态 |

### 5. 保险公司保单接口

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| policyNo | string(50) | 是 | 保单号 |
| insurerName | string(200) | 是 | 保险公司名称 |
| applicantName | string(200) | 是 | 投保人名称 |
| applicantCode | string(18) | 是 | 投保人统一社会信用代码 |
| beneficiaryName | string(200) | 是 | 受益人名称 |
| beneficiaryCode | string(18) | 是 | 受益人统一社会信用代码 |
| premium | decimal(18,2) | 是 | 保费金额 |

| startDate | datetime | 是 | 保险生效时间 |
| endDate | datetime | 是 | 保险到期时间 |
| projectName | string(500) | 是 | 项目名称 |

## 五、电子保函文件格式

### 1. 文件格式要求

| 格式 | 说明 | 使用场景 |
|------|------|----------|
| **PDF** | 主流格式，需签章 | 通用 |
| **OFD** | 国产化格式 | 政务系统 |
| **XML** | 结构化数据 | 数据交换 |
| **OFD+电子签章** | 国产化+签名 | 政务信创 |

### 2. 电子签章要求

| 要求 | 说明 |
|------|------|
| 文档签名 | 必须覆盖整个文档 |
| 签章位置 | 页面右下角或指定位置 |
| CA证书 | 使用权威CA机构颁发 |
| 时间戳 | 包含可信时间戳 |

### 3. 保函文件结构

```
OFD/PDF电子保函文件结构:
┌─────────────────────────────────────────────────────────────┐
│                     电子保函文件                            │
├─────────────────────────────────────────────────────────────┤
│  1. 保函封面                                                  │
│     - 保函编号                                                  │
│     - 开具日期                                                  │
│     - 有效期                                                    │
├─────────────────────────────────────────────────────────────┤
│  2. 保函正文                                                  │
│     - 投标保证内容                                             │
│     - 受益人信息                                               │
│     - 项目信息                                                 │
├─────────────────────────────────────────────────────────────┤
│  3. 担保条款                                                  │
│     - 担保金额                                                 │
│     - 有效期                                                   │
│     - 索赔条件                                                 │
├─────────────────────────────────────────────────────────────┤
│  4. 电子签章                                                  │
│     - 出函机构电子章                                           │
│     - 法定代表人章                                             │
│     - 时间戳                                                   │
└─────────────────────────────────────────────────────────────┘
```

## 七、资质要求

### 1. 银行准入条件

| 要求 | 说明 |
|------|------|
| 金融许可证 | 有效的银行金融许可证 |
| 业务资质 | 具备保函业务资质 |
| 系统要求 | 有成熟的电子保函系统 |
| 安全认证 | 通过相关安全认证 |

### 2. 保险公司准入条件

| 要求 | 说明 |
|------|------|
| 保险许可证 | 有效的保险业务许可证 |
| 保证保险产品 | 已备案的保证保险产品 |
| 银保监会批复 | 具备相关业务批复 |
| 系统对接 | 完成技术系统对接 |

### 3. 担保公司准入条件

| 要求 | 说明 |
|------|------|
| 担保牌照 | 融资性/非融资性担保许可证 |
| 注册资本 | 符合监管要求 |
| 信用评级 | 达到一定信用等级 |
| 银行授信 | 已获得银行授信额度 |
