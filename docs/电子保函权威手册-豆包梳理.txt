# 第一篇 基础定义与核心概念

## 1.1 电子保函定义

电子保函是依据《民法典》《电子签名法》《招标投标法》，由**持牌金融机构**出具，以电子数据为载体、具备法律效力、用于替代**现金保证金**的担保凭证。广泛应用于公共资源交易、工程建设、政府采购、土地出让等领域。

## 1.2 核心价值

- 替代现金保证金，释放企业资金压力

- 全程线上化、可追溯、防篡改、安全合规

- 交易中心统一监管，信用约束更强

## 1.3 全参与角色及职责

1. **招标人**：项目发起方，提出保证金要求，拥有索赔权利

2. **投标人**：保函申请人、付费方

3. **公共资源交易中心**：监管机构、规则制定方、机构准入方、保函核验方

4. **出函金融机构**：银行、融资担保公司、保险公司

5. **交易系统厂商**：新点4.0、广联达、福莱特、筑龙

6. **技术服务商**：为金融机构提供出函系统、风控、加解密服务（你的定位）

7. **CA机构**：提供数字证书、电子签章、国密安全支撑

# 第二篇 保函业务类型全体系

## 2.1 按业务用途分类

1. **投标保函**

投标阶段使用，保证不弃标、不造假、中标后签约，开标后自动失效。

1. **履约保函**

中标后使用，保证按合同完工，金额通常为合同额的10%。

1. **预付款保函**

保证业主预付款专款专用、按约施工。

1. **质量保函**

保证工程在质保期内履行维修责任。

1. **农民工工资支付保函**

保证按时足额支付农民工工资，强制类险种。

## 2.2 按出函主体分类（权威标准）

### （1）银行保函

- 出具主体：商业银行、政策性银行

- 资质要求：金融许可证

- 认可度：**最高**

- 包含两种模式：

    - **银行自营保函**：银行直接审核、直接出函

    - **银行分离式保函**：银行出函 + 融资担保公司操盘（见2.4）

### （2）融资担保公司保函

- 出具主体：**持《融资担保业务经营许可证》**的融资担保公司

- 市场占比：**70%以上，行业绝对主流**

- 可直接入围交易中心、直接出函、直接承担赔付责任

### （3）保证保险（保险保函）

- 出具主体：财产保险公司

- 产品名称：**投标保证保险**

- 形式：保险单，法律效力等效保函

- 监管：银保监会

## 2.3 按法律形式分类

- 一般保证

- **连带责任保证**（公共资源交易强制要求）

## 2.4 分离式保函专项说明

### 定义

**分离式保函 = 银行出函 + 融资担保公司负责业务全流程**

是银行与持牌融资担保公司的专属合作模式。

### 核心逻辑

1. 银行给予**融资担保公司**专项授信额度

2. 担保公司推荐客户、审核风险

3. 银行使用担保公司额度，以**银行名义**出具保函

4. 保函效力 = **标准银行保函**

5. 投标人仅对接担保公司，无需对接银行

### 重要结论

**非融资性担保公司不能出电子保函**，无牌照、无授信、无法入围、银行不合作。

# 第三篇 保函文件格式标准

## 3.1 PDF 格式

- 通用可视化格式，支持电子签章、兼容性最强

## 3.2 OFD 格式

- 国家版式标准（GB/T 33190-2016），政务/交易中心优先使用

## 3.3 XML 格式

- 结构化数据，用于接口自动解析、系统校验

## 3.4 P7B / PKCS7 加密格式

- 国密加密签名容器，用于验真、防篡改、法律存证

## 3.5 行业标准组合

- **展示层：OFD + PDF**

- **传输层：XML + 加密容器**

- **验签层：P7B/PKCS7**

# 第四篇 按是否加密分类：明文函 vs 密文函（新点4.0精准对齐）

## 4.1 明文函（普通电子保函）

### 定义

保函内容**不加密、可直接查看**，各方均可阅读全文。

### 适用场景

- 履约保函、预付款保函、质量保函、农民工工资保函

### 特点

- 结构透明

- 无需密钥即可查看

- 仅做电子签名防篡改，不做内容加密

## 4.2 密文函（加密电子保函·新点4.0标准）

### 定义

**仅投标保函使用**。开标前，核心标段/招标人信息加密，金融机构按密文出函；开标后由交易中心推送解密Key，机构还原为明文。

**目的：防止开标前泄露投标人信息、围标串标。**

### ✅ 新点4.0 必加密字段（接口官方规范）

1. **标段编号（biaoduanno）**

2. **标段名称（biaoduanname）**

3. **招标人（zbr）**

4. **招标人统一社会信用代码（zbrorgnum）**（可选）

### ✅ 新点4.0 不加密字段（接口原文）

- 业务流水号（applyno）

- 投标企业名称（biddername）

- 统一社会信用代码（biddercode）

- 保证金金额（bzjamount）

- 保险公司code（insurecode）

- 接口公共参数（appkey、timestamp、sign）

### ✅ 加密算法规范

- 报文内容加密：**国密SM4**

- 密钥加密传输：**国密SM2**

- 防篡改签名：**国密SM3**

### ✅ 完整密文流程（新点4.0）

1. **开标前**

    - 中心推送加密后的标段/招标人信息

    - 金融机构生成**密文保函**（仅上述4字段密文）

    - 保函密文存储，敏感信息不可见

2. **开标后**

    - 中心调用**还原保函（baohanrestore）**接口

    - 推送applyno、baohanno、kaibiaotime、**解密Key**

    - 系统自动解密 → 生成明文保函

    - 明文保函入库归档

## 4.3 适用总结

- **明文函**：履约、预付款、质量、工资保函（全程明文）

- **密文函**：**仅投标保函**（开标前密文、开标后解密）

# 第五篇 准入与入围体系

## 5.1 可入围机构（仅三类）

1. 商业银行（金融许可证）

- 资质审核：牌照、注册资本、评级、合规记录

## 5.3 技术服务商定位

- 不具备入围资格

- 为金融机构提供**出函系统**

- 由金融机构以自身名义入围对接

# 第六篇 授信、额度、反担保体系

## 6.1 授信模式

1. **银行授信模式（主流）**

银行给予担保公司/保险机构总额度 → 中心备案 → 逐笔占用/释放

1. **资金监管模式**

机构存入保证金到监管户，按冻结额度出函

1. **主体信用模式**

大型机构凭自身信用获得中心认可额度

## 6.2 额度生命周期

总额度 → 可用额度 → 冻结/占用 → 出函生效 → 到期/开标 → 释放额度

## 6.3 反担保机制

- 企业反担保

- 实际控制人连带责任

- 第三方担保

- 保证金反担保

# 第七篇 支付、保费、结算、计费模式

## 7.1 保费计价模式（行业4种标准）

### （1）按保额比例计费（最主流）

- 公式：保函金额 × 约定费率

- 投标保函费率：**0.2%～1.5%**

### （2）固定金额计费（小额项目）

- 一口价：300/500/800/1000元

### （3）阶梯比例计费（大额项目）

- 金额越大，费率越低

### （4）按时间计费（长期保函）

- 公式：保额 × 年化费率 × 期限

## 7.2 支付模式

1. **交易中心代收（90%地区）**

2. **金融机构直收**

3. **线下对公转账**

## 7.3 结算规则

- T+1、周结、月结

- 自动对账、清分、差错处理

## 7.4 退费规则

- 未支付：无费用

- 已支付未出函：全额退

- 已出函未开标：按规则退

- 已开标/违约：不退

# 第八篇 全业务流程 + 完整数据流（闭环无断点）

## 阶段1：项目与准入准备

1. 招标人发布项目，中心备案

2. 金融机构入围、额度备案、接口联调

## 阶段2：投标人申请

1. 投标人选择项目 → 电子保函 → 选择金融机构

2. **数据流1：中心 → 金融机构（申请推送）**

## 阶段3：风控审核

1. 机构自动审核：工商、失信、司法、黑名单、额度

2. **数据流2：金融机构 → 中心（审核结果）**

## 阶段4：支付

1. 投标人支付保费

2. **数据流3：中心 → 金融机构（支付结果）**

## 阶段5：出函（密文/明文分支）

1. **投标保函（密文函）**

    - 接收中心加密的标段/招标人信息

    - 生成密文保函（仅4个核心字段加密）

    - 签章、加密、组装报文

2. **其他保函（明文函）**

    - 直接生成明文保函

3. **数据流4：金融机构 → 中心（出函结果）**

## 阶段6：生效与投标

1. 中心验签、核验

2. 投标保函：保持密文状态，标记生效

3. 其他保函：直接明文生效

4. **数据流5：中心 → 金融机构（生效通知）**

## 阶段7：开标与终结

1. 项目开标

2. **投标保函**

    - 中心调用**还原保函**接口，推送解密Key

    - 系统自动解密 → 生成明文保函

    - 明文归档

3. 中心同步开标结果

4. **数据流6：中心 → 金融机构（开标结果）**

5. 额度释放、保函失效、归档

# 第十篇 国密加解密与签名验签标准

## 10.1 国密算法体系

- **SM2**：签名、验签、非对称加密（密钥加密）

- **SM3**：哈希摘要、防篡改（签名）

- **SM4**：大文件/内容加密（报文加密）

## 10.2 标准加密流程

1. 文件/内容用 **SM4** 加密

2. SM4密钥用 **SM2（中心公钥）** 加密

3. 整包用 **SM2（机构私钥）** 签名

## 10.3 密文函专用流程（新点4.0）

1. 对指定4个字段（标段/招标人）做 **SM4** 加密

2. 对SM4密钥做 **SM2** 公钥加密

3. 密文报文 + 签名 → 推送交易中心

4. 开标后用中心私钥解密，恢复明文

## 10.4 证书与密钥

- 公钥：备案、公开、验签使用

- 私钥：保密、签名使用

- CA证书：确保身份真实有效
