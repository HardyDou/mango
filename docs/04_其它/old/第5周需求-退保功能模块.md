**状态**: 已优化

---

## 1. 业务概述

退保功能允许投标人（投保人）在基础合同履行完毕后，申请解除担保责任并退还保费或释放保证金。

### 1.1 退保场景

| 场景 | 说明 | 适用保函类型 |
|------|------|-------------|
| 合同履行完毕 | 基础合同已履行完成，正常退保 | 全部 |
| 部分履行退保 | 合同部分履行，按比例退保 | 全部 |
| 司法判决退保 | 法院判决需要退保 | 全部 |
| 提前解除退保 | 双方协商提前解除合同 | 全部 |

### 1.2 退保类型

| 类型 | 说明 | 退款规则 |
|------|------|---------|
| 見费出单 | 保费已缴纳 | 全额退还保费 |
| 见费不出单 | 保费未缴纳 | 无需退款，释放保证金 |
| 见费后出单 | 已出函收取保费 | 按剩余天数折算退款 |

## 2. 功能需求

### 2.1 退保申请

**功能描述**: 投标人提交退保申请

**用户角色**: 投标人（投保人）

**前置条件**:
- 保函状态为"已出具"或"生效中"
- 距保函到期不少于X个工作日（可配置）
- 无未完成的退保申请

**业务规则**:
1. 退保申请需提交退保原因和相关证明材料
2. 系统自动计算可退金额（基于保函类型、剩余天数）
3. 退保申请需经过金融机构审核

**数据项**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 保函编号 | string | 是 | 关联的保函编号 |
| 退保原因 | enum | 是 | 合同履行完毕/部分履行/司法判决/提前解除 |
| 退保说明 | string | 否 | 补充说明 |
| 证明材料 | file[] | 是 | 合同履行证明/法院文书等 |
| 退款账户 | object | 是 | 退款银行账户信息 |
| 联系人 | string | 是 | 退保申请人 |
| 联系电话 | string | 是 | 联系电话 |

### 2.2 退保审核

**功能描述**: 金融机构审核退保申请

**用户角色**: 金融机构审核人员

**前置条件**:
- 存在待审核的退保申请
- 审核人员有退保审核权限

**业务规则**:
1. 审核人员可查看退保申请详情和证明材料
2. 支持通过或拒绝退保申请
3. 拒绝需填写拒绝原因
4. 可选择退款方式（银行转账/保证金释放）

**审核流程**:
```
退保申请提交 → 金融机构初审 → 金融机构复审 → 执行退款/释放保证金 → 退保完成
```

### 2.3 退保费用计算

**功能描述**: 系统自动计算应退金额

**计算规则**:

| 退款类型 | 计算公式 |
|----------|---------|
| 见费出单 | 退费金额 = 已缴保费 × (剩余天数 / 总保障天数) × 退费比例 |
| 见费后出单 | 退费金额 = 已缴保费 - (已保障天数 × 日均保费) |
| 部分履行 | 退费金额 = 已缴保费 × (1 - 已履行比例) |

**退费比例**:
- 合同履行完毕：100%
- 部分履行：按实际履行比例计算
- 司法判决：按判决书执行
- 提前解除：按合同约定或双方协商

### 2.4 退保进度查询

**功能描述**: 投标人查询退保处理进度

**用户角色**: 投标人

**展示信息**:
- 退保申请基本信息
- 当前审核状态
- 审核历史记录
- 预计退款时间

## 3. 接口需求

### 3.1 退保申请接口

**接口路径**: `POST /api/v1/surrender/apply`

**请求参数**:
```json
{
  "guaranteeNo": "string",        // 保函编号
  "surrenderReason": "string",   // 退保原因
  "surrenderDesc": "string",     // 退保说明
  "attachments": [
    {
      "type": "string",          // 材料类型
      "url": "string"           // 材料URL
    }
  ],
  "refundAccount": {
    "bankName": "string",        // 开户行
    "accountName": "string",     // 账户名
    "accountNo": "string"        // 账号
  },
  "contact": "string",           // 联系人
  "contactPhone": "string"      // 联系电话
}
```

**响应参数**:
```json
{
  "code": "200",
  "message": "success",
  "data": {
    "surrenderNo": "string",     // 退保申请编号
    "refundAmount": "number",    // 预计退款金额
    "applyTime": "string"        // 申请时间
  }
}
```

### 3.2 退保审核接口

**接口路径**: `POST /api/v1/surrender/{surrenderNo}/audit`

**请求参数**:
```json
{
  "auditResult": "string",       // 审核结果：PASS/REJECT
  "auditOpinion": "string",      // 审核意见
  "refundMethod": "string"       // 退款方式：TRANSFER/RELEASE
}
```

### 3.3 退保进度查询接口

**接口路径**: `GET /api/v1/surrender/{surrenderNo}/status`

**响应参数**:
```json
{
  "surrenderNo": "string",       // 退保编号
  "guaranteeNo": "string",       // 保函编号
  "status": "string",            // 状态：PENDING/AUDITING/APPROVED/REJECTED/COMPLETED
  "refundAmount": "number",      // 实际退款金额
  "auditRecords": [
    {
      "auditor": "string",
      "result": "string",
      "opinion": "string",
      "time": "string"
    }
  ]
}
```

## 7. 业务规则补充

### 7.1 退保手续费收取规则

| 退保类型 | 手续费标准 | 最低 | 最高 |
|----------|-----------|------|------|
| 合同履行完毕 | 退费金额的1% | 100元 | 1000元 |
| 部分履行 | 退费金额的1.5% | 100元 | 1000元 |
| 司法判决 | 免费 | - | - |
| 提前解除 | 退费金额的2% | 100元 | 1000元 |

**说明**：
- 手续费从应退金额中扣除
- 司法判决退保免收手续费
- 投保时间超过1年的，提前解除免收手续费

### 7.2 退款周期要求

| 退款方式 | 处理周期 | 说明 |
|----------|---------|------|
| 银行转账 | 7个工作日内 | 审核通过后 |
| 保证金释放 | 3个工作日内 | 审核通过后 |

### 7.3 退保审核层级

- 初审：金融机构客服人员
- 复审：金融机构风控人员
- 大额退保（>50万）：需部门负责人审批
