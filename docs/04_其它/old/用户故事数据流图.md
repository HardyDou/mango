```
      │ 2.填写项目信息     │                   │                   │
      │ 3.上传申请材料     │                   │                   │
      │ 4.查看保费计算     │                   │                   │
      │ 5.提交投保申请     │                   │                   │
      │                   │                   │                   │
      │◀──────┐          │                   │                   │
      │        │          │                   │                   │
      │ 申请编号           │                   │                   │
      │ 投保状态           │                   │                   │
```

**数据流详情**：

| 步骤 | 数据输入 | 处理过程 | 数据输出 |
|------|---------|---------|---------|
| 1 | 保函类型选择 | 验证保函类型 | 保函类型ID |
| 2 | 项目信息表单 | 数据校验 | 项目信息JSON |
| 3 | 材料文件上传 | 文件存储(OSS) | 文件URL |
| 4 | 保额/期限 | 保费计算引擎 | 保费金额 |
| 5 | 完整投保数据 | 创建投保记录 | 投保申请ID |

### 2.2 投保申请数据流（交易系统推送）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   交易系统   │────▶│  API网关    │────▶│  电子保函   │────▶│   数据库    │
│ (投标平台)   │     │            │     │    系统     │     │            │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
      │                   │                   │                   │
      │ 1.选择电子保函     │                   │                   │
      │ 2.选择标段/项目    │                   │                   │
      │ 3.填写投保信息     │                   │                   │
      │ 4.提交投保申请     │                   │                   │
      │ (API推送)         │ 验签/鉴权          │                   │
      │                   │                   │                   │
      │◀──────┐          │                   │                   │
      │        │          │                   │                   │
      │ 投保申请ID         │                   │                   │
      │ 投保状态           │                   │                   │
```

**数据流详情**：

| 步骤 | 数据输入 | 处理过程 | 数据输出 |
|------|---------|---------|---------|
| 1 | 交易平台选择保函入口 | 跳转/嵌入 | 电子保函系统页面 |
| 2 | 标段/项目信息 | 数据同步 | 项目信息ID |
| 3 | 投保人信息+项目信息 | API调用 | 投保申请数据 |
| 4 | 完整投保数据 | 验签+创建记录 | 投保申请ID |
| - | 回调通知 | 推送状态 | 交易平台接收状态通知 |

**重要说明**：此场景是**最常见的投保方式**，投标人通过其常用的交易平台（如公共资源交易中心、招标投标平台）发起投保申请，数据通过API接口推送至电子保函系统。这种方式的特点是：
- 投标人无需单独注册/登录电子保函系统
- 数据一次录入，多方同步
- 交易平台与电子保函系统数据实时互通

**API接口详情**：

| 接口类型 | 接口路径 | 数据方向 | 说明 |
|---------|---------|---------|------|
| 投保申请 | POST /api/v1/applications | 交易系统→电子保函 | 推送投保申请数据 |
| 投保查询 | GET /api/v1/applications/{id} | 交易系统→电子保函 | 查询投保进度 |
| 保函下载 | GET /api/v1/bonds/{id}/download | 交易系统→电子保函 | 获取保函文件 |
| 状态回调 | POST /api/v1/callbacks/status | 电子保函→交易系统 | 状态变更通知 |
| 保费计算 | POST /api/v1/premium/calculate | 交易系统→电子保函 | 实时保费试算 |

## 七、系统外部接口

| 外部系统 | 接口方向 | 数据内容 | 接入方式 |
|---------|---------|---------|---------|
| 交易平台 | 双向 | 投保申请、保函数据、状态同步 | API接口（RESTful） |

| 财务系统 | 双向 | 退款、发票、对账 | API接口 |
| 短信/邮件服务 | 出 | 通知推送 | API接口 |
