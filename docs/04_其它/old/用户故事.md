## 一、投标人（投标方）

### 1.1 投保申请

**用户故事**：作为投标人，我希望能够在线提交保函申请，以便快速办理电子保函业务。

**详细描述**：
- 我需要选择保函类型（投标/履约/预付款/质量）
- 我需要填写项目信息和保函金额
- 我需要上传申请材料（营业执照、招标文件等）
- 我需要查看保费计算结果
- 我需要提交投保申请并获得申请编号

**验收标准**：
- [ ] 能够选择4种保函类型
- [ ] 能够填写完整的投保信息
- [ ] 能够上传申请材料
- [ ] 能够实时查看保费金额
- [ ] 提交后能够获得申请编号

### 1.2 投保进度查询

**用户故事**：作为投标人，我希望能够实时查看投保进度，以便了解保函办理状态。

**详细描述**：
- 我需要查看当前投保状态（待审核/审核中/已出函/已拒绝）
- 我需要查看审核记录和审核意见
- 我需要收到状态变更通知

**验收标准**：
- [ ] 能够查看投保状态
- [ ] 能够查看审核记录
- [ ] 能够收到状态变更通知

### 1.3 保函下载

**用户故事**：作为投标人，我希望能够下载电子保函文件，以便用于投标或合同签订。

**详细描述**：
- 我需要查看已出具的保函信息
- 我需要下载OFD格式的电子保函
- 我需要验证保函真伪

**验收标准**：
- [ ] 能够查看保函详情
- [ ] 能够下载电子保函文件
- [ ] 能够进行保函验真

### 1.4 退保申请

**用户故事**：作为投标人，我希望能够申请退保，以便在需要时终止保函业务。

**详细描述**：
- 我需要提交退保申请
- 我需要选择退保原因
- 我需要查看退保进度
- 我需要收到退款通知

**验收标准**：
- [ ] 能够提交退保申请
- [ ] 能够选择退保原因
- [ ] 能够查看退保进度
- [ ] 能够收到退款通知

### 1.5 发票管理

**用户故事**：作为投标人，我希望能够申请开具保费发票，以便进行财务报销。

**详细描述**：
- 我需要申请开具发票
- 我需要选择发票类型（纸质/电子）
- 我需要查看发票开具状态

**验收标准**：
- [ ] 能够申请开具发票
- [ ] 能够选择发票类型
- [ ] 能够查看发票状态

## 五、用户故事优先级

### P0（必须实现）

| 角色 | 用户故事 |
|------|---------|
| 投标人 | 投保申请、投保进度查询、保函下载 |
| 担保公司 | 机构管理、投保管理、审核管理、出函管理 |
| 出函机构 | 机构接入、投保审核、保函出具 |
| 系统管理员 | 用户管理、角色权限、机构管理 |

### P1（重要功能）

| 角色 | 用户故事 |
|------|---------|
| 投标人 | 退保申请、发票管理 |
| 担保公司 | 保函管理、退保管理、报表统计 |
| 出函机构 | 保函管理、财务对账 |
| 系统管理员 | 系统配置、日志审计、接口管理 |

### P2（增强功能）

| 角色 | 用户故事 |
|------|---------|
| 投标人 | 保函转让、保函延期 |
| 担保公司 | 数据分析 |
| 出函机构 | 自动化审核 |
| 系统管理员 | 性能监控 |
