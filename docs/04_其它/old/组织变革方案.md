# 电子保函项目组织变革方案

## 一、背景

结合AI公司特点：
- 招聘 = 创建 AGENTS
- 一人可以7*24小时工作
- 一件事情有一个人做就行

## 二、组织架构

### 2.1 岗位设置

| 岗位 | 数量 | 汇报对象 | 职责 |
|------|------|----------|------|
| CEO | 1 | - | 公司战略决策、重大审批 |
| CTO | 1 | CEO | 技术架构、技术选型、研发团队管理 |
| 后端开发工程师 |若干 | CTO | 后端开发、技术实现 |
| 前端开发工程师 |若干 | CTO | 前端开发、界面实现 |
| 移动端开发工程师 |若干 | CTO | iOS/Android开发 |
| 测试工程师 |若干 | CTO | 测试、质保 |
| 产品经理 |若干 | CTO | 产品规划、需求管理 |

### 2.2 汇报关系

```
CEO
 └── CTO
      ├── 后端开发工程师
      ├── 前端开发工程师
      ├── 移动端开发工程师
      ├── 测试工程师
      └── 产品经理
```

## 三、岗位职责与权限

### 3.1 CEO
- **职责**：公司战略决策、重大业务审批
- **权限**：
  - 审批招聘需求
  - 审批重大项目决策
  - 最终业务决策权

### 3.2 CTO
- **职责**：
  - 技术架构设计和技术选型
  - 组建和管理研发团队
  - 带领团队完成产品研发
  - 理解电子保函业务需求，设计并实现系统解决方案
- **权限**：
  - 提出招聘需求
  - 分配研发任务
  - 技术决策权

### 3.3 团队成员（开发/测试/产品）
- **职责**：根据岗位完成具体研发/测试/产品工作
- **权限**：在CTO分配的范围内开展工作

## 四、招聘流程

### 4.1 流程概述

```
团队负责人(CTO)提出招聘需求 → CEO审批 → 完成招聘
```

### 4.2 具体流程

1. **需求提出**：CTO根据项目需要提出招聘需求
2. **CEO审批**：CEO审批招聘需求
3. **招聘执行**：由CTO负责招聘执行（发布招聘信息、筛选简历、安排面试）
4. **入职**：新成员加入团队

### 4.3 招聘方式

- 招聘 = 创建 AGENTS（AI员工）
- 由CTO通过Paperclip系统创建新的agent

## 五、非必要岗位调整

### 5.1 取消的岗位

- **HR**：招聘工作由CTO直接负责，不再设立专门HR岗位
- **行政**：日常行政事务由各岗位自行处理或CEO协助

### 5.2 理由

- AI公司人效高，一人可完成多人工作
- 扁平化管理减少沟通成本
- 招聘流程简化，由团队负责人直接负责

## 六、联系机制

### 6.1 内部沟通

- 通过Paperclip系统进行任务分配和沟通
- 使用@mention触发相关agent响应

### 6.2 审批流程

- 招聘审批：CTO申请 → CEO审批
- 重大决策：团队负责人提案 → CEO审批
