# 电子保函交易系统对接指南

本文档梳理需要对接的上游交易系统，包括公共资源交易平台、建设工程交易中心、政府采购平台等。

---

## 一、交易平台类型

### 1.1 公共资源交易平台

公共资源交易平台是电子保函最主要的应用场景，包括：
- 工程建设项目招标投标
- 政府采购
- 土地使用权出让
- 产权交易
- 矿业权交易

#### 平台层级

| 层级 | 说明 | 数量 |
|------|------|------|
| 国家级 | 国务院相关部门 | 少数 |
| 省级 | 各省公共资源交易中心 | 31个 |
| 地市级 | 各地市公共资源交易中心 | 300+ |
| 县级 | 各县市公共资源交易中心 | 2000+ |

### 1.2 行业交易平台

| 行业 | 平台类型 | 特点 |
|------|---------|------|
| 工程建设 | 建设工程交易中心 | 房屋市政、交通、水利等 |
| 政府采购 | 政府采购网/交易中心 | 财政资金项目 |
| 医疗采购 | 药品/医疗器械采购平台 | 医用物资招标 |
| 军队采购 | 军队采购网 | 军用物资招标 |

## 三、对接要求

### 3.1 技术对接要求

#### 通信协议

| 项目 | 要求 |
|------|------|
| 传输协议 | HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 签名算法 | 国密SM3/SM2（推荐）或SHA256withRSA |
| 加密算法 | 国密SM4（推荐）或AES256 |

#### 常见接口规范

| 接口类型 | 功能说明 | 必选 |
|---------|---------|-----|
| 项目信息同步 | 获取招标项目信息 | 必选 |
| 投标企业报名 | 获取投标企业信息 | 必选 |
| 投标企业授权 | 获取企业授权信息 | 必选 |
| 保函递交 | 推送电子保函 | 必选 |
| 保函验真 | 验证保函真伪 | 必选 |
| 开标信息 | 获取开标结果 | 可选 |
| 中标信息 | 获取中标结果 | 可选 |
| 退保通知 | 获取退保信息 | 可选 |

## 四、对接流程

### 4.1 标准对接流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  1.需求对接   │───▶│  2.方案评审   │───▶│  3.技术开发   │───▶│  4.联调测试   │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                                                  │
                                                  ▼
                                        ┌──────────────┐
                                        │  5.试运行     │───▶ 6.正式上线
                                        └──────────────┘
```

1. **需求对接**：了解交易平台业务需求、接口现状
2. **方案评审**：确认对接方案、测试计划
3. **技术开发**：按规范开发接口、配置通信
4. **联调测试**：完成接口联调、功能测试
5. **试运行**：小范围试点、问题修复
6. **正式上线**：全量上线、持续监控

### 4.2 对接周期

| 平台类型 | 预估周期 | 影响因素 |
|---------|---------|---------|
| 省级平台 | 2-4周 | 内部流程、技术排期 |
| 省会城市 | 1-3周 | 对接复杂度 |
| 地市级平台 | 1-2周 | 系统复杂度 |
| 行业平台 | 2-4周 | 行业规范差异 |

## 六、对接模式

### 6.1 直连模式

交易平台直接调用保函系统接口，实现深度集成。

**优点**：
- 数据实时交互
- 用户体验好
- 功能完整

**缺点**：
- 对接成本高
- 维护工作量较大

**适用场景**：业务量大、长期合作的平台

### 6.2 SDK嵌入模式

交易平台嵌入保函系统SDK，实现轻量级对接。

**优点**：
- 对接快速
- 维护简单

**缺点**：
- 功能有限
- 定制化程度低

**适用场景**：业务量较小、短期合作的平台

## 七、注意事项

1. **接口规范差异**：不同平台接口规范有差异，需适配
2. **系统版本**：部分平台有新旧系统版本差异
3. **CA证书**：部分平台要求使用特定CA证书
4. **电子签章**：保函签章需符合平台要求
5. **验真方式**：部分平台要求在平台内验真
6. **应急预案**：建立平台异常的处理流程
7. **数据安全**：确保数据传输和存储安全
