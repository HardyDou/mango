## 一、模块概述

基础用户管理模块是电子保函系统的核心支撑模块，负责用户账户管理、角色权限配置、机构用户管理和操作日志审计。本模块为其他业务模块提供统一的用户认证和权限控制能力。

#### 2.1.1 用户注册

**注册方式**：
- 手机号注册
- 企业账号注册

**用户基本信息**：
| 字段 | 类型 | 说明 |
|------|------|------|
| username | String | 用户名（登录账号） |
| password | String | 登录密码（SM3加密存储） |
| phone | String | 手机号码 |
| email | String | 邮箱（可选） |
| userType | String | 用户类型（PERSONAL/ENTERPRISE） |
| realName | String | 真实姓名 |
| idCard | String | 身份证号码（个人用户） |
| companyName | String | 企业名称（企业用户） |
| businessLicense | String | 营业执照号（企业用户） |

**注册规则**：
- 用户名唯一，长度4-20个字符，支持字母、数字、下划线
- 密码长度8-20位，需包含字母和数字
- 手机号需通过短信验证码验证

#### 2.1.2 用户登录

**登录方式**：
- 用户名+密码登录
- 手机号+短信验证码登录
- 微信/支付宝第三方登录（后续版本）

**认证流程**：
```
1. 用户输入账号密码
2. 系统验证账号密码（密码SM3哈希比对）
3. 生成JWT Token（access_token + refresh_token）
4. 返回Token和用户信息
```

**Token机制**：
| Token类型 | 有效期 | 用途 |
|-----------|--------|------|
| access_token | 2小时 | API访问授权 |
| refresh_token | 7天 | 刷新access_token |

#### 2.2.2 权限模型

**权限类型**：
- **菜单权限**：控制用户可见的菜单
- **功能权限**：控制用户可操作的按钮/功能
- **数据权限**：控制用户可见的数据范围

**权限配置结构**：
```json
{
  "roleId": "xxx",
  "permissions": {
    "menus": ["menu1", "menu2", "menu3"],
    "functions": ["btn_add", "btn_edit", "btn_delete"],
    "dataScope": "SELF"  // SELF/DEPT/ORG/ALL
  }
}
```

## 三、状态流转

### 3.1 用户状态

```
注册待审核(REGISTER_PENDING) → 正常(ACTIVE)
        ↓
    审核拒绝(REJECTED)

正常(ACTIVE) → 禁用(DISABLED)
禁用(DISABLED) → 正常(ACTIVE)
```

### 3.2 状态说明

| 状态 | 说明 | 可转换状态 |
|------|------|-----------|
| REGISTER_PENDING | 注册待审核 | ACTIVE, REJECTED |
| ACTIVE | 正常 | DISABLED |
| DISABLED | 禁用 | ACTIVE |
| REJECTED | 审核拒绝 | - |

## 六、安全要求

### 6.1 密码安全
- 密码使用SM3哈希存储，禁止明文
- 密码复杂度要求：8-20位，需包含字母和数字
- 登录失败5次后锁定账户15分钟

### 6.2 认证安全
- JWT Token签名使用HMAC-SHA256
- access_token有效期2小时
- refresh_token有效期7天
- 支持Token主动失效

### 6.3 权限安全
- 所有API需登录后访问
- 敏感操作需二次验证
- 操作日志记录完整
