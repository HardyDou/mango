## 1. 审核流程层级统一规范

### 1.1 审核流程层级定义

| 审核类型 | 初审 | 复审 | 终审（可选） |
|----------|------|------|-------------|
| 投保审核 | 客服人员 | 风控人员 | 部门负责人（>50万） |
| 退保审核 | 客服人员 | 风控人员 | 部门负责人（>50万） |
| 保函变更 | 客服人员 | 风控人员 | - |
| 延期审核 | 客服人员 | 风控人员 | 部门负责人（>30万） |
| 红冲审核 | 财务人员 | 财务主管 | - |

### 1.2 审核权限配置

```
审核层级：
- L1: 客服人员（初审）
- L2: 风控人员（复审）
- L3: 部门负责人（终审）
- L4: 总经理（大额审批）

金额阈值：
- 10万以下：L1 → L2
- 10-30万：L1 → L2 → L3
- 30万以上：L1 → L2 → L3 → L4
```

```
日期格式：YYYY-MM-DD
时间格式：HH:mm:ss
```

## 5. 退保业务规则

### 5.1 退保手续费规则

| 退保场景 | 手续费标准 | 备注 |
|----------|-----------|------|
| 合同履行完毕 | 免费 | 免收手续费 |
| 部分履行 | 按剩余天数5%收取 | 最低100元 |
| 司法判决 | 免费 | 免收手续费 |
| 提前解除 | 按剩余天数10%收取 | 最低200元 |

**计算公式**：
- 退费金额 = 已缴保费 - 手续费
- 手续费 = 已缴保费 × 剩余天数占比 × 手续费率
- 剩余天数占比 = (保函到期日 - 退保申请日) / (保函到期日 - 保函生效日)

### 5.2 退款周期要求

| 退款方式 | 处理周期 | 到账周期 |
|----------|---------|---------|
| 原路退回 | T+1个工作日 | 3-5个工作日 |
| 人工转账 | T+3个工作日 | 5-7个工作日 |

**说明**：
- T为退保审核通过日
- 原路退回优先，如原支付渠道已关闭则转为人工转账
- 节假日至顺延

### 5.3 审批时限

| 审批类型 | 时限要求 | 超时处理 |
|----------|---------|----------|
| 初审 | 2个工作日 | 自动催办 |
| 复审 | 3个工作日 | 自动催办 |
| 终审 | 5个工作日 | 自动催办 |

### 5.4 通知规则

| 通知类型 | 通知方式 | 通知时机 |
|----------|---------|----------|
| 审核进度 | APP推送+短信 | 状态变更 |
| 审核结果 | APP推送+短信+邮件 | 审核完成 |
| 退款到账 | APP推送+短信 | 退款完成 |

## 6. 保函管理业务规则

### 6.1 保函变更通知规则

| 变更类型 | 是否通知受益人 | 通知方式 | 通知时限 |
|----------|---------------|---------|----------|
| 保函金额减少 | 是 | 邮件+系统 | 变更生效后即时 |
| 保函期限缩短 | 是 | 邮件+系统 | 变更生效后即时 |
| 保函金额增加 | 否 | - | - |
| 保函期限延长 | 否 | - | - |
| 受益人变更 | 是 | 邮件+系统 | 变更生效后即时 |

### 6.2 延期申请次数限制

| 保函类型 | 延期次数上限 | 单次延期时长 | 累计延期时长上限 |
|----------|-------------|-------------|-----------------|
| 投标保函 | 2次 | 6个月 | 12个月 |
| 履约保函 | 3次 | 6个月 | 18个月 |
| 预付款保函 | 2次 | 3个月 | 6个月 |
| 质量保函 | 3次 | 6个月 | 18个月 |

**延期费用计算**：
- 延期保费 = 原保函金额 × 延期天数 × 日费率
- 日费率 = 年费率 / 365
- 年费率标准：2%-5%（根据保函类型和风险等级）

### 6.3 保函变更可编辑字段

| 字段 | 可变更 | 变更限制 |
|------|--------|---------|
| 保函金额 | 是 | 不可低于已发生理赔金额 |
| 保函期限 | 是 | 不可短于已投保期限 |
| 受益人 | 是 | 需提供变更说明 |
| 项目名称 | 是 | 需提供变更证明 |
| 联系地址 | 是 | 无限制 |
| 联系电话 | 是 | 无限制 |

### 6.4 不可变更字段

以下字段在保函出具后不可变更：
- 担保品种
- 投保人/申请人
- 担保公司
- 金融机构

## 7. 发票管理业务规则

### 7.1 税务系统对接规划

| 对接阶段 | 功能范围 | 预计时间 |
|----------|---------|----------|
| 第一阶段 | 手动开票，本地存储 | 第7周MVP |
| 第二阶段 | API对接税控盘 | 第9-10周 |
| 第三阶段 | 全电发票/区块链发票 | 第11-12周 |

**第一阶段实现方式**：
- 暂时使用本地发票管理系统
- 手动录入税控盘开票
- 发票信息本地存储，定期汇总申报

**第二阶段规划**：
- 对接税务UKey开票API
- 实现自动开票功能
- 发票数据实时同步

### 7.2 发票红冲时间限制

| 发票类型 | 红冲时限 | 红冲规则 |
|----------|---------|----------|
| 增值税专用发票 | 开票后360天内 | 可部分红冲或全额红冲 |
| 增值税普通发票 | 开票后360天内 | 只能全额红冲 |
| 电子发票 | 开票后360天内 | 只能全额红冲 |

**红冲要求**：
- 红冲必须遵循"先红冲后重开"原则
- 红冲后重新开票需在7个工作日内完成
- 已跨月发票红冲需税务申报配合

### 7.3 发票类型与税率

| 发票类型 | 税率 | 适用范围 |
|----------|-----|----------|
| 增值税专用发票 | 6% | 一般纳税人 |
| 增值税普通发票 | 6% | 小规模纳税人、个人 |
| 电子普通发票 | 6% | 所有客户 |

### 7.4 发票交付方式

| 交付方式 | 交付时效 | 费用 |
|----------|---------|------|
| 系统下载 | 即时 | 免费 |
| 邮件发送 | 即时 | 免费 |
| 纸质邮寄 | 3-5个工作日 | 快递费到付 |
