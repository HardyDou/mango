# 第6周需求 - 保函管理模块

**版本**: V1.0
**制定日期**: 2026-03-14
**制定人**: 电子保函业务专家
**状态**: 已优化

## 1. 业务概述

保函管理功能提供保函的查询、变更、延期、终止、验真等全生命周期管理功能。

### 1.1 功能范围

| 功能 | 说明 | 用户角色 |
|------|------|---------|
| 保函查询 | 多维度查询保函信息 | 投标人、担保公司、金融机构 |
| 保函验真 | 验证保函真伪 | 招标方、业主 |
| 保函变更 | 修改保函部分信息 | 投标人、金融机构 |
| 保函延期 | 延长保函有效期 | 投标人、金融机构 |
| 保函终止 | 提前终止保函效力 | 金融机构 |

## 2. 功能需求

### 2.1 保函查询

**功能描述**: 多条件查询保函信息

**用户角色**: 投标人、担保公司、金融机构

**查询条件**:

| 条件类型 | 说明 |
|----------|------|
| 保函编号 | 精确查询 |
| 项目名称 | 模糊查询 |
| 保函类型 | 投标/履约/预付款/质量 |

| 出函机构 | 下拉选择 |
| 保函状态 | 全部/已出具/生效中/已过期/已退保/已终止 |
| 出函日期 | 日期范围 |
| 到期日期 | 日期范围 |

**展示字段**:

| 字段 | 说明 |
|------|------|
| 保函编号 | 系统生成唯一编号 |
| 项目名称 | 对应招标项目名称 |
| 保函类型 | 投标/履约/预付款/质量 |
| 投保人 | 投标人名称 |
| 受益人 | 招标方/业主 |
| 担保金额 | 保函担保金额 |

| 出函日期 | 保函出具日期 |
| 到期日期 | 保函到期日期 |
| 担保公司 | 担保公司名称 |
| 出函机构 | 出函机构名称 |

### 2.3 保函验真

**功能描述**: 验证保函真伪

**用户角色**: 招标方、业主、任意第三方

**验证方式**:

| 方式 | 说明 |
|------|------|
| 保函编号验真 | 通过保函编号查询保函状态 |
| 二维码验真 | 扫描保函上的二维码 |
| 文件哈希验真 | 验证OFD文件哈希值 |

**验真结果**:

| 字段 | 说明 |
|------|------|
| 验真结果 | 真/假 |
| 保函基本信息 | 编号、类型、金额、状态 |
| 保障期限 | 生效日期至到期日期 |
| 签章信息 | 签章机构、时间 |
| 文件哈希 | OFD文件哈希值 |

### 2.4 保函变更

**功能描述**: 修改保函部分信息

**用户角色**: 投标人、金融机构

**可变更字段**:

| 字段 | 说明 | 权限 |
|------|------|------|
| 项目名称 | 招标项目名称变更 | 投标人申请、金融机构审批 |
| 受益人 | 受益人变更 | 投标人申请、金融机构审批 |
| 担保金额 | 担保金额变更（增加） | 投标人申请、金融机构审批 |
| 联系方式 | 联系方式变更 | 投标人申请 |

**不可变更**:
- 保函类型
- 投保人
- 保障期限（延期除外）

**变更流程**:
```
变更申请 → 金融机构审核 → 执行变更 → 变更生效
```

### 2.5 保函延期

**功能描述**: 延长保函有效期

**用户角色**: 投标人、金融机构

**前置条件**:
- 保函状态为"生效中"
- 距原到期日不少于X个工作日
- 无未完成的延期申请

**延期规则**:
- 延期时长不少于30天
- 延期次数不限（视金融机构审批）
- 延期需缴纳延期保费

**延期费用计算**:
```
延期保费 = 原担保金额 × 延期天数 × 日费率
```

**延期流程**:
```
延期申请 → 金融机构审核 → 缴纳延期保费 → 执行延期 → 延期生效
```

### 2.6 保函终止

**功能描述**: 提前终止保函效力

**用户角色**: 金融机构

**适用场景**:
- 法院判决终止
- 双方协商终止
- 担保公司承担赔付责任后终止

**终止类型**:

| 类型 | 说明 |
|------|------|
| 正常终止 | 保函到期自动终止 |
| 提前终止 | 合同履行完毕提前终止 |
| 赔付终止 | 担保公司已赔付 |
| 司法终止 | 法院判决终止 |

**终止流程**:
```
终止申请 → 审核（如需）→ 执行终止 → 通知受益人 → 终止生效
```

## 3. 接口需求

### 3.1 保函列表查询接口

**接口路径**: `GET /api/v1/guarantee/list`

**请求参数**:
```json
{
  "guaranteeNo": "string",        // 保函编号
  "projectName": "string",       // 项目名称（模糊）
  "guaranteeType": "string",     // 保函类型
  "status": "string",            // 保函状态
  "insurerId": "string",         // 担保公司ID
  "issuerId": "string",           // 出函机构ID
  "startDate": "string",         // 出函日期开始
  "endDate": "string",           // 出函日期结束
  "page": "number",              // 页码
  "pageSize": "number"           // 每页条数
}
```

**响应参数**:
```json
{
  "code": "200",
  "message": "success",
  "data": {
    "total": "number",
    "list": [
      {
        "guaranteeNo": "string",
        "projectName": "string",
        "guaranteeType": "string",
        "applicant": "string",
        "beneficiary": "string",
        "amount": "number",
        "premium": "number",
        "status": "string",
        "issueDate": "string",
        "expireDate": "string",
        "insurerName": "string",
        "issuerName": "string"
      }
    ]
  }
}
```

## 7. 业务规则补充

### 7.1 保函变更通知规则

| 变更类型 | 是否通知受益人 | 通知方式 |
|----------|---------------|----------|
| 项目名称变更 | 是 | 系统通知+邮件 |
| 受益人变更 | 是 | 系统通知+邮件 |
| 担保金额变更 | 是 | 系统通知+邮件 |
| 联系方式变更 | 否 | - |

**通知内容**：变更后的保函信息、变更时间、变更原因

### 7.2 延期申请次数限制

| 保函类型 | 最多次数 | 累计最长延期 |
|----------|---------|-------------|
| 投标保函 | 2次 | 180天 |
| 履约保函 | 3次 | 365天 |
| 预付款保函 | 2次 | 180天 |
| 质量保函 | 3次 | 365天 |

### 7.3 延期保费计算规则

```
延期保费 = 原担保金额 × 延期天数 × 日费率

日费率标准：
- 延期30天以内：0.02%/天
- 延期31-90天：0.015%/天
- 延期91-180天：0.01%/天
- 延期180天以上：0.008%/天
```

### 7.4 保函终止审批规则

| 终止类型 | 是否需要审批 | 审批层级 |
|----------|-------------|----------|
| 正常终止 | 否 | 系统自动 |
| 提前终止（双方协商） | 是 | 金融机构风控 |
| 赔付终止 | 是 | 金融机构审批 |
| 司法终止 | 是 | 金融机构+法务 |
