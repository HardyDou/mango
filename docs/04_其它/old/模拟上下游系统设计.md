# 模拟上下游系统设计文档

## 一、概述

本文档描述电子保函系统的模拟上下游系统，用于开发和测试环境下的接口联调。模拟系统作为"桩子"，模拟真实交易平台和出函机构的行为，与电子保函系统对接。

---

## 二、系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      电子保函系统                                 │
│                     (localhost:8080)                            │
└─────────────────────────────────────────────────────────────────┘
            ▲                                    │
            │ 交易系统接口                        │ 出函机构接口
            │                                    ▼
┌───────────────────────┐          ┌───────────────────────────────┐
│    模拟交易系统        │          │       模拟出函机构            │
│  (localhost:8081)    │          │     (localhost:8082)          │
│                       │          │                               │
│ - 江西省公共资源交易   │          │ - 光大银行模拟                │
│ - 项目信息同步        │          │ - 中银保险模拟                │
│ - 投标企业报名        │          │ - 投保申请/审核/出函           │
│ - 保函递交/验真       │          │ - 退保通知                    │
└───────────────────────┘          └───────────────────────────────┘
```

---

## 三、模拟交易系统

### 3.1 系统信息

| 项目 | 值 |
|------|-----|
| 服务名称 | mock-trade-system |
| 端口 | 8081 |
| 类型 | 模拟上游交易平台 |

### 3.2 功能模块

#### 3.2.1 项目信息同步

接收电子保函系统推送的项目信息。

**接口地址**: `POST /api/mock/project/sync`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectNo | String | 是 | 项目编号 |
| projectName | String | 是 | 项目名称 |
| tenderName | String | 是 | 招标人名称 |
| guaranteeType | Integer | 是 | 保函类型 |
| deadline | Date | 是 | 投标截止时间 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "projectId": "P001",
    "status": "SYNCED"
  }
}
```

#### 3.2.2 投标企业报名

接收投标企业报名信息。

**接口地址**: `POST /api/mock/tender/register`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectNo | String | 是 | 项目编号 |
| bidderName | String | 是 | 投标人名称 |
| bidderCreditCode | String | 是 | 投标人统一社会信用代码 |
| contactPerson | String | 否 | 联系人 |
| contactMobile | String | 否 | 联系电话 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "registerId": "R001",
    "status": "REGISTERED"
  }
}
```

#### 3.2.3 保函递交

接收电子保函递交。

**接口地址**: `POST /api/mock/guarantee/submit`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectNo | String | 是 | 项目编号 |
| bidderName | String | 是 | 投标人名称 |
| guaranteeNo | String | 是 | 保函编号 |
| guaranteeAmount | BigDecimal | 是 | 保函金额 |
| guaranteeType | Integer | 是 | 保函类型 |
| startDate | Date | 是 | 生效日期 |
| endDate | Date | 是 | 到期日期 |
| documentUrl | String | 是 | 保函文档URL |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "submitId": "S001",
    "status": "ACCEPTED"
  }
}
```

#### 3.2.4 保函验真

验证保函真伪。

**接口地址**: `GET /api/mock/guarantee/verify`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| guaranteeNo | String | 是 | 保函编号 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "guaranteeNo": "GZ202601010001",
    "status": "VALID",
    "projectName": "XX项目",
    "amount": 100000.00,
    "startDate": "2026-01-01",
    "endDate": "2027-01-01"
  }
}
```

#### 3.2.5 退保通知

接收退保通知。

**接口地址**: `POST /api/mock/guarantee/refundnotify`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| guaranteeNo | String | 是 | 保函编号 |
| refundType | Integer | 是 | 退保类型：1-正常退保，2-提前退保 |
| refundDate | Date | 是 | 退保日期 |
| reason | String | 否 | 退保原因 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "notifyId": "N001",
    "status": "CONFIRMED"
  }
}
```

## 四、模拟出函机构

### 4.1 系统信息

| 项目 | 值 |
|------|-----|
| 服务名称 | mock-institution-system |
| 端口 | 8082 |
| 类型 | 模拟下游出函机构 |

### 4.2 功能模块

#### 4.2.1 投保申请

接收电子保函投保申请。

**接口地址**: `POST /api/mock/insurance/apply`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| applyNo | String | 是 | 投保申请号 |
| guaranteeType | Integer | 是 | 保函类型 |
| projectName | String | 是 | 项目名称 |
| beneficiaryName | String | 是 | 受益人名称 |
| applicantName | String | 是 | 申请人名称 |
| amount | BigDecimal | 是 | 投保金额 |
| startDate | Date | 是 | 生效日期 |
| endDate | Date | 是 | 到期日期 |
| callbackUrl | String | 是 | 回调地址 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "applyId": "A001",
    "status": "APPLY_RECEIVED",
    "auditDays": 1
  }
}
```

#### 4.2.2 审核通知（回调）

模拟出函机构审核完成后，通过回调通知电子保函系统。

**接口地址**: `POST /api/mock/insurance/audit/callback`

（此接口由电子保函系统提供，模拟出函机构调用）

**回调参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| applyNo | String | 是 | 投保申请号 |
| status | Integer | 是 | 审核状态：1-通过，2-拒绝 |
| opinion | String | 否 | 审核意见 |
| guaranteeNo | String | 是 | 保函编号（如通过） |

#### 4.2.3 保函出具

模拟出函机构出具保函后，推送保函文件。

**接口地址**: `POST /api/mock/insurance/issue/callback`

（此接口由电子保函系统提供，模拟出函机构调用）

**回调参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| guaranteeNo | String | 是 | 保函编号 |
| issueDate | Date | 是 | 出具日期 |
| documentUrl | String | 是 | 保函文件URL |
| signInfo | String | 否 | 签章信息 |

#### 4.2.4 退保处理

接收退保请求。

**接口地址**: `POST /api/mock/insurance/refund`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| guaranteeNo | String | 是 | 保函编号 |
| refundType | Integer | 是 | 退保类型 |
| applyNo | String | 是 | 原投保申请号 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "refundId": "R001",
    "status": "REFUND_PROCESSING",
    "refundAmount": 1000.00,
    "refundDays": 3
  }
}
```

#### 4.2.5 验真查询

提供保函验真查询接口。

**接口地址**: `GET /api/mock/insurance/verify`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| guaranteeNo | String | 是 | 保函编号 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "guaranteeNo": "GZ202601010001",
    "status": "VALID",
    "institutionName": "光大银行",
    "amount": 100000.00,
    "startDate": "2026-01-01",
    "endDate": "2027-01-01"
  }
}
```

## 六、使用说明

### 6.1 启动模拟系统

```bash
# 启动模拟交易系统（端口8081）
cd mock-trade-system
mvn spring-boot:run

# 启动模拟出函机构（端口8082）
cd mock-institution-system
mvn spring-boot:run
```

### 6.2 配置电子保函系统

在电子保函系统的配置文件中添加：

```yaml
mock:
  trade-system:
    url: http://localhost:8081
  institution-system:
    url: http://localhost:8082
```

### 6.3 测试流程

1. 启动电子保函系统（8080）
2. 启动模拟交易系统（8081）
3. 启动模拟出函机构（8082）
4. 通过电子保函系统发起投保申请
5. 模拟出函机构自动/手动审核
6. 模拟出函机构出具保函
7. 验证保函递交到模拟交易系统
8. 测试退保流程

---

## 七、注意事项

1. 模拟系统仅用于开发和测试环境
2. 所有接口响应均为模拟数据
3. 回调接口需要电子保函系统提供对应的接收端点
4. 可以根据测试需要调整模拟逻辑
