### 3. 保费计算规则 - 计算逻辑是否合理 ⚠️ 需补充

**现有规则**：
```
保费 = 保函金额 × 年费率 × 有效期(年)
```

**建议补充说明**：
1. **有效期计算方式**：是按精确天数还是按年计算？建议明确
2. **金额精度处理**：保费计算涉及小数，建议明确舍入规则（向上取整/四舍五入）
3. **最低保费**：建议明确"不低于200元"的具体规则
4. **阶梯费率**：文档提到100万-500万有折扣、大于500万可协商，建议补充具体的费率区间

### 4. 状态流转 - 状态定义和流转是否正确 ⚠️ 需补充

**现有流转**：
```
DRAFT → SUBMITTED → REVIEWING → APPROVED → PREMIUM_PAID → 保函出具
                                    ↓
                              REJECTED → 重新提交
                                    ↓
                              CANCELLED
```

**建议补充**：
1. **PREMIUM_REFUSED（缴费失败）**状态：状态列表中定义了此状态，但流转图中未体现，需补充
2. **已缴费后的撤销流程**：已缴费用户撤销需退款审核，建议明确退款审核流程
3. **REJECTED后重新提交**：是否回到DRAFT状态还是直接修改后再次提交？建议明确
