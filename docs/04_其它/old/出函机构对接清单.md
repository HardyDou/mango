# 电子保函出函机构对接指南

本文档梳理需要对接的出函机构，包括银行、保险公司、担保公司，并提供对接要求和注意事项。

---

## 一、出函机构类型

### 1.1 银行类

银行是电子保函的主要出函机构，具有以下特点：
- 信用度高，被广泛接受
- 具备完善的电子签章系统
- 与各省市公共资源交易中心有成熟对接经验

#### 主要对接银行（按业务规模排序）

| 银行名称 | 特点 | 对接难度 | 备注 |
|---------|------|---------|------|
| **中国光大银行** | 最早开展电子保函业务，对接案例最多 | 低 | 报文规范完善 |
| **招商银行** | 电子保函系统成熟 | 低 | 支持多种保函类型 |
| **浦发银行** | 交易银行业务覆盖广 | 中 | 需要总行审批 |
| **建设银行** | 网点覆盖广，品牌认可度高 | 中 | 省级分行对接 |
| **工商银行** | 信贷系统强大 | 中 | 需总行科技部门配合 |
| **农业银行** | 普惠金融业务活跃 | 中 | 需确认开通电子保函业务 |
| **中国银行** | 国际业务经验丰富 | 中 | 跨境保函有优势 |
| **交通银行** | 交易银行板块发展快 | 中 | 报文规范较好 |
| **民生银行** | 民企服务经验丰富 | 中高 | 需确认业务开通 |
| **平安银行** | 科技能力较强 | 中 | 需省级分行对接 |
| **中信银行** | 对公业务较强 | 中 | 报文规范需确认 |
| **兴业银行** | 绿色金融业务活跃 | 中 | 需确认业务权限 |

#### 城商行（地方性银行）

| 银行名称 | 覆盖区域 | 适用场景 |
|---------|---------|---------|
| 江西银行 | 江西省 | 当地公共资源交易项目 |
| 福建海峡银行 | 福建省 | 当地公共资源交易项目 |
| 四川天府银行 | 四川省 | 当地公共资源交易项目 |
| 齐鲁银行 | 山东省 | 当地公共资源交易项目 |

> **建议**：优先对接全国性股份制银行，再根据业务需要对接地方性城商行。

### 1.2 保险公司

保险公司是电子保函市场的重要参与者，具有以下特点：
- 费率相对银行更有竞争力
- 审核流程相对灵活
- 见保模式（先出函后付费）更受投标人欢迎

#### 主要对接保险公司

| 保险公司 | 特点 | 对接难度 | 备注 |
|---------|------|---------|------|
| **中银保险** | 国有背景，信用度高 | 低 | 中国银行旗下子公司 |
| **大家财产保险** | 业务发展快 | 低 | 大家保险集团子公司 |
| **中国人保** | 品牌影响力大 | 中 | 需确认开通保函业务 |
| **中国平安** | 科技能力强 | 中 | 平安银行协同 |
| **中国太平洋保险** | 网点覆盖广 | 中 | 需省级分公司对接 |
| **中华联合保险** | 专业担保险 | 中 | 信保业务成熟 |
| **阳光保险** | 业务创新活跃 | 中 | 需确认业务权限 |
| **大地保险** | 财产险业务强 | 中 | 需确认对接意愿 |
| **华泰保险** | 工程险经验丰富 | 中高 | 工程保函有优势 |
| **太平保险** | 央企背景 | 中 | 需确认业务开通 |

#### 保险资质的特殊要求

- **经营许可**：需确认保险公司具有保证保险业务资质
- **产品备案**：电子保函产品需在银保监会备案
- **额度管理**：保险公司有承保额度限制
- **见保模式**：部分保险公司支持见保模式（先出函后付费）

### 1.3 担保公司

担保公司是电子保函的重要补充，具有以下特点：
- 业务灵活，可针对特定行业
- 审核效率高
- 可提供见保模式

#### 主要对接担保公司类型

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| **省级信用担保公司** | 政府背景，信用度高 | 政府投资项目 |
| **市级担保公司** | 地方支持 | 当地项目 |
| **专业工程担保公司** | 工程领域专业 | 工程建设项目 |
| **民营担保公司** | 审核灵活 | 民营企业项目 |

#### 知名担保公司示例

- 江西省信用担保股份有限公司
- 福建省融资担保有限责任公司
- 山东省再担保集团股份有限公司
- 安徽省信用担保集团有限公司
- 湖北省融资担保集团有限责任公司
- 湖南省融资担保集团有限公司

## 二、对接要求

### 2.1 技术对接要求

#### 通信协议

| 项目 | 要求 |
|------|------|
| 传输协议 | HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 签名算法 | 国密SM3/SM2 |
| 加密算法 | 国密SM4 |

#### 接口规范

出函机构需要提供以下接口能力：

| 接口类型 | 功能说明 | 必选 |
|---------|---------|-----|
| 投保申请 | 接收投保请求 | 必选 |
| 审核通知 | 发送审核结果 | 必选 |
| 保函出具 | 推送保函文件 | 必选 |
| 退保通知 | 接收退保请求 | 必选 |
| 验真查询 | 保函真伪验证 | 必选 |
| 对账单 | 业务对账 | 可选 |

#### 电子签章要求

- 支持CA数字证书签章
- 签章需符合《电子签名法》要求
- 保函文件格式：OFD或PDF

### 2.2 业务对接要求

#### 资质要求

| 机构类型 | 必备资质 |
|---------|---------|
| 银行 | 金融许可证、营业执照 |
| 保险公司 | 保险业务许可证、保证保险业务备案 |
| 担保公司 | 融资担保业务经营许可证 |

#### 额度要求

- 单笔保函额度：根据机构资质确定
- 日累计额度：根据机构规模确定
- 年度总承保额度：根据机构实力确定

#### 费率要求

- 保函费率：年化费率0.5%-3%（根据保函类型和客户资质）
- 见费模式：保费到账后出函
- 见保模式：出函后N日内付费

## 三、对接流程

### 3.1 对接步骤

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1.需求调研  │───▶│  2.方案确认  │───▶│  3.技术对接  │───▶│  4.联调测试  │───▶│  5.正式上线  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

1. **需求调研**：了解出函机构现有系统、业务流程
2. **方案确认**：确认接口方案、报文规范、测试计划
3. **技术对接**：按规范开发接口、配置证书
4. **联调测试**：完成接口联调、业务测试
5. **正式上线**：切换生产环境、持续监控

### 3.2 对接周期

| 机构类型 | 预估周期 | 影响因素 |
|---------|---------|---------|
| 全国性银行 | 2-4周 | 内部审批、科技排期 |
| 股份制银行 | 2-3周 | 分行/总行协调 |
| 保险公司 | 1-3周 | 业务部门配合 |
| 担保公司 | 1-2周 | 系统复杂度低 |

## 四、优先级建议

### 4.1 第一批次（优先对接）

| 机构类型 | 机构名称 | 理由 |
|---------|---------|------|
| 银行 | 中国光大银行 | 对接案例多，规范成熟 |
| 银行 | 招商银行 | 系统成熟，支持多种类型 |
| 保险公司 | 中银保险 | 国有背景，信用度高 |
| 保险公司 | 大家财产保险 | 业务活跃，对接快 |

### 4.2 第二批次（业务扩展）

| 机构类型 | 机构名称 | 理由 |
|---------|---------|------|
| 银行 | 浦发银行 | 交易银行覆盖广 |
| 银行 | 建设银行 | 网点多，品牌认可 |
| 保险公司 | 中国人保 | 品牌影响力大 |
| 保险公司 | 中国平安 | 科技能力强 |

### 4.3 第三批次（区域补充）

根据业务发展区域，对接当地城商行和省级担保公司。

## 五、注意事项

1. **机构准入**：确认出函机构具有相应业务资质
2. **额度管理**：实时监控各机构承保额度
3. **费率竞争**：不同机构费率有差异，需做好比价
4. **见费/见保**：明确出函机构支持的模式
5. **系统稳定性**：关注出函机构系统可用性
6. **应急预案**：建立出函异常的处理流程
