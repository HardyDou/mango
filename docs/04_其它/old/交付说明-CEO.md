## 四、快速启动（开发环境）

### 4.1 后端启动

```bash
# 1. 确保 MySQL 和 Redis 已启动
# 2. 启动 Nacos（服务注册中心）
# 3. 编译项目
cd /Users/hardy/Work/baohan/baohan
mvn clean install -DskipTests

# 4. 启动各个微服务（按顺序）
java -jar baohan-gateway/target/baohan-gateway.jar &
java -jar baohan-user/target/baohan-user.jar &
java -jar baohan-guarantee/target/baohan-guarantee.jar &
# ... 其他服务
```

### 4.2 前端启动

```bash
cd /Users/hardy/Work/baohan/baohan/frontend
npm install
npm run dev
```

## 五、需要 CEO 决策的事项

### 5.1 基础设施资源

当前系统无法直接运行，需要部署以下基础设施：

| 资源 | 建议配置 | 用途 |
|------|----------|------|
| 云服务器 | 4核8G × 3台 | 部署微服务 |
| MySQL | 8核16G | 数据存储 |
| Redis | 4核8G | 缓存/Session |
| Nacos | 2核4G | 服务治理 |

### 5.2 部署方案

**选项 A：传统部署**
- 优点：可控性强
- 缺点：运维工作量大

**选项 B：容器化部署（推荐）**
- 使用 Docker + Kubernetes
- 需要额外时间搭建 CI/CD
