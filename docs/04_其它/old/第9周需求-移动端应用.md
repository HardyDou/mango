## 一、功能概述

本章描述电子保函系统移动端应用（iOS/Android）的功能需求，为投标人提供移动端的保函业务办理渠道。

## 三、功能详细需求

### 3.1 登录注册

#### 3.1.1 手机号注册登录

| 功能 | 需求描述 |
|------|---------|
| 注册入口 | 支持手机号+验证码注册 |
| 验证码发送 | 点击发送验证码，60秒后可重新发送 |
| 密码设置 | 设置登录密码（8-20位，含字母+数字） |
| 登录 | 支持手机号+密码登录 |
| 记住登录 | 支持记住用户名 |

#### 3.1.2 快捷登录

| 功能 | 需求描述 |
|------|---------|
| 验证码登录 | 手机号+验证码直接登录 |
| 首次登录自动注册 | 未注册手机号自动创建账户 |

#### 3.1.3 忘记密码

| 功能 | 需求描述 |
|------|---------|
| 验证码验证 | 通过验证码验证身份 |
| 密码重置 | 设置新密码 |

---

### 3.2 投保申请

#### 3.2.1 投保申请入口

| 功能 | 需求描述 |
|------|---------|
| 选择保函类型 | 支持投标保函、履约保函、预付款保函、质量保函 |
| 选择交易平台 | 选择拟投标的交易平台 |
| 选择出函机构 | 选择出函机构（银行/保险/担保公司） |

#### 3.2.2 项目信息填写

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 项目名称 | 文本 | 是 | 招标项目名称 |
| 项目编号 | 文本 | 是 | 招标项目编号 |
| 受益人 | 文本 | 是 | 招标单位名称 |
| 保函金额 | 数字 | 是 | 担保金额 |
| 保函有效期 | 日期 | 是 | 截止日期 |
| 招标代理 | 文本 | 否 | 招标代理机构 |

#### 3.2.3 企业信息填写

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 企业名称 | 文本 | 是 | 投标人企业名称 |
| 统一社会信用代码 | 文本 | 是 | 18位信用代码 |
| 联系人 | 文本 | 是 | 联系人姓名 |
| 联系电话 | 文本 | 是 | 联系人手机号 |
| 邮箱 | 文本 | 否 | 联系人邮箱 |

#### 3.2.4 材料上传

| 材料名称 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| 营业执照 | 图片 | 是 | 支持拍照或相册选择 |
| 法人身份证 | 图片 | 是 | 正反面 |
| 招标文件 | PDF/图片 | 是 | 招标公告或招标文件 |
| 业绩证明 | 图片 | 条件必填 | 根据项目要求 |
| 授权委托书 | PDF/图片 | 条件必填 | 非法人办理时必填 |

**材料上传要求**：
- 支持拍照和相册选择
- 图片格式：JPG/PNG
- 单个文件大小≤10MB
- 材料自动OCR识别（如适用）

#### 3.2.5 费用预览与提交

| 功能 | 需求描述 |
|------|---------|
| 保费计算 | 根据保函类型、金额、出函机构自动计算保费 |
| 费用明细 | 显示保费计算明细 |
| 出函模式 | 显示见费模式/见保模式 |
| 投保确认 | 确认投保信息并提交 |

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | ≤3秒 |
| 接口响应时间 | ≤2秒 |
| 图片加载 | 支持懒加载 |
| 离线缓存 | 支持基础数据离线缓存 |

### 4.2 安全性要求

| 要求 | 说明 |
|------|------|
| 通信安全 | HTTPS传输 |
| 本地存储安全 | 敏感数据加密存储 |
| Token安全 | 登录Token安全存储，定期刷新 |
| 生物识别 | 支持指纹/面容ID登录（可选） |
| 国产密码算法 | 支持国密SM3/SM4（可选） |

### 4.3 兼容性要求

| 要求 | 说明 |
|------|------|
| iOS | iOS 13.0+ |
| Android | Android 8.0+ |
| 分辨率 | 适配各种屏幕尺寸 |

### 4.4 用户体验要求

| 要求 | 说明 |
|------|------|
| 界面风格 | 符合移动端设计规范 |
| 交互反馈 | 所有操作有明确反馈 |
| 错误处理 | 友好的错误提示 |
| 加载状态 | 显示加载动画 |

## 五、接口需求

### 5.1 移动端API

| 接口 | 功能 | 说明 |
|------|------|------|
| POST /api/v1/mobile/auth/sendCode | 发送验证码 | 登录注册 |
| POST /api/v1/mobile/auth/login | 手机号登录 | 快捷登录 |
| POST /api/v1/mobile/auth/register | 手机号注册 | 新用户注册 |
| POST /api/v1/mobile/auth/resetPwd | 忘记密码 | 重置密码 |
| GET /api/v1/mobile/apply/list | 投保申请列表 | 进度查询 |
| GET /api/v1/mobile/apply/{id} | 投保申请详情 | 进度详情 |
| POST /api/v1/mobile/apply/create | 创建投保申请 | 投保申请 |
| POST /api/v1/mobile/apply/{id}/upload | 上传申请材料 | 材料上传 |
| POST /api/v1/mobile/apply/{id}/submit | 提交投保申请 | 提交 |
| GET /api/v1/mobile/guarantee/list | 保函列表 | 保函管理 |
| GET /api/v1/mobile/guarantee/{id} | 保函详情 | 保函详情 |
| GET /api/v1/mobile/guarantee/{id}/download | 下载保函文件 | 保函下载 |
| GET /api/v1/mobile/guarantee/verify | 保函验真 | 验真 |
| GET /api/v1/mobile/notice/list | 消息列表 | 通知 |
| GET /api/v1/mobile/notice/{id} | 通知详情 | 详情 |
| PUT /api/v1/mobile/notice/read | 标记已读 | 标记 |
| GET /api/v1/mobile/user/profile | 用户信息 | 个人中心 |
| PUT /api/v1/mobile/user/profile | 更新用户信息 | 修改信息 |
| POST /api/v1/mobile/user/realname | 实名认证 | 实名 |
| GET /api/v1/mobile/invoice/list | 发票列表 | 发票记录 |
| POST /api/v1/mobile/invoice/apply | 申请发票 | 发票申请 |
