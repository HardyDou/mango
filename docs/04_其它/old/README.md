本文档是电子保函系统的共享知识库，旨在帮助团队成员快速了解电子保函业务和技术要点，无需阅读所有原始文档。

## 一、业务概述

### 1.1 什么是电子保函

电子保函是**电子化的保证凭证**，由保险公司、银行或担保公司应申请人请求，向受益人开具的电子担保文件。在公共资源交易领域主要用于投标保证、合同履约、预付款支付等场景。

### 1.2 服务对象

| 角色 | 说明 |
|------|------|
| 投标人 | 保函申请方 |
| 招标代理 | 业务发起方 |
| 交易中心 | 平台运营方 |
| 金融机构 | 银行/保险/担保公司 |

## 二、电子保函类型

### 2.1 主要类型

| 类型 | 用途 | 有效期 |
|------|------|--------|
| **投标保函 (Bid Bond)** | 投标人投标时提供，保证中标后签约 | 随投标有效期 |
| **履约保函 (Performance Bond)** | 中标后提供，保证合同正常履行 | 合同工期+质保期 |
| **预付款保函 (Advance Payment Bond)** | 业主支付预付款时提供，保证专款专用 | 预付款退还完毕 |
| **质量保函 (Quality Bond)** | 质保期内的质量保证 | 质保期满 |

### 2.2 保函格式

- **独立格式**: 保函独立于基础合同，银行/担保公司独自承担保函责任
- **连带格式**: 保函与基础合同连带，依赖基础合同条款

## 三、核心业务流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 投保申请  │───▶│ 审核确认  │───▶│ 出具保函  │───▶│ 开标/公示 │───▶│ 退保/理赔 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 3.1 主要流程节点

1. **投保申请**: 投标人通过交易平台提交保函申请
2. **审核确认**: 金融机构审核申请材料
3. **出具保函**: 审核通过后生成电子保函文件
4. **开标/公示**: 保函在开标时验证有效性
5. **退保/理赔**: 合同履行完毕后退保，或触发理赔

### 3.2 发票管理

- 投保成功后可申请开具发票
- 支持纸质发票和电子发票

| 接口名称 | 功能 |

| `baohanapply` | 投保申请 |

- **文档格式**: OFD版式文档

## 八、常见问题

### 8.1 电子保函 vs 纸质保函

电子保函具有以下优势：
- **便捷**: 全程线上办理，无需线下递交
- **高效**: 实时出具，快速通知
- **防伪**: 数字签名，难以伪造
- **环保**: 无纸化办公

### 8.2 见费模式 vs 见保模式

- **见费模式**: 保费到账后出具保函
- **见保模式**: 保险公司出具保函后，再缴纳保费
