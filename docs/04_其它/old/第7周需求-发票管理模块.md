# 第7周需求 - 发票管理模块

**版本**: V1.0
**制定日期**: 2026-03-14
**制定人**: 电子保函业务专家
**状态**: 已优化

---

## 1. 业务概述

发票管理功能提供电子保函费用的发票开具、查询、红冲等全流程管理。

### 1.1 发票类型

| 类型 | 说明 | 税率 |
|------|------|------|
| 增值税专用发票 | 可抵扣进项税 | 6% |
| 增值税普通发票 | 不可抵扣 | 6% |
| 电子发票 | 电子形式 | 6% |

### 1.2 发票场景

| 场景 | 说明 |
|------|------|
| 投保开票 | 投保成功后开具保费发票 |
| 延期开票 | 延期成功后开具延期费发票 |
| 退保红冲 | 退保成功后红冲原发票 |

---

## 2. 功能需求

### 2.1 发票开具

**功能描述**: 开具保费发票

**用户角色**: 财务人员（金融机构）

**前置条件**:
- 投保申请已审核通过
- 保费已缴纳（见费模式）
- 未开具过对应保费的发票

**发票信息**:

| 字段 | 说明 | 必填 |
|------|------|------|
| 发票类型 | 专用/普通/电子 | 是 |
| 购买方名称 | 投保人名称 | 是 |
| 购买方税号 | 统一社会信用代码 | 是 |
| 购买方地址 | 注册地址 | 专票必填 |
| 购买方电话 | 联系电话 | 专票必填 |
| 购买方银行 | 开户银行 | 专票必填 |
| 购买方账号 | 银行账号 | 专票必填 |
| 销售方名称 | 出函机构名称 | 是 |
| 销售方税号 | 出函机构税号 | 是 |
| 货物或应税劳务名称 | 担保服务费 | 是 |
| 金额 | 不含税金额 | 是 |
| 税额 | 增值税税额 | 是 |
| 价税合计 | 含税金额 | 是 |

**开票流程**:
```
投保成功 → 申请开票 → 开具发票 → 交付投保人
```

### 2.2 发票查询

**功能描述**: 查询已开具发票信息

**用户角色**: 投标人、担保公司、金融机构

**查询条件**:

| 条件类型 | 说明 |
|----------|------|
| 发票号码 | 精确查询 |
| 发票代码 | 精确查询 |
| 保函编号 | 关联查询 |
| 投保人 | 模糊查询 |
| 开票日期 | 日期范围 |
| 发票状态 | 可用/已红冲/作废 |

**展示字段**:

| 字段 | 说明 |
|------|------|
| 发票号码 | 发票编号 |
| 发票代码 | 税务代码 |
| 发票类型 | 专用/普通/电子 |
| 购买方 | 投保人名称 |
| 销售方 | 出函机构名称 |
| 金额 | 不含税金额 |
| 税额 | 增值税税额 |
| 价税合计 | 含税金额 |
| 开票日期 | 开票时间 |
| 发票状态 | 可用/已红冲/作废 |
| 保函编号 | 关联保函 |

### 2.3 发票红冲

**功能描述**: 红冲（作废）已开具发票

**用户角色**: 财务人员（金融机构）

**适用场景**:
- 退保退款
- 开票信息错误
- 金额调整

**红冲类型**:

| 类型 | 说明 |
|------|------|
| 部分红冲 | 红冲部分金额 |
| 全额红冲 | 红冲全部金额 |

**红冲规则**:
- 退保红冲：自动红冲原发票
- 错误红冲：需填写红冲原因
- 部分红冲：红冲后需重新开具正确金额发票

**红冲流程**:
```
选择发票 → 填写红冲原因 → 执行红冲 → 生成红字发票 → 通知投保人
```

### 2.4 发票交付

**功能描述**: 向投保人交付发票

**交付方式**:

| 方式 | 说明 |
|------|------|
| 电子邮箱 | 发送到投保人邮箱 |
| 系统下载 | 提供发票下载链接 |
| 纸质寄送 | 纸质发票快递（需配置）|

## 3. 接口需求

### 3.1 发票开具接口

**接口路径**: `POST /api/v1/invoice/issue`

**请求参数**:
```json
{
  "guaranteeNo": "string",       // 保函编号
  "invoiceType": "string",        // INVOICE_TYPE_SPECIAL/INVOICE_TYPE_NORMAL/INVOICE_TYPE_ELECTRONIC
  "buyer": {
    "name": "string",            // 购买方名称
    "taxNo": "string",           // 购买方税号
    "address": "string",         // 购买方地址
    "phone": "string",           // 购买方电话
    "bank": "string",            // 购买方开户行
    "account": "string"          // 购买方账号
  },
  "items": [
    {
      "name": "string",          // 货物名称
      "spec": "string",          // 规格型号
      "unit": "string",          // 单位
      "quantity": "number",      // 数量
      "price": "number",         // 单价
      "amount": "number",        // 金额
      "taxRate": "number",       // 税率
      "taxAmount": "number"      // 税额
    }
  ],
  "deliveryMethod": "string"     // EMAIL/SYSTEM/POST
}
```

**响应参数**:
```json
{
  "code": "200",
  "message": "success",
  "data": {
    "invoiceNo": "string",        // 发票编号
    "invoiceCode": "string",     // 发票代码
    "issueDate": "string"        // 开票日期
  }
}
```

### 3.2 发票查询接口

**接口路径**: `GET /api/v1/invoice/list`

**请求参数**:
```json
{
  "invoiceNo": "string",         // 发票号码
  "invoiceCode": "string",       // 发票代码
  "guaranteeNo": "string",       // 保函编号
  "buyerName": "string",         // 购买方名称
  "status": "string",            // 发票状态
  "startDate": "string",         // 开票日期开始
  "endDate": "string",           // 开票日期结束
  "page": "number",
  "pageSize": "number"
}
```

### 3.3 发票详情接口

**接口路径**: `GET /api/v1/invoice/{invoiceNo}`

### 3.4 发票红冲接口

**接口路径**: `POST /api/v1/invoice/{invoiceNo}/void`

**请求参数**:
```json
{
  "voidType": "string",          // FULL/PARTIAL
  "voidAmount": "number",        // 红冲金额（部分红冲时）
  "reason": "string"             // 红冲原因
}
```

### 3.5 发票下载接口

**接口路径**: `GET /api/v1/invoice/{invoiceNo}/download`

**响应**: 返回发票文件（PDF/OFD）

## 7. 业务规则补充

### 7.1 税务系统对接规划

**阶段一（V1.0）**：
- 暂不接入税务系统实现在线开票
- 采用本地开具电子发票模式
- 发票格式符合税务规范要求

**阶段二（V2.0）**：
- 预留税务系统对接接口
- 支持百望、航天信息等主流税控厂商对接
- 实现在线开具增值税专用发票

### 7.2 发票红冲时间限制

| 红冲类型 | 时间限制 | 说明 |
|----------|---------|------|
| 退保红冲 | 无限制 | 退保成功后自动红冲 |
| 错误红冲 | 360天 | 开票之日起360天内 |
| 部分红冲 | 360天 | 开票之日起360天内 |

**说明**：
- 超过360天的发票不可红冲，需走作废流程
- 红冲后原发票状态变更为"已红冲"
- 红冲发票保留3年供审计查阅

### 7.3 纸质发票寄送规则

| 寄送方式 | 费用 | 时效 |
|----------|------|------|
| 普通快递 | 免费 | 3-5个工作日 |
| 加急快递 | 15元 | 1-2个工作日 |
| 到付 | 到付 | 以快递时效为准 |

### 7.4 合并开票规则

支持合并开票，具体规则：
- 合并条件：同一投保人、同一月份内的多笔保费
- 最高合并数量：10笔
- 合并后自动拆分明细
