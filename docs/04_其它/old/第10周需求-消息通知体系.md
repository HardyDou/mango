# 第10周需求-消息通知体系

**版本**: V1.0
**创建日期**: 2026-03-14
**产品经理**: 产品经理
**状态**: 草稿

---

## 一、功能概述

本章描述电子保函系统消息通知体系的功能需求，建立统一的站内消息、短信、邮件、App推送通知渠道，确保用户及时获取业务进度和系统动态。

---

## 二、通知场景

### 2.1 业务通知场景

| 场景 | 通知类型 | 接收角色 | 默认推送方式 |
|------|---------|---------|-------------|
| 投保申请提交 | 站内消息+App推送 | 投标人 | 站内消息+App推送 |
| 投保审核通过 | 站内消息+App推送+短信 | 投标人 | 站内消息+App推送 |
| 投保审核拒绝 | 站内消息+App推送+短信 | 投标人 | 站内消息+App推送 |
| 保函出具成功 | 站内消息+App推送+短信 | 投标人 | 站内消息+App推送 |
| 保函即将到期 | 站内消息+App推送 | 投标人 | 站内消息+App推送 |
| 退保申请提交 | 站内消息+App推送 | 投标人 | 站内消息+App推送 |
| 退保审核通过 | 站内消息+App推送+短信 | 投标人 | 站内消息+App推送 |
| 退款到账 | 站内消息+App推送+短信 | 投标人 | 站内消息+App推送 |
| 发票开具成功 | 站内消息+App推送 | 投标人 | 站内消息+App推送 |
| 理赔申请收到 | 站内消息 | 出函机构 | 站内消息 |
| 理赔审核结果 | 站内消息+App推送 | 受益人 | 站内消息+App推送 |

### 2.2 系统通知场景

| 场景 | 通知类型 | 接收角色 |
|------|---------|---------|
| 系统维护通知 | 站内消息+邮件 | 所有用户 |
| 新功能上线通知 | 站内消息 | 所有用户 |
| 账户安全提醒 | 站内消息+短信+邮件 | 相关用户 |
| 密码修改 | 站内消息+短信 | 相关用户 |
| 实名认证结果 | 站内消息+App推送 | 申请人 |

## 四、推送渠道

### 4.1 App推送（Push Notification）

#### 4.1.1 推送内容

| 字段 | 说明 |
|------|------|
| 标题 | 推送标题（≤50字符） |
| 内容 | 推送内容（≤200字符） |
| 跳转类型 | 打开App/跳转页面/打开链接 |
| 跳转参数 | 跳转目标参数 |

#### 4.1.2 推送策略

| 场景 | 推送策略 |
|------|---------|
| 普通通知 | 实时推送 |
| 营销通知 | 限定推送时间（9:00-21:00） |
| 敏感操作 | 需要用户确认后推送 |

#### 4.1.3 推送统计

| 指标 | 说明 |
|------|------|
| 发送量 | 推送发送总数 |
| 到达率 | 成功到达设备的比例 |
| 点击率 | 用户点击推送的比例 |
| 打开率 | 用户打开App的比例 |

---

### 4.2 短信通知（SMS）

#### 4.2.1 短信模板

| 模板类型 | 模板内容示例 |
|---------|-------------|
| 审核通知 | 【电子保函】您的投保申请（项目：{projectName}）已{status}，详情请登录App查看。 |
| 出函通知 | 【电子保函】您的保函已出具，保函编号：{guaranteeNo}，请登录App查看详情。 |
| 退款通知 | 【电子保函】您的退款（金额：{amount}元）已到账，请注意查收。 |
| 验证码 | 【电子保函】验证码：{code}，5分钟内有效，请勿泄露。 |

#### 4.2.2 短信发送规则

| 规则 | 说明 |
|------|------|
| 发送时机 | 业务状态变更时触发 |
| 发送频率 | 同一业务单不超过3条 |
| 退订 | 支持回复TD退订营销短信 |
| 发送时间 | 限制发送时间（8:00-22:00） |

## 五、通知服务架构

### 5.1 通知服务模块

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  业务触发    │────▶│  通知服务    │────▶│  消息存储   │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
            ┌──────────────┼──────────────┐
            ▼              ▼              ▼
     ┌───────────┐  ┌───────────┐  ┌───────────┐
     │ App推送    │  │  短信     │  │  邮件     │
     └───────────┘  └───────────┘  └───────────┘
```

### 5.2 通知队列设计

| 队列 | 说明 | 处理方式 |
|------|------|---------|
| 站内消息队列 | 站内消息写入 | 同步写入 |
| App推送队列 | App推送任务 | 异步处理 |
| 短信队列 | 短信发送任务 | 异步处理 |
| 邮件队列 | 邮件发送任务 | 异步处理 |

## 七、接口需求

### 7.1 通知管理API

| 接口 | 功能 |
|------|------|
| POST /api/v1/notice/send | 发送通知 |
| POST /api/v1/notice/batchSend | 批量发送通知 |
| GET /api/v1/notice/list | 消息列表 |
| GET /api/v1/notice/{id} | 消息详情 |
| PUT /api/v1/notice/{id}/read | 标记已读 |
| PUT /api/v1/notice/readAll | 全部已读 |
| DELETE /api/v1/notice/{id} | 删除消息 |
| GET /api/v1/notice/config | 获取通知配置 |
| PUT /api/v1/notice/config | 更新通知配置 |

### 7.2 用户通知设置API

| 接口 | 功能 |
|------|------|
| GET /api/v1/user/notice/settings | 获取通知设置 |
| PUT /api/v1/user/notice/settings | 更新通知设置 |

### 7.3 推送服务API

### 7.4 短信服务API

### 7.5 邮件服务API

| 接口 | 功能 |

## 八、非功能需求

### 8.1 性能要求

| 指标 | 要求 |
|------|------|
| 消息推送延迟 | ≤5秒 |
| 短信发送延迟 | ≤10秒 |
| 邮件发送延迟 | ≤30秒 |
| 并发推送能力 | 支持10000+并发推送 |

### 8.2 可靠性要求

| 指标 | 要求 |
|------|------|
| 推送成功率 | ≥99% |
| 短信发送成功率 | ≥99% |
| 邮件发送成功率 | ≥95% |
| 消息不丢失 | 消息持久化存储 |

### 8.3 安全性要求

| 要求 | 说明 |
|------|------|
| 内容安全 | 敏感信息脱敏处理 |
| 推送安全 | 签名验证 |
| 短信安全 | 验证码防刷、时效控制 |
