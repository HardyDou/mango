## 2.3 国密算法实现方案

### 2.3.1 方案选型

| 方案 | 实现方式 | 适用场景 | 优点 | 缺点 |
|------|---------|---------|------|------|
| **软件方案** | BouncyCastle | 通用场景（推荐） | 部署简单、成本低 | 性能略低于硬件 |
| 硬件方案 | SJJ1312金融数据密码机 | 高并发交易 | 性能高、安全性强 | 成本高、需要运维 |

**推荐方案**: 初期采用 **BouncyCastle软件方案**，后续根据业务量可平滑升级至硬件方案。

### 2.3.2 SM3 报文摘要

- **算法**: 国密SM3杂凑算法
- **输出**: 256位（32字节）哈希值
- **应用场景**:
  - 文件完整性校验
  - 数字签名原文摘要
  - 关键数据防篡改

```java
// SM3 消息摘要示例
SM3Digest digest = new SM3Digest();
byte[] data = "保函内容".getBytes(StandardCharsets.UTF_8);
digest.update(data, 0, data.length);
byte[] hash = new byte[digest.getDigestSize()];
digest.doFinal(hash, 0);
// hash 即为SM3摘要结果（32字节）
```

### 2.3.3 SM4 对称加密

- **算法**: 国密SM4分组密码（ECB/CBC模式）
- **密钥长度**: 128位
- **块大小**: 128位
- **应用场景**:
  - 敏感数据加密存储
  - 通信数据加密传输
  - 密钥加密

```java
// SM4 CBC模式加密示例
SM4Util sm4 = new SM4Util(secretKey); // 16字节密钥
String encrypted = sm4.encryptCBC(plainText);
String decrypted = sm4.decryptCBC(encrypted);
```

### 2.3.4 密钥管理方案

| 层级 | 密钥类型 | 存储方式 | 轮换周期 |
|------|---------|---------|---------|
| 主密钥 | KEK | 物理安全模块/密钥托管 | 年度 |
| 工作密钥 | 数据加密密钥 | 加密存储 | 季度 |
| 会话密钥 | 临时加密 | 内存存储 | 每次会话 |

**密钥管理要点**:
1. 主密钥采用分开管理机制（3人2票）
2. 工作密钥自动轮换，无需人工干预
3. 密钥使用硬件安全模块(HSM)保护
4. 完整的密钥备份与恢复机制

## 2.4 OFD版式文档方案

### 2.4.1 OFD格式说明

OFD（Open Fixed-layout Document）是我国自主研发的版式文档格式，符合国家标准GB/T 33190-2016。电子保函需生成OFD格式文档，具备不可篡改性。

### 2.4.2 OFD生成库选型

| 供应商 | 产品 | 特点 | 适用场景 | 成本 |
|--------|------|------|---------|------|
| **数科OFD** | 数科OFD SDK | 性能稳定、支持批量处理 | 政务、金融（推荐） | 中等 |
| 方正电子 | 方正OFD | 版式设计能力强 | 出版、印刷 | 中等 |
| **弘成科技** | OFDToolkit | Java SDK、集成简单 | 通用场景（推荐） | 较低 |
| 书生电子 | 书生OFD | 老牌厂商 | 政务系统 | 中等 |

**推荐方案**: 初期采用 **弘成科技OFDToolkit**，成本较低且易于集成；后期可根据政务项目需求升级至数科OFD。

### 2.4.3 OFD处理流程

```
保函数据 → 模板填充 → OFD渲染 → 数字签名 → 文件存储
```

关键功能：
- 模板引擎：支持动态数据填充
- 电子签章：支持骑缝章、坐标准签章
- 数字签名：支持国密SM2签名验证

## 5. 安全设计

### 5.1 认证授权

- 基于 JWT 的无状态认证
- 权限控制采用 RBAC 模型
- 接口调用需要签名验证
- 敏感操作需要二次验证

### 5.2 数据安全

- 传输层: HTTPS + TLS 1.3
- 存储层: 敏感数据 SM4 加密存储
- 文件: SM3 哈希校验
- 数据库: 定期备份 + 读写分离

### 5.3 国密算法

```java
// SM3 消息摘要
String sm3Hash = SM3Util.hash(data);

// SM4 加密/解密
String encrypted = SM4Util.encrypt(data, key);
String decrypted = SM4Util.decrypt(encrypted, key);
```
