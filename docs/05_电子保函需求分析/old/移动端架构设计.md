# 电子保函系统移动端架构设计

## 一、技术选型

### 1.1 跨平台方案选择

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Flutter (推荐)** | 性能好、自研渲染引擎、跨平台一致 | 生态比 RN 稍弱 | 需要高性能、高体验 |
| React Native | 生态成熟、前端开发者多 | 性能略差、系统原生感弱 | 快速开发、团队熟悉 RN |

### 1.2 推荐技术栈

```
# Flutter 方案
- 框架: Flutter 3.x
- 语言: Dart 3.x
- 状态管理: Riverpod / BLoC
- 网络: Dio
- 本地存储: Hive / SharedPreferences
- 路由: GoRouter
- UI 组件: Material Design 3

# Android 原生
- 语言: Kotlin 1.9+
- 框架: Jetpack Compose
- 架构: MVVM + Clean Architecture
- DI: Hilt
- 网络: Retrofit + OkHttp
- 存储: Room + DataStore

# iOS 原生
- 语言: Swift 5.9+
- 框架: SwiftUI
- 架构: MVVM
- 网络: Alamofire
- 存储: Core Data / SQLite
```

## 五、接口设计

### 5.1 API 模块

移动端 API 与 Web 端共用后端接口，参考前端架构设计中的 API 规范。

```dart
// bond_api.dart
class BondApi {
  // 获取保函列表
  Future<List<Bond>> getBondList(BondListParams params);

  // 获取保函详情
  Future<Bond> getBondDetail(String id);

  // 提交保函申请
  Future<BondApplyResult> applyBond(BondApplyParams params);

  // 获取申请进度
  Future<BondProgress> getProgress(String id);

  // 下载保函文件
  Future<String> downloadBondFile(String id);

  // 签章上传
  Future<void> uploadSignature(SignatureData data);
}
```

### 5.2 关键接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/bond/list` | GET | 获取保函列表 |
| `/api/bond/:id` | GET | 获取保函详情 |
| `/api/bond/apply` | POST | 提交保函申请 |
| `/api/bond/:id/progress` | GET | 获取申请进度 |
| `/api/bond/:id/download` | GET | 下载保函文件 |
| `/api/auth/sms/send` | POST | 发送验证码 |
| `/api/auth/login` | POST | 用户登录 |
| `/api/user/profile` | GET/POST | 用户资料 |

## 六、安全设计

### 6.1 认证安全

- Token 存储在安全存储区域（Keychain/Keystore）
- 每次请求携带 Token
- Token 过期自动刷新
- 支持生物识别（指纹/ Face ID）

### 6.2 数据安全

- HTTPS 传输加密
- 国密 SM3/SM4 算法支持
- 本地敏感数据加密存储
- 防止应用截图/录屏

### 6.3 应用安全

- 应用完整性检测
- 防止调试/root 检测
- 敏感操作二次验证
