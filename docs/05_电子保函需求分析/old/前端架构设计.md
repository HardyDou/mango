# 电子保函系统前端架构设计

## 一、技术选型

### 1.1 核心框架

| 技术 | 选择 | 理由 |
|------|------|------|
| **前端框架** | Vue 3 + TypeScript | 生态成熟、学习成本低、Composition API 支持良好 |
| **构建工具** | Vite | 开发体验好、构建速度快 |
| **UI 组件库** | Element Plus | Vue 3 官方支持、组件丰富、文档完善 |
| **状态管理** | Pinia | Vue 3 官方推荐、TypeScript 支持好 |
| **路由** | Vue Router 4 | Vue 3 官方路由 |

### 1.2 技术栈清单

```
- 核心框架: Vue 3.4+ (Composition API + `<script setup>`)
- 语言: TypeScript 5.0+
- 构建工具: Vite 5.0+
- UI 组件: Element Plus 2.5+
- 状态管理: Pinia 2.1+
- 路由: Vue Router 4.5+
- HTTP 客户端: Axios
- 工具库: VueUse, Lodash
- CSS 预处理器: SCSS
- 代码规范: ESLint + Prettier
```

## 七、接口规范

### 7.1 请求封装

```typescript
// utils/request.ts
import axios from 'axios'
import type { AxiosInstance, AxiosRequestConfig, AxiosResponse } from 'axios'

const service: AxiosInstance = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 30000,
})

// 请求拦截器
service.interceptors.request.use(
  (config) => {
    const token = useAuthStore().token
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  (error) => Promise.reject(error)
)

// 响应拦截器
service.interceptors.response.use(
  (response: AxiosResponse) => response.data,
  (error) => {
    // 统一错误处理
    return Promise.reject(error)
  }
)

export default service
```

### 7.2 API 模块划分

```typescript
// api/modules/bond.ts
import request from '@/utils/request'
import type { Bond, BondApplyParams, BondListParams } from '@/types/bond'

export const getBondList = (params: BondListParams) =>
  request.get<{ list: Bond[]; total: number }>('/bond/list', { params })

export const getBondDetail = (id: string) =>
  request.get<Bond>(`/bond/${id}`)

export const createBond = (data: BondApplyParams) =>
  request.post('/bond', data)

export const approveBond = (id: string, data: { status: string; comment?: string }) =>
  request.patch(`/bond/${id}/approve`, data)
```
