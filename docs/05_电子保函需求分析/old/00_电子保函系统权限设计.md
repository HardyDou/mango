# 电子保函系统权限系统设计

## 一、权限模型设计

### 1.1 模型选择：RBAC + BRAC

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        权限模型: RBAC + BRAC                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   RBAC (Role-Based Access Control)                                        │
│   - 功能权限: 菜单、按钮、页面                                            │
│                                                                             │
│   BRAC (Business Role-Based Access Control)                               │
│   - 数据权限: 组织、通道、业务范围                                         │
│                                                                             │
│   ABAC (Attribute-Based Access Control)                                   │
│   - 动态权限: 时间、环境、上下文                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 整体架构

```
┌─────────────────────────────────────────────────────────────────────────────┐

│  ┌──────────────────────────────────────────────────────────────────┐    │
│  │                        权限决策引擎                                  │    │
│  │   ┌──────────────┐  ┌──────────────┐  ┌──────────────┐        │    │
│  │   │ 功能权限检查  │  │  数据权限检查  │  │  动态策略检查 │        │    │
```

### 2.2 角色定义表

```sql
-- 角色定义表
CREATE TABLE sys_role (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_code       VARCHAR(50) NOT NULL COMMENT '角色编码',
    role_name       VARCHAR(100) NOT NULL COMMENT '角色名称',
    role_type       VARCHAR(20) NOT NULL COMMENT '角色类型(PLATFORM/TENANT/BUSINESS)',
    parent_id       BIGINT COMMENT '父角色ID',
    role_level      INT DEFAULT 1 COMMENT '角色级别',
    data_scope      VARCHAR(20) DEFAULT 'SELF' COMMENT '数据范围(SELF/DEPT/ALL/CUSTOM)',
    status          VARCHAR(20) DEFAULT 'ACTIVE' COMMENT '状态',
    description     VARCHAR(500) COMMENT '描述',
    create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_role_code (role_code)
);

-- 角色权限关联表
CREATE TABLE sys_role_permission (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_id         BIGINT NOT NULL COMMENT '角色ID',
    permission_id   BIGINT NOT NULL COMMENT '权限ID',
    permission_type VARCHAR(20) NOT NULL COMMENT '权限类型(MENU/BUTTON/API/DATA)',
    create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_role_perm (role_id, permission_id, permission_type)
);

-- 用户角色关联表
CREATE TABLE sys_user_role (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT NOT NULL COMMENT '用户ID',
    role_id         BIGINT NOT NULL COMMENT '角色ID',
    tenant_id       BIGINT COMMENT '租户ID',
    create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_role (user_id, role_id, tenant_id)
);
```

### 4.1 权限分类

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           权限分类                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  功能权限 (Functional Permission)                                            │
│  ├── 菜单权限    - 能看到哪些菜单                                           │
│  ├── 页面权限    - 能访问哪些页面                                           │

│  数据权限 (Data Permission)                                                 │
│  ├── 组织权限    - 能查看哪些组织的数据                                       │
│  ├── 通道权限    - 能操作哪些通道(银行/保险)                                │
│  ├── 项目权限    - 能操作哪些项目/标段                                      │
│  └── 金额权限    - 能操作多大金额的保单                                      │
│                                                                             │
│  动态权限 (Dynamic Permission)                                              │
│  ├── 时间权限    - 允许访问的时间段                                          │
│  ├── IP权限     - 允许访问的IP范围                                          │
│  └── 设备权限    - 允许访问的设备类型                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 数据权限实现方案

#### 方案一：注解方式 (推荐)

```java
// 数据权限注解
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface DataScope {
    String dataType() default "ORG";       // 数据类型
    String dataField() default "org_id";    // 关联字段
    String dataScope() default "SELF";     // 权限范围
}

// Service层使用
@Service
public class GuaranteeService {
    
    @DataScope(dataType = "CHANNEL", dataField = "channel_id")
    public List<Guarantee> listGuarantees(GuaranteeQuery query) {
        // 自动注入数据权限过滤
        return guaranteeMapper.selectList(query);
    }

@Aspect
@Component
public class DataScopeAspect {
```

```java
// MyBatis拦截器实现
@Intercepts({
    @Signature(type = StatementHandler.class, method = "prepare", 
               args = {Connection.class, Integer.class})
})
public class DataScopeInterceptor implements Interceptor {
    
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        StatementHandler statementHandler = (StatementHandler) invocation.getTarget();
        BoundSql boundSql = statementHandler.getBoundSql();
        
        // 获取数据权限SQL
        String dataScopeSql = DataScopeContextHolder.get();
        if (StringUtils.isNotEmpty(dataScopeSql)) {
            // 注入SQL
            String originalSql = boundSql.getSql();
            String newSql = injectDataScopeSql(originalSql, dataScopeSql);
            MetaObject.forObject(boundSql).setValue("sql", newSql);
        }
        
        return invocation.proceed();
    }
    
    private String injectDataScopeSql(String originalSql, String dataScopeSql) {
        // 在WHERE条件后添加数据权限条件
        if (originalSql.toUpperCase().contains("WHERE")) {
            return originalSql + " AND " + dataScopeSql;
        } else {
            return originalSql + " WHERE " + dataScopeSql;
        }
    }
}
```

```java
// 权限检查服务
@Service("ps")
public class PermissionService {
    
    // 检查用户是否有权限
    public boolean hasPermission(String module, String action) {
        User currentUser = SecurityUtils.getCurrentUser();
        if (currentUser.isAdmin()) {
            return true; // 管理员拥有所有权限
        }
        
        // 获取用户权限
        List<String> permissions = getUserPermissions(currentUser.getId());
        
        // 构建权限标识
        String permCode = module + ":" + action;
        
        return permissions.contains(permCode);
    }
    
    // 检查用户角色
    public boolean hasRole(String roleCode) {
        User currentUser = SecurityUtils.getCurrentUser();
        return currentUser.getRoles().contains(roleCode);
    }
    
    // 获取用户权限列表
    public List<String> getUserPermissions(Long userId) {
        // 从缓存获取
        List<String> cached = redisTemplate.opsForValue()
            .get("user:permissions:" + userId);
        if (cached != null) {
            return cached;
        }
        
        // 从数据库查询
        List<String> permissions = permissionMapper
            .selectByUserId(userId)
            .stream()
            .map(Permission::getPermCode)
            .collect(Collectors.toList());
        
        // 缓存5分钟
        redisTemplate.opsForValue()
            .set("user:permissions:" + userId, permissions, 
                 5, TimeUnit.MINUTES);
        
        return permissions;
    }
}
```

## 九、总结

### 9.1 设计要点

| 要点 | 说明 |
|------|------|
| **RBAC为主** | 功能权限基于角色，简单易维护 |
| **BRAC为辅** | 数据权限基于业务，灵活可控 |
| **注解驱动** | 声明式权限控制，代码简洁 |
| **AOP拦截** | 非侵入式，统一处理 |
| **缓存优化** | 减少数据库查询，提升性能 |
| **多租户** | 租户隔离 + 数据隔离 |
