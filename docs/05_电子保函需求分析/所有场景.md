## 2. 核心场景

### 2.1 项目信息同步

| 场景 | 说明 |
|------|------|
| **触发方** | 交易系统 |
| **触发条件** | 新项目发布/项目信息变更 |
| **处理内容** | 接收并解析项目信息、标段信息、保证金规则 |
| **后续动作** | 创建项目缓存、配置支付规则 |

**数据项**：

| 字段 | 说明 |
|------|------|
| projectno | 项目编号 |
| tenderno | 招标编号 |
| projectname | 项目名称 |
| bidbondamount | 保证金金额 |
| deadline | 投标截止时间 |

### 2.2 投保申请

| 场景 | 说明 |
|------|------|
| **触发方** | 投标人 |
| **触发条件** | 投标人选择保函、提交申请 |
| **处理内容** | 校验资格、计算保费、推送金融机构 |
| **后续动作** | 等待支付、等待出函 |

**数据项**：

| 字段 | 说明 |
|------|------|
| biddername | 投标人名称 |
| biddercode | 统一社会信用代码 |
| bzjamount | 保证金金额 |
| biaoduanno | 标段编号 |

### 2.3 保费支付

| 场景 | 说明 |
|------|------|
| **触发方** | 投标人 |
| **触发条件** | 投保申请成功后发起支付 |
| **处理内容** | 发起支付、接收回调、更新订单状态 |
| **后续动作** | 支付成功→触发出函 |

**支付方式**：

| 方式 | 主体 | 说明 |
|------|------|------|
| 企业网银 | 企业 | 对公转账 |
| 微信支付 | 个人 | 扫码 |
| 支付宝 | 个人 | 扫码 |

### 2.4 保函开具

| 场景 | 说明 |
|------|------|
| **触发方** | 支付成功 / 金融机构主动 |
| **触发条件** | 保费支付成功 + 风控通过 |
| **处理内容** | 生成保函文件、电子签章、返回结果 |
| **后续动作** | 推送保函给交易系统/投标人 |

**出函类型**：

| 类型 | 说明 |
|------|------|
| 银行直开 | 银行直接开具 |
| 分离式 | 担保公司担保，银行出具 |
| 保证保险 | 保险公司出具保单 |
| 担保保函 | 担保公司直接出具 |

### 2.5 保函验真

| 场景 | 说明 |
|------|------|
| **触发方** | 交易系统/招标人 |
| **触发条件** | 开标前/开标时验证保函真伪 |
| **处理内容** | 验证签章、校验保函状态 |
| **后续动作** | 返回验真结果 |

#### 2.5.1 批量验真（新增）

| 场景 | 说明 |
|------|------|
| **触发方** | 交易系统/招标人 |
| **触发条件** | 开标时批量验证多家投标人保函 |
| **处理内容** | 批量验证签章、校验状态 |
| **后续动作** | 返回批量验真结果报告 |

**批量验真流程**：
```
┌─────────────────────────────────────────────────────────────────┐
│                    批量验真流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   交易系统 ──▶ 平台                                            │
│                                                                  │
│   1. 提交批量验真请求                                          │
│      ├── 标段编号                                               │
│      ├── 投标人列表（批量）                                     │
│      └── 验真时间点                                            │
│                                                                  │
│   2. 平台处理                                                  │
│      ├── 逐条验证每份保函                                       │
│      ├── 汇总验真结果                                           │
│      └── 生成验真报告                                           │
│                                                                  │
│   3. 返回结果                                                  │
│      ├── 成功列表                                               │
│      ├── 失败列表                                               │
│      └── 异常列表                                               │
└─────────────────────────────────────────────────────────────────┘
```

**批量验真接口设计**：
```json
{
  "batchVerifyRequest": {
    "sectionNo": "标段编号",
    "tenderNo": "招标编号",
    "bidders": [
      {
        "bidderName": "投标人名称",
        "bidderCode": "统一社会信用代码",
        "guaranteeNo": "保函编号"
      }
    ],
    "verifyTime": "验真时间"
  }
}
```

#### 2.5.2 批量出函（新增）

| 场景 | 说明 |
|------|------|
| **触发方** | 平台/运营人员 |
| **触发条件** | 多家机构同时响应出函 |
| **处理内容** | 收集多份保函、汇总结果 |
| **后续动作** | 返回批量出函结果 |

**批量出函流程**：
```
┌─────────────────────────────────────────────────────────────────┐
│                    批量出函流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   场景：同一标段多家投标人同时投保，多家机构响应                  │
│                                                                  │
│   1. 批量出函请求                                              │
│      ├── 标段编号                                               │
│      └── 投标人列表                                            │
│                                                                  │
│   2. 平台处理                                                  │
│      ├── 并行向多家机构获取保函                                 │
│      ├── 汇总保函结果                                           │
│      └── 生成出函汇总                                           │
│                                                                  │
│   3. 返回结果                                                  │
│      ├── 出函成功列表                                           │
│      ├── 出函失败列表                                           │
│      └── 处理中列表                                             │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 延伸场景

### 3.1 退保注销

| 场景 | 说明 |
|------|------|
| **触发方** | 投标人/交易系统 |
| **触发条件** | 投标未中标/项目取消/保函到期 |
| **处理内容** | 注销保函、释放额度、退费 |
| **后续动作** | 通知各方、记录台账 |

**退保类型**：

| 类型 | 说明 |
|------|------|
| 自愿退保 | 投标人主动申请 |
| 过期退保 | 保函到期自动注销 |
| 强制退保 | 违规强制注销 |

### 3.2 理赔

| 场景 | 说明 |
|------|------|
| **触发方** | 招标人/交易系统 |
| **触发条件** | 触发赔付条件（如投标人违约） |
| **处理内容** | 受理理赔、核实情况、赔付 |
| **后续动作** | 支付赔款、追偿 |

### 3.3 保单查询

| 场景 | 说明 |
|------|------|
| **触发方** | 投标人/招标人/平台运营 |
| **触发条件** | 查询保函状态 |
| **处理内容** | 查询订单/保函信息 |
| **后续动作** | 返回查询结果 |

### 3.4 发票

| 场景 | 说明 |
|------|------|
| **触发方** | 投标人 |
| **触发条件** | 申请开具发票 |
| **处理内容** | 转发给出函机构、开具发票 |
| **后续动作** | 下载/寄送发票 |

> 注：发票由出函机构（银行/保险/担保）开具，平台仅做展示/转发

### 3.5 通知

| 场景 | 说明 |
|------|------|
| **触发方** | 系统自动 |
| **触发条件** | 状态变更 |
| **处理内容** | 短信/邮件/站内通知 |
| **后续动作** | 送达用户 |

## 5. 异常场景

### 5.1 支付异常

| 场景 | 处理 |
|------|------|
| 支付超时 | 自动取消订单 |
| 支付失败 | 通知投标人重试 |
| 重复支付 | 自动退款 |

### 5.2 出函异常

| 场景 | 处理 |
|------|------|
| 出函超时 | 自动重试/人工介入 |
| 出函失败 | 退款+通知 |
| 金融机构异常 | 切换通道/人工处理 |

### 5.3 验真异常

| 场景 | 处理 |
|------|------|
| 验真失败 | 返回详细原因 |
| 签章无效 | 标记异常、预警 |
