# DDD领域划分与微服务设计

本文档基于领域驱动设计（DDD）对电子保函系统进行服务划分。

---

## 一、DDD领域划分

### 1.1 领域模型

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         电子保函系统领域模型                                 │

│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                      支撑域 (Supporting Domain)                      │   │

│  │  │   支付域     │  │   通知域     │  │   风控域     │  │  文件域  │ │   │
│  │  │  (Payment) │  │  (Notify)   │  │  (Risk)    │  │  (File) │ │   │
│  │  └─────────────┘  └─────────────┘  └─────────────┘  └─────────┘ │   │
│  │                                                                     │   │

│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │

│  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐               │   │

└─────────────────────────────────────────────────────────────────────────────┘
```

```
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  baohan-notify     (通知服务)                                         │  │
│  │  - 短信通知                                                           │  │

│  │  - 微信通知                                                           │  │

│  ┌─────────────────────────────────────────────────────────────────────┐  │

│  ┌─────────────────────────────────────────────────────────────────────┐  │
```

## 七、与原规划对比

### 变更说明

| 原规划 | 新规划 | 说明 |
|--------|--------|------|
| issue-service | 合并到baohan-guarantee | 核心域统一管理 |
| refund-service | 合并到baohan-guarantee | 核心域统一管理 |
| user-service | baohan-upmsp | 统一用户权限 |
| institution-service | baohan-institution | 出函机构服务 |
| 新增 | baohan-channel | 交易系统通道服务 |
| invoice-service | 合并到baohan-file | 文件服务统一管理 |

### 服务数量

| 维度 | 数量 |
|------|------|
| 核心域 | 1个 |
| 支撑域 | 5个（支付、发票、通知、风控、文件） |
| 通用域 | 3个 |
| 基础设施 | 1个 |
| **总计** | **10个** |
