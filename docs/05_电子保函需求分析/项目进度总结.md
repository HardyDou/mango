## 项目概述

电子保函系统是一个 B2B 平台，连接：
- **上游**：交易系统（公共资源交易中心、招标平台）
- **下游**：金融机构（银行、保险公司、担保公司）

## 需求要点

### 业务类型
| 类型 | 说明 |
|------|------|
| 银行电子保函 | 银行直接开具 |
| 分离式银行保函 | 担保公司担保，银行出具 |
| 保证保险保单 | 保险公司出具 |
| 担保保函 | 担保公司出具 |

### 使用场景
- 投标保证金保函（≤80万）
- 履约保证金保函（合同价5-10%）
- 工程质量保证金保函（合同价3%）
- 农民工工资支付保函

### 核心流程
```
项目信息同步 → 投保申请 → 保费支付 → 保函开具 → 电子签章 → 保函验真
                                ↓
                         退保/注销/理赔
```

---

## 关键技术决策

### 1. 分布式事务
- **方案**：可靠消息 + 本地消息表
- **原因**：无账户余额概念，TCC 不适用

### 2. 电子签章
- **方案**：多方案设计
- **测试**：自研 OFD 生成
- **生产**：第三方 SDK

### 3. 发票服务
- **独立服务**：baohan-invoice

### 4. 保费计算
- **灵活模型**：支持多种计费方式
