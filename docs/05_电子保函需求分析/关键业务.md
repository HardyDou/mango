#### 2.2.1 分离式银行保函流程

```
┌─────────────────────────────────────────────────────────────────┐
│                分离式银行保函流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   概述：担保公司提供担保，银行出具保函                          │
│                                                                  │
│   流程：                                                       │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│   │  担保公司  │◀───│  电子保函  │───▶│   银行   │              │
│   │  (担保)    │    │   平台    │    │  (出函)  │              │
│   └──────────┘    └──────────┘    └──────────┘              │
│                                                                  │
│   详细步骤：                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  1. 投标人向担保公司申请担保                              │ │
│   │     ├── 提交企业资料                                     │ │
│   │     └── 担保公司审核授信                                  │ │
│   ├─────────────────────────────────────────────────────────┤ │
│   │  2. 担保公司向银行提交分离式保函申请                     │ │
│   │     ├── 担保公司出具担保函                               │ │
│   │     └── 推送至银行                                       │ │
│   ├─────────────────────────────────────────────────────────┤ │
│   │  3. 银行审核并出具保函                                  │ │
│   │     ├── 审核担保函有效性                                 │ │
│   │     └── 开具银行保函                                     │ │
│   ├─────────────────────────────────────────────────────────┤ │
│   │  4. 保函回传至平台                                      │ │
│   │     ├── 银行推送保函至平台                              │ │
│   │     └── 平台通知投标人                                   │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                  │
│   涉及主体：                                                    │
│   ├── 投标人（申请方）                                        │
│   ├── 担保公司（担保方）                                      │
│   ├── 银行（出函方）                                          │
│   ├── 电子保函平台（中介方）                                   │
│   └── 交易系统（接收方）                                       │
│                                                                  │
│   数据流：                                                     │
│   担保公司授信 → 平台投保申请 → 担保公司担保确认 →            │
│   银行出函 → 平台接收保函 → 推送至交易系统                     │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 业务规则

### 3.1 保函开具校验规则

```
┌─────────────────────────────────────────────────────────────┐
│                     保函开具校验规则                          │
├─────────────────────────────────────────────────────────────┤
│  ✓ 投标人资格校验                                           │
│    - 营业执照有效性                                          │
│    - 未被列入失信名单                                        │
│    - 投标资格符合                                           │
│                                                              │
│  ✓ 项目信息校验                                             │
│    - 项目状态为有效                                         │
│    - 标段可投                                               │
│    - 保证金金额符合                                         │
│                                                              │
│  ✓ 额度校验                                                 │
│    - 授信额度充足                                           │
│    - 保额在产品限额内                                       │
│    - 保费计算正确                                           │
│                                                              │
│  ✓ 有效期校验                                              │
│    - 保函生效日期 >= 投标截止日期                            │
│    - 保函到期日期 >= 投标有效期 + 30天                       │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 风控规则

| 规则类型 | 规则内容 | 处理方式 |
|----------|----------|----------|
| **黑名单校验** | 投标人/企业是否在黑名单 | 拒绝投保 |
| **关联校验** | 是否存在关联企业重复投保 | 预警 |
| **额度校验** | 是否超过授信额度 | 拒绝/降额 |
| **历史违约** | 历史是否有违约记录 | 拒绝/提高保费 |
| **反洗钱** | 是否符合反洗钱要求 | 名单筛查 |

## 4. 保费计算规则

### 4.1 保费计算特点

> **重要**：保费计算方式需灵活配置，支持多种计算模型

### 4.2 计算模型

```
┌─────────────────────────────────────────────────────────────────┐
│                    保费计算模型                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  模型一: 固定费率                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  保费 = 保证金金额 × 费率                                │   │
│  │  例：80万 × 0.5% = 4000元                              │   │
│  └─────────────────────────────────────────────────────────┘   │
│  配置项：                                                       │
│  ├── 费率 (如 0.5%)                                          │
│  └── 最低/最高保费                                            │
│                                                                  │
│  模型二: 阶梯费率                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  保证金 < 10万: 0.6%                                    │   │
│  │  10万 ≤ 保证金 < 50万: 0.5%                            │   │
│  │  保证金 ≥ 50万: 0.4%                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│  配置项：                                                       │
│  └── 阶梯区间 + 对应费率                                        │
│                                                                  │
│  模型三: 固定保费                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  保费 = 固定金额                                         │   │
│  │  例：无论金额多少，保费均为 500元                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  模型四: 公式计算                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  保费 = f(保证金金额, 保函类型, 机构, 期限...)          │   │
│  │  例：保证金金额 × 机构费率 × 期限系数                    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 保费计算配置

```
┌─────────────────────────────────────────────────────────────────┐
│                保费计算配置表设计                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  机构费率配置 (baohan-institution)                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  institution_id: 机构ID                                 │   │
│  │  guarantee_type: 保函类型(投标/履约/农民工)            │   │
│  │  calc_model: 计算模型(fixed/step/fixed_amount/formula) │   │
│  │  rate: 费率 (如 0.005)                                  │   │
│  │  min_premium: 最低保费                                   │   │
│  │  max_premium: 最高保费                                   │   │
│  │  step_config: 阶梯配置(JSON)                            │   │
│  │  formula: 公式配置                                      │   │
│  │  effective_date: 生效日期                                │   │
│  │  status: 状态                                            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  配置优先级：                                                   │
│  机构费率 > 通道默认费率 > 系统默认费率                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 4.4 计算示例

| 场景 | 机构 | 计算模型 | 保证金 | 费率 | 保费 |
|------|------|----------|--------|------|------|
| 投标保函 | 银行A | 固定费率 | 80万 | 0.5% | 4000元 |
| 投标保函 | 担保B | 阶梯费率 | 80万 | 0.4% | 3200元 |
| 履约保函 | 银行A | 固定费率 | 50万 | 0.6% | 3000元 |
| 投标保函 | 保险C | 固定保费 | 80万 | 固定 | 500元 |

## 7. 信息加密

### 7.1 加密场景

| 阶段 | 内容 | 说明 |
|------|------|------|
| 开标前 | 敏感信息加密 | 投标人名称、保证金金额等 |
| 开标后 | 自动解密 | 开标时间到达自动触发 |

### 7.2 加密方案

```
┌─────────────────────────────────────────────────────────────────┐
│                    数字信封加密原理                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   发送方：                                                      │
│   1. 生成随机对称密钥 (SM4)                                     │
│   2. 用对称密钥加密保函文件                                     │
│   3. 用接收方公钥 (SM2) 加密对称密钥                           │
│   4. 合并 = 加密文件 + 加密密钥(数字信封)                    │
│                                                                  │
│   接收方：                                                      │
│   1. 用私钥 (SM2) 解密对称密钥                                 │
│   2. 用对称密钥 (SM4) 解密文件                                │
└─────────────────────────────────────────────────────────────────┘
```
