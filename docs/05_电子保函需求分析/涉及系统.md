# 涉及系统

本文档说明电子保函系统涉及的相关系统及其交互关系。

---

## 1. 系统架构总览

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           电子保函系统架构                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│    ┌─────────────────┐         ┌─────────────────┐                       │
│    │   上游交易系统    │         │   下游出函机构   │                       │
│    │  (交易平台/招标)  │◀──────▶│ (银行/保险/担保)  │                       │
│    └────────┬────────┘         └────────┬────────┘                       │
│             │                            │                                 │
│             └──────────┬─────────────────┘                                 │
│                        │                                                  │
│                        ▼                                                  │
│              ┌─────────────────────┐                                      │
│              │    电子保函平台      │                                      │
│              │   (核心系统)        │                                      │
│              └─────────────────────┘                                      │
│                        │                                                  │
│                        ▼                                                  │
│              ┌─────────────────────┐                                      │
│              │   支付通道/CA机构    │                                      │
│              │ (微信/支付宝/网银)   │                                      │
│              └─────────────────────┘                                      │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 2. 上游交易系统

### 2.1 系统类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **公共资源交易中心** | 政府采购、工程建设 | 各省市公共资源交易中心 |
| **招标平台** | 建设工程招标 | 招标代理机构系统 |
| **政府采购系统** | 政府招标采购 | 政府采购中心 |

### 2.2 主要厂商

| 厂商 | 系统 | 特点 |
|------|------|------|
| **国泰新点** | 公共资源交易平台 | 市场份额大 |
| **广联达** | 建设工程交易系统 | 建筑行业 |
| **筑龙** | 建设工程平台 | 建筑行业 |
| **金润** | 电子保函平台 | 保函专业 |

### 2.3 对接内容

```
上游交易系统 ──▶ 电子保函平台

推送内容：
├── 项目信息（项目编号、名称、招标编号等）
├── 标段信息（标段编号、保证金金额等）
├── 投标人信息（名称、代码等）
└── 接收内容：
    ├── 保函开具结果
    ├── 保函文件
    └── 验真结果
```

## 3. 下游出函机构

### 3.1 机构类型

| 类型 | 产品 | 特点 |
|------|------|------|
| **银行** | 银行保函(直开/分离式) | 信用度高，费率低 |
| **保险公司** | 保证保险保单 | 审批快，覆盖广 |
| **担保公司** | 担保保函 | 灵活，门槛低 |

### 3.2 对接方式

| 机构 | 对接方式 | 说明 |
|------|----------|------|
| 中国光大银行 | SDK/直连接口 | 报文规范完整 |
| 招商银行 | API接口 | 支持电子签章 |
| 建设银行 | API接口 | 支持分离式 |
| 太平洋保险 | API接口 | 保证保险 |
| 平安保险 | API接口 | 支持电子保单 |

### 3.3 对接内容

```
电子保函平台 ──▶ 出函机构

发送内容：
├── 投保申请（投标人、项目、保额等）
├── 保函开具请求
├── 退保/注销请求
└── 接收内容：
    ├── 保函开具结果
    ├── 保函文件（PDF/OFD）
    └── 电子签章信息
```

## 4. 外部服务

### 4.1 支付通道

| 类型 | 说明 | 用途 |
|------|------|------|
| 企业网银 | 银行直连 | 对公转账 |
| 微信支付 | 扫码支付 | 个人支付 |
| 支付宝 | 扫码支付 | 个人支付 |
| 连连支付 | 聚合支付 | 对公+个人 |

### 4.2 CA机构

| 服务 | 说明 |
|------|------|
| 数字证书 | 企业/个人身份认证 |
| 电子签章 | 文档数字签名 |
| 时间戳 | 可信时间证明 |

### 4.3 其他

| 服务 | 说明 |
|------|------|
| 短信通知 | 验证码、状态通知 |
| 邮件通知 | 重要文件发送 |
| 实名认证 | 企业/个人实名 |

## 6. 系统边界

### 6.1 平台负责

| 范围 | 说明 |
|------|------|
| 通道适配 | 多交易系统接入 |
| 业务处理 | 投保、支付、出函 |
| 机构对接 | 银行/保险/担保 |
| 用户管理 | 投标人/机构管理 |

### 6.2 外部负责

| 范围 | 说明 |
|------|------|
| 交易核心 | 项目发布、评标 |
| 资金流转 | 银行/支付系统 |
| 法律效力 | 保函法律效力 |
| 资质审批 | 金融机构资质 |
