# 电子保函系统需求分析-v0.1

---

## 一、项目概述

### 1.1 项目背景

电子保函系统是一个 B2B 平台，连接：
- **上游**：交易系统（公共资源交易中心、招标平台）
- **下游**：金融机构（银行、保险公司、担保公司）

### 1.2 痛点与机会

| 痛点 | 解决方案 |
|------|---------|
| 保证金占用企业资金 | 电子保函替代现金 |
| 线下办理不便 | 全程线上化 |
| 各省、市平台独立 | 统一平台接入 |
| 接口标准不一 | 适配器模式 |

### 1.3 目标用户

| 用户类型 | 核心诉求 |
|----------|---------|
| 投标人 | 快速出函、降低资金压力 |
| 招标人 | 保函验真、流程合规 |
| 金融机构 | 批量出函、风险控制 |
| 平台运营 | 统一管理、多渠道接入 |

## 三、核心内容摘要

### 3.1 业务类型

| 类型 | 说明 |
|------|------|
| 银行电子保函 | 银行直接开具 |
| 分离式银行保函 | 担保公司担保，银行出具 |
| 保证保险保单 | 保险公司出具 |
| 担保保函 | 担保公司出具 |

### 3.2 使用场景

- 投标保证金保函（≤80万）
- 履约保证金保函（合同价5-10%）
- 工程质量保证金保函（合同价3%）
- 农民工工资支付保函

### 3.3 核心流程

```
项目信息同步 → 投保申请 → 保费支付 → 保函开具 → 电子签章 → 保函验真
                                ↓
                         退保/注销/理赔
```

### 3.4 微服务架构（10个服务）

| 服务 | 领域 | 说明 |
|------|------|------|
| baohan-gateway | 基础设施 | API网关 |
| baohan-upmsp | 通用域 | 用户权限 |
| baohan-guarantee | 核心域 | 保函核心（含投保/订单/保函/退保） |
| baohan-institution | 通用域 | 出函机构管理 |
| baohan-channel | 通用域 | 交易通道适配 |
| baohan-payment | 支撑域 | 支付服务 |
| baohan-invoice | 支撑域 | 发票服务（独立） |
| baohan-notify | 支撑域 | 通知服务 |
| baohan-risk | 支撑域 | 风控服务 |
| baohan-file | 支撑域 | 文件服务（含电子签章） |

### 3.5 关键技术

| 技术点 | 方案 |
|--------|------|
| 分布式事务 | 可靠消息+本地消息表（去TCC） |
| 电子签章 | 自研OFD（测试）+ 第三方SDK（生产） |
| 接口安全 | SM3/MD5/SM2 可配置 |
| 加密 | 数字信封/平台密钥 可配置 |
