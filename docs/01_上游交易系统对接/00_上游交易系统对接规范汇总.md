## 二、常见对接方式

### 1. 对接模式分类

| 模式 | 说明 | 代表厂商 |
|------|------|----------|
| **API实时对接** | HTTP/HTTPS接口，实时交互 | 主流方式 |
| **SDK接入** | 提供开发工具包 | 国泰新点、筑龙 |
| **消息队列** | 异步消息推送 | 大型交易平台 |

| 协议 | 要求 |
|------|------|

| **JSON** | 主流数据格式 |
| **XML** | 部分老系统使用 |

### 3. 常见厂商/系统

## 三、数据交互方式

### 1. 请求/响应模式

#### 标准请求结构
```json
{
  "head": {
    "channel_code": "HN_TENDER_001",
    "timestamp": "2025-01-01T10:00:00Z",
    "sign": "SM2签名值",
    "msg_id": "UUID"
  },
  "body": {
    "business_data": {}
  }
}
```

#### 标准响应结构
```json
{
  "code": "0000",
  "message": "success",
  "msg_id": "UUID",
  "timestamp": "2025-01-01T10:00:00Z",
  "data": {},
  "sign": "SM2签名值"
}
```

### 2. 核心业务流程

```
┌──────────────────────────────────────────────────────────────────┐
│                        完整业务流程                               │
├──────────────────────────────────────────────────────────────────┤
│  1. 项目信息同步   (交易系统 → 平台)                            │
│         ↓                                                        │
│  2. 投保申请       (投标人 → 交易系统 → 平台 → 金融机构)        │
│         ↓                                                        │
│  3. 保函开具       (金融机构 → 平台 → 交易系统)                 │
│         ↓                                                        │
│  4. 保函验真       (交易系统 → 平台 → 金融机构)                 │
│         ↓                                                        │
│  5. 退保/注销      (交易系统 → 平台 → 金融机构)                 │
│         ↓                                                        │
│  6. 理赔处理       (金融机构 → 平台 → 交易系统)                  │
└──────────────────────────────────────────────────────────────────┘
```

## 五、加密/签名要求

### 1. 加密算法

| 用途 | 算法 | 说明 |
|------|------|------|
| 报文签名 | SM3 | 国密消息摘要 |
| 数据加密 | SM4 | 国密对称加密 |
| 数字信封 | SM2 | 国密非对称加密 |
| 传输加密 | SSL/TLS | HTTPS通道 |

### 2. 签名流程

```
1. 将报文字段按ASCII升序排序
2. 按"参数=参数值"模式，用"&"拼接
3. 使用appsecret对报文进行SM3摘要
4. 将摘要值放入sign字段
```

### 3. 密钥管理

| 密钥类型 | 用途 | 分配方式 |
|----------|------|----------|
| appkey | 请求标识 | 平台分配 |
| appsecret | 签名密钥 | 平台分配，保密 |
| SM4_KEY | 数据加密 | 平台分配 |
| SM2_PRIVATE | 解密数字信封 | 机构自有 |

## 六、特殊要求汇总

### 1. 安全要求

| 要求 | 说明 |
|------|------|
| 三级等保 | 需提供等保三级证明 |
| 安全扫描 | 源码安全扫描、漏洞扫描 |
| CA认证 | 使用CA数字证书 |
| 日志留存 | 不少于3年 |

### 2. 数据格式要求

| 要求 | 说明 |
|------|------|
| 编码 | UTF-8 |
| 日期格式 | yyyy-MM-dd HH:mm:ss |
| 金额 | 数字格式字符串，如"800000.00" |
| 加密 | 敏感信息必须加密传输 |

### 3. 接口差异

| 交易系统 | 特殊字段 | 备注 |
|----------|----------|------|
| 国泰新点 | 增加jbrname、jbrphone | 经办人信息 |
| 广联达 | 项目分类字段 | 建筑行业专用 |
| 福建标准 | 统一社会信用代码 | DB35/T 2055 |
| 安徽标准 | 保证金缴退分开 | DB3401/T 308 |
