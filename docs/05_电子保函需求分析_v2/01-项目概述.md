# 01-项目概述

## 1.1 项目背景

电子保函系统是一个 B2B 平台，连接：
- **上游**：交易系统（公共资源交易中心、招标平台、政府采购系统）
- **下游**：金融机构（银行、保险公司、担保公司）

### 1.1.1 行业痛点

| 痛点 | 解决方案 |
|------|---------|
| 保证金占用企业资金 | 电子保函替代现金 |
| 线下办理不便 | 全程线上化 |
| 各省、市平台独立 | 统一平台接入 |
| 接口标准不一 | 适配器模式 |

### 1.1.2 目标用户

| 用户类型 | 核心诉求 |
|----------|---------|
| 投标人 | 快速出函、降低资金压力 |
| 招标人 | 保函验真、流程合规 |
| 金融机构 | 批量出函、风险控制 |
| 平台运营 | 统一管理、多渠道接入 |

---

## 1.2 项目目标

### 1.2.1 建设目标

1. **统一平台**：一个平台对接多个交易系统
2. **多机构整合**：同时对接银行、保险、担保公司
3. **全程线上**：投保→支付→出函→验真全流程线上
4. **标准适配**：通过适配器支持不同接口标准

### 1.2.2 业务目标

| 目标 | 指标 |
|------|------|
| 出函时效 | 线上投保 1-5 分钟出函 |
| 覆盖率 | 支持 80% 以上主流交易系统 |
| 机构接入 | 支持 20+ 金融机构 |
| 系统可用性 | 99.9% |

---

## 1.3 项目范围

### 1.3.1 范围边界

**包含**：
- 投保申请管理
- 保费计算与支付
- 保函生成与签章
- 保函验真
- 退保/注销处理
- 机构管理
- 通道适配

**不包含**：
- 交易系统核心业务（项目发布、评标）
- 金融机构内部审批
- 资金划转（支付通道负责）

### 1.3.2 业务类型

| 类型 | 说明 |
|------|------|
| 银行电子保函 | 银行直接开具 |
| 分离式银行保函 | 担保公司担保，银行出具 |
| 保证保险保单 | 保险公司出具 |
| 担保保函 | 担保公司出具 |

### 1.3.3 使用场景

| 场景 | 金额范围 |
|------|----------|
| 投标保证金保函 | ≤80万 |
| 履约保证金保函 | 合同价5-10% |
| 工程质量保证金保函 | 合同价3% |
| 农民工工资支付保函 | 按项目约定 |
