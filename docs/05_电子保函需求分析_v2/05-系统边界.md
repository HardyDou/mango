# 05-系统边界

本文档明确电子保函系统涉及的各系统边界，清晰划分各系统职责。

---

## 5.1 边界划分原则

### 5.1.1 划分标准

| 原则 | 说明 |
|------|------|
| **业务归属** | 核心业务逻辑在电子保函平台 |
| **数据主权** | 核心数据存储在电子保函平台 |
| **控制权** | 流程由谁发起/控制 |
| **复用性** | 通用能力可复用外部服务 |

### 5.1.2 边界判断流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     边界判断流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   功能需求                                                     │
│       │                                                        │
│       ▼                                                        │
│   1. 谁发起？                                                 │
│      ├── 交易系统发起 → 上游接口                               │
│      ├── 投标人发起 → 内部处理                                 │
│      └── 机构发起 → 下游接口                                   │
│       │                                                        │
│       ▼                                                        │
│   2. 数据存储在谁那？                                         │
│      ├── 核心业务数据 → 电子保函平台                           │
│      ├── 用户数据 → 交易系统/平台各自存储                      │
│      └── 外部数据 → 调用外部接口                               │
│       │                                                        │
│       ▼                                                        │
│   3. 流程控制在谁那？                                         │
│      ├── 平台控制 → 内部流程                                   │
│      └── 外部控制 → 接口回调                                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 5.3 功能边界对照表

### 5.3.1 实名认证

| 功能 | 归属 | 说明 |
|------|------|------|
| 实名认证 | 交易系统或第三方 | 有两种模式 |

**模式A（交易系统认证）**：
```
交易系统完成实名认证 → 跳转电子保函平台（带认证标识）
适用场景：交易系统已有用户体系
```

**模式B（平台独立认证）**：
```
电子保函平台独立完成实名认证
适用场景：交易系统无用户体系
```

### 5.3.2 支付

| 功能 | 归属 | 说明 |
|------|------|------|
| 支付页面 | 电子保函平台 | 生成支付链接 |
| 支付跳转 | 支付通道 | 跳转第三方 |
| 支付结果回调 | 支付通道 → 电子保函平台 | 异步通知 |
| 资金划转 | 支付通道 | 银行内部处理 |

### 5.3.3 电子签章

| 功能 | 归属 | 说明 |
|------|------|------|
| 证书申请 | CA机构 | CA机构处理 |
| 文档签名 | 电子保函平台/第三方 | 我方负责 |
| 签名验证 | CA机构 | 验签服务 |

### 5.3.4 发票

| 功能 | 归属 | 说明 |
|------|------|------|
| 发票开具 | 出函机构 | 银行/保险/担保开具 |
| 发票申请 | 电子保函平台 | baohan-invoice 服务 |
| 发票展示 | 电子保函平台 | 平台展示/下载 |
| 发票寄送 | 出函机构 | 纸质发票寄送 |

## 5.5 接口边界

### 5.5.1 上游接口（交易系统 → 平台）

| 接口 | 方向 | 边界说明 |
|------|------|----------|
| 项目同步 | 交易系统→平台 | 交易系统推送，平台接收存储 |
| 投保申请 | 交易系统→平台 | 交易系统发起，平台处理 |
| 保函查询 | 交易系统→平台 | 交易系统查询，平台返回 |
| 保函验真 | 交易系统→平台 | 交易系统发起，平台验签 |
| 退保通知 | 交易系统→平台 | 交易系统通知，平台处理 |

### 5.5.2 下游接口（平台 → 出函机构）

| 接口 | 方向 | 边界说明 |
|------|------|----------|
| 投保推送 | 平台→机构 | 平台发起，机构处理 |
| 支付通知 | 平台→机构 | 平台通知，机构确认 |
| 保函回传 | 机构→平台 | 机构回传，平台存储 |

### 5.5.3 外部接口

| 接口 | 方向 | 说明 |
|------|------|------|
| 支付 | 平台→支付通道 | 发起支付 |
| 回调 | 支付通道→平台 | 支付结果 |
| 实名认证 | 平台→CA | 认证请求 |
| 签章 | 平台→CA | 签名请求 |
| 通知 | 平台→第三方 | 发送通知 |

## 5.6 边界争议点

| 序号 | 问题 | 当前建议 | 需确认 |
|------|------|---------|--------|
| 1 | 实名认证在哪个系统做？ | 交易系统 | 确认各交易系统是否支持 |
| 2 | 支付页面在哪个系统？ | 电子保函平台 | 确认用户体验 |
| 3 | 保函文件存在哪里？ | 电子保函平台 | 或机构？ |
| 4 | 额度查询谁提供？ | 出函机构 | 机构是否开放接口 |
