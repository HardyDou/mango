# DDD领域分析报告

本文档从系统分析师角度，结合电子保函业务专家视角，进行DDD领域分析。

---

## 一、领域划分总览

### 1.1 领域类型定义

| 领域类型 | 定义 | 原则 |
|----------|------|------|
| **核心域** | 业务核心价值所在 | 电子保函业务 |
| **通用域** | 可复用的通用能力 | 用户权限、机构、通道 |
| **支撑域** | 支持核心域的辅助能力 | 支付、发票、通知、风控、文件 |
| **基础设施** | 技术底层能力 | 网关、注册、配置 |

## 二、限界上下文（Bounded Context）

### 2.1 限界上下文定义

| 限界上下文 | 职责 | 核心实体 | 领域 |
|------------|------|----------|------|
| **保函上下文** | 投保、订单、保函、退保全生命周期管理 | 投保申请、订单、保函 | 核心域 |
| **通道上下文** | 上游交易系统接入、协议适配 | 通道配置、密钥 | 通用域 |
| **机构上下文** | 出函机构管理、额度管理 | 机构、额度 | 通用域 |
| **用户上下文** | 用户认证、授权、权限 | 用户、角色、权限 | 通用域 |
| **支付上下文** | 支付交易、回调处理 | 支付单、退款单 | 支撑域 |
| **发票上下文** | 发票申请、查询、下载 | 发票申请 | 支撑域 |
| **通知上下文** | 短信、邮件、站内通知 | 通知 | 支撑域 |
| **风控上下文** | 风控规则、黑名单、额度控制 | 规则、黑名单 | 支撑域 |
| **文件上下文** | 文件存储、电子签章 | 文件、签章 | 支撑域 |
