## 6.2 保函上下文场景（核心域）

### 6.2.1 项目信息同步

| 场景 | 说明 |
|------|------|
| **限界上下文** | 通道上下文 |
| **触发方** | 交易系统 |
| **触发条件** | 新项目发布/项目信息变更 |
| **处理内容** | 接收并解析项目信息、标段信息、保证金规则 |
| **后续动作** | 创建项目缓存、配置支付规则 |

**数据项**：

| 字段 | 说明 |
|------|------|
| projectno | 项目编号 |
| tenderno | 招标编号 |
| projectname | 项目名称 |
| bidbondamount | 保证金金额 |
| deadline | 投标截止时间 |

### 6.2.2 投保申请

| 场景 | 说明 |
|------|------|
| **限界上下文** | 保函上下文 |
| **触发方** | 投标人 |
| **触发条件** | 投标人选择保函、提交申请 |
| **处理内容** | 校验资格、计算保费、推送金融机构 |
| **后续动作** | 等待支付、等待出函 |

**数据项**：

| 字段 | 说明 |
|------|------|
| biddername | 投标人名称 |
| biddercode | 统一社会信用代码 |
| bzjamount | 保证金金额 |
| biaoduanno | 标段编号 |

### 6.2.3 保费支付

| 场景 | 说明 |
|------|------|
| **限界上下文** | 支付上下文 |
| **触发方** | 投标人 |
| **触发条件** | 投保申请成功后发起支付 |
| **处理内容** | 发起支付、接收回调、更新订单状态 |
| **后续动作** | 支付成功→触发出函 |

**支付方式**：

| 方式 | 主体 | 说明 |
|------|------|------|
| 企业网银 | 企业 | 对公转账 |
| 微信支付 | 个人 | 扫码 |
| 支付宝 | 个人 | 扫码 |

### 6.2.4 保函开具

| 场景 | 说明 |
|------|------|
| **限界上下文** | 保函上下文 + 文件上下文 |
| **触发方** | 支付成功 / 金融机构主动 |
| **触发条件** | 保费支付成功 + 风控通过 |
| **处理内容** | 生成保函文件、电子签章、返回结果 |
| **后续动作** | 推送保函给交易系统/投标人 |

**出函类型**：

| 类型 | 说明 |
|------|------|
| 银行直开 | 银行直接开具 |
| 分离式 | 担保公司担保，银行出具 |
| 保证保险 | 保险公司出具保单 |
| 担保保函 | 担保公司直接出具 |

### 6.2.5 保函验真

| 场景 | 说明 |
|------|------|
| **限界上下文** | 保函上下文 |
| **触发方** | 交易系统/招标人 |
| **触发条件** | 开标前/开标时验证保函真伪 |
| **处理内容** | 验证签章、校验保函状态 |
| **后续动作** | 返回验真结果 |

#### 6.2.5.1 批量验真

| 场景 | 说明 |
|------|------|
| **触发方** | 交易系统/招标人 |
| **触发条件** | 开标时批量验证多家投标人保函 |
| **处理内容** | 批量验证签章、校验状态 |
| **后续动作** | 返回批量验真结果报告 |

**批量验真流程**：
```
交易系统 ──▶ 平台

1. 提交批量验真请求
   ├── 标段编号
   ├── 投标人列表（批量）
   └── 验真时间

2. 平台处理
   ├── 逐条验证每份保函
   ├── 汇总验真结果
   └── 生成验真报告

3. 返回结果
   ├── 成功列表
   ├── 失败列表
   └── 异常列表
```

## 6.11 批量场景

### 6.11.1 批量出函

| 场景 | 说明 |
|------|------|
| **限界上下文** | 保函上下文 |
| **触发方** | 平台/运营人员 |
| **触发条件** | 多家机构同时响应出函 |
| **处理内容** | 收集多份保函、汇总结果 |
| **后续动作** | 返回批量出函结果 |

**批量出函流程**：
```
场景：同一标段多家投标人同时投保，多家机构响应

1. 批量出函请求
   ├── 标段编号
   └── 投标人列表

2. 平台处理
   ├── 并行向多家机构获取保函
   ├── 汇总保函结果
   └── 生成出函汇总

3. 返回结果
   ├── 出函成功列表
   ├── 出函失败列表
   └── 处理中列表
```
