## 7.2 核心业务流程详解

### 7.2.1 投保出函主流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    投保出函主流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   投标人 ──▶ 交易系统 ──▶ 平台 ──▶ 金融机构                   │
│                                                                  │
│   1. 投标人发起投保                                             │
│      ├── 选择保函类型                                          │
│      ├── 选择出函机构                                          │
│      └── 确认投保信息                                          │
│                                                                  │
│   2. 平台处理                                                   │
│      ├── 校验投标人资格                                        │
│      ├── 校验项目信息                                          │
│      ├── 计算保费                                              │
│      ├── 冻结额度（如需要）                                    │
│      └── 推送投保请求到金融机构                                │
│                                                                  │
│   3. 金融机构响应                                              │
│      ├── 审核投保信息                                          │
│      ├── 返回审核结果                                          │
│      └── 确认出函                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 7.3 领域模型

### 7.3.1 聚合根

#### 7.3.1.1 投保申请聚合根（Application）

```
┌─────────────────────────────────────────────────────────────────┐
│                    投保申请聚合根                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Application (聚合根)                                         │
│   │                                                              │
│   ├── applicationNo: String (唯一标识)                        │
│   ├── bidder: BidderInfo (投标人信息 - 值对象)               │
│   ├── project: ProjectInfo (项目信息 - 值对象)                │
│   ├── guaranteeType: Enum (保函类型)                          │
│   ├── institutionId: Long (出函机构ID)                         │
│   ├── status: Enum (状态)                                     │
│   │   ├── PENDING (待支付)                                    │
│   │   ├── PAID (已支付)                                        │
│   │   ├── ISSUED (已出函)                                      │
│   │   └── CANCELLED (已取消)                                   │
│   └── applyTime: LocalDateTime (申请时间)                     │
│                                                                  │
│   业务规则：                                                    │
│   1. 申请创建后 → 自动创建订单                                │
│   2. 支付成功 → 触发出函                                       │
│   3. 保函开具成功 → 状态变更为已开具                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 7.3.1.2 订单聚合根（Order）

```
┌─────────────────────────────────────────────────────────────────┐
│                      订单聚合根                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Order (聚合根)                                                │
│   │                                                              │
│   ├── orderNo: String (唯一标识)                              │
│   ├── applicationNo: String (关联投保申请)                     │
│   ├── amount: BigDecimal (保费金额)                            │
│   ├── paymentMethod: Enum (支付方式)                           │
│   │   ├── WECHAT (微信)                                       │
│   │   ├── ALIPAY (支付宝)                                      │
│   │   └── BANK (网银)                                          │
│   ├── paymentStatus: Enum (支付状态)                           │
│   │   ├── PENDING (待支付)                                     │
│   │   ├── SUCCESS (成功)                                       │
│   │   ├── FAILED (失败)                                        │
│   │   └── REFUNDED (已退款)                                    │
│   └── payTime: LocalDateTime (支付时间)                        │
│                                                                  │
│   业务规则：                                                    │
│   1. 创建订单时计算保费                                        │
│   2. 支付成功回调更新状态                                      │
│   3. 超时未支付自动取消                                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 7.3.1.3 保函聚合根（Guarantee）

```
┌─────────────────────────────────────────────────────────────────┐
│                      保函聚合根                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Guarantee (聚合根)                                           │
│   │                                                              │
│   ├── guaranteeNo: String (唯一标识)                          │
│   ├── applicationNo: String (关联投保申请)                    │
│   ├── orderNo: String (关联订单)                              │
│   ├── guaranteeAmount: BigDecimal (保函金额)                   │
│   ├── period: GuaranteePeriod (有效期 - 值对象)               │
│   │   ├── startDate: LocalDate                                 │
│   │   └── endDate: LocalDate                                   │
│   ├── status: Enum (状态)                                     │
│   │   ├── PENDING (待开具)                                     │
│   │   ├── ISSUED (已开具)                                      │
│   │   ├── CANCELLED (已注销)                                   │
│   │   └── EXPIRED (已到期)                                     │
│   ├── fileInfo: FileInfo (文件信息 - 值对象)                  │
│   │   ├── filePath: String                                     │
│   │   ├── fileFormat: String (PDF/OFD)                        │
│   │   └── signInfo: String                                     │
│   └── issueDate: LocalDateTime (开具时间)                      │
│                                                                  │
│   业务规则：                                                    │
│   1. 保函开具后不可修改                                        │
│   2. 到期自动注销                                              │
│   3. 退保需审核                                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 7.4 业务规则

### 7.4.1 保费计算规则

| 规则类型 | 说明 |
|----------|------|
| 按比例 | 保证金金额 × 费率 |
| 按金额 | 固定金额 |
| 按梯度 | 不同金额区间不同费率 |

### 7.4.2 支付规则

| 规则 | 说明 |
|------|------|
| 支付时效 | 投保成功后XX分钟内支付 |
| 支付方式 | 按平台配置 |
| 个人支付 | 按平台配置允许/禁止 |

### 7.4.3 保函有效期规则

| 场景 | 有效期 |
|------|--------|
| 投标保证金保函 | 投标截止日后30天 |
| 履约保证金保函 | 合同工期+质保期 |
| 工程质量保证金保函 | 质保期满后2年 |
| 农民工工资支付保函 | 工期结束后1年 |

## 7.6 异常处理

### 7.6.1 支付异常

| 场景 | 限界上下文 | 处理 |
|------|------------|------|
| 支付超时 | 支付上下文 | 自动取消订单 |
| 支付失败 | 支付上下文 | 通知投标人重试 |
| 重复支付 | 支付上下文 | 自动退款 |

### 7.6.2 出函异常

| 场景 | 限界上下文 | 处理 |
|------|------------|------|
| 出函超时 | 保函上下文 | 自动重试/人工介入 |
| 出函失败 | 保函上下文 | 退款+通知 |
| 金融机构异常 | 机构上下文 | 切换通道/人工处理 |

### 7.6.3 分布式事务处理

```
方案：可靠消息 + 本地消息表

适用场景：
- 电子保函无账户余额体系
- 实时支付（微信/支付宝/网银）
- TCC不适用

流程：
1. 投保申请 → 本地事务写入业务表 + 消息表
2. 发送消息到MQ
3. 消费消息处理下游业务
4. 定时扫描消息表，失败则补偿重试
```
