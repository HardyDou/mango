## 7.2 业务流程详解

### 7.2.1 项目信息同步

```
交易系统 ──▶ 平台

1. 推送项目基本信息
   ├── 项目编号、名称
   ├── 招标人信息
   └── 公告时间

2. 推送标段信息
   ├── 标段编号、名称
   ├── 保证金金额
   ├── 投标截止时间
   └── 保函格式要求

3. 平台处理
   ├── 解析并保存项目
   ├── 配置支付规则
   └── 缓存项目信息
```

### 7.2.2 投保申请

```
投标人 ──▶ 交易系统 ──▶ 平台 ──▶ 金融机构

1. 投标人发起投保
   ├── 选择保函类型
   ├── 选择出函机构
   └── 确认投保信息

2. 平台处理
   ├── 校验投标人资格
   ├── 校验项目信息
   ├── 计算保费
   ├── 冻结额度（如需要）
   └── 推送投保请求到金融机构

3. 金融机构响应
   ├── 审核投保信息
   ├── 返回审核结果
   └── 确认出函
```

### 7.2.3 分离式银行保函流程

```
概述：担保公司提供担保，银行出具保函

流程：
┌─────────────────────────────────────────────────────────────────┐
│                  分离式银行保函流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   投标人 ──▶ 担保公司 ──▶ 银行 ──▶ 平台 ──▶ 交易系统          │
│                                                                  │
│   1. 担保公司审核                                               │
│      ├── 审核投标人资质                                         │
│      ├── 审批担保额度                                           │
│      └── 出具担保函                                             │
│                                                                  │
│   2. 银行审核出函                                               │
│      ├── 审核担保函有效性                                       │
│      ├── 审核投标人资质                                         │
│      └── 开具银行保函                                           │
│                                                                  │
│   3. 平台处理                                                   │
│      ├── 接收担保函                                             │
│      ├── 接收银行保函                                           │
│      ├── 生成完整保函文件                                       │
│      └── 电子签章                                               │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2.4 保费支付

```
1. 支付方式
   ├── 企业网银（对公转账）
   ├── 微信支付（个人扫码）
   └── 支付宝（个人扫码）

2. 支付规则
   ├── 每个上游平台独立配置
   ├── 是否允许个人支付
   └── 可用支付方式

3. 支付流程
   ├── 发起支付请求
   ├── 跳转支付通道
   ├── 支付成功回调
   └── 触发后续流程
```

### 7.2.5 保函开具

```
1. 出函类型
   ├── 银行直开
   ├── 分离式（担保公司+银行）
   ├── 保证保险（保险公司）
   └── 担保保函（担保公司）

2. 出函流程
   ├── 接收出函请求
   ├── 生成保函文件
   ├── 电子签章
   └── 返回保函文件
```

### 7.2.6 电子签章

```
1. 签章流程
   ├── 生成OFD/PDF文件
   ├── 获取时间戳
   ├── SM2签名
   └── 签章验签

2. 签章方案
   ├── 测试环境：自研OFD
   └── 生产环境：第三方SDK
```

### 7.2.7 保函验真

```
1. 验真内容
   ├── 签章有效性验证
   ├── 保函状态查询
   ├── 关键信息核对

2. 验真方式
   ├── 单条验真
   └── 批量验真
```

### 7.2.8 退保注销

```
1. 退保类型
   ├── 自愿退保（投标人主动申请）
   ├── 过期退保（保函到期自动注销）
   └── 强制退保（违规强制注销）

2. 退保流程
   ├── 提交退保申请
   ├── 审核退保
   ├── 注销保函
   ├── 释放额度
   └── 退费（如适用）
```

## 7.3 业务规则

### 7.3.1 保费计算规则

| 规则类型 | 说明 |
|----------|------|
| 按比例 | 保证金金额 × 费率 |
| 按金额 | 固定金额 |
| 按梯度 | 不同金额区间不同费率 |

### 7.3.2 支付规则

| 规则 | 说明 |
|------|------|
| 支付时效 | 投保成功后XX分钟内支付 |
| 支付方式 | 按平台配置 |
| 个人支付 | 按平台配置允许/禁止 |

### 7.3.3 保函有效期规则

| 场景 | 有效期 |
|------|--------|
| 投标保证金保函 | 投标截止日后30天 |
| 履约保证金保函 | 合同工期+质保期 |
| 工程质量保证金保函 | 质保期满后2年 |
| 农民工工资支付保函 | 工期结束后1年 |

## 7.5 异常处理

### 7.5.1 支付异常

| 场景 | 处理 |
|------|------|
| 支付超时 | 自动取消订单 |
| 支付失败 | 通知投标人重试 |
| 重复支付 | 自动退款 |

### 7.5.2 出函异常

| 场景 | 处理 |
|------|------|
| 出函超时 | 自动重试/人工介入 |
| 出函失败 | 退款+通知 |
| 金融机构异常 | 切换通道/人工处理 |

### 7.5.3 分布式事务处理

```
方案：可靠消息 + 本地消息表

适用场景：
- 电子保函无账户余额体系
- 实时支付（微信/支付宝/网银）
- TCC不适用

流程：
1. 投保申请 → 本地事务写入业务表 + 消息表
2. 发送消息到MQ
3. 消费消息处理下游业务
4. 定时扫描消息表，失败则补偿重试
```
