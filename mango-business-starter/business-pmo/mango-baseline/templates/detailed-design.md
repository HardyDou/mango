## 20. AI 输出自检

自检结果只能填写 `PASS`、`FAIL`、`BLOCKED` 或 `EXCEPTION`。任一 `FAIL/BLOCKED` 时，最终动作只能是 `STOP/ASK`；`EXCEPTION` 必须引用用户确认来源；全部 `PASS` 时，最终动作才可为 `NEXT`。

| 检查项 | 结果 | 说明 |
|---|---|---|
| PRD 关键对象、流程、规则、页面、字典、验收项是否都有覆盖矩阵记录 |  |  |
| 每条 PRD 业务规则是否映射到状态、数据、页面反馈或异常处理 |  |  |
| 每个关键对象关系是否有结构化表和图 |  |  |
| 每个有生命周期的对象是否有状态定义表、状态流转表和状态图 |  |  |
| 数据流是否写清来源、去向、变更对象、事务边界、失败处理和用户可见结果 |  |  |
| 关键技术是否覆盖本次相关的接口、数据、权限、租户、文件、事务、并发、兼容和验证方式 |  |  |
| 每个 PRD 验收项是否映射到至少一个设计交付物 |  |  |
| 是否不存在无法追溯到 PRD、用户确认或现有系统约束的设计结论 |  |  |
