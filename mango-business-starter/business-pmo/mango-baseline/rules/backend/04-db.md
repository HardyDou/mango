# 数据库规范

## 1. 表规则

- 表名使用小写下划线。
- 表名必须体现模块边界。
- 不使用外键作为默认方案。
- 一个表只服务一个领域。

## 2. 字段规则

- 字段名使用小写下划线。
- 字段类型按真实业务选择。
- 公共审计字段按项目基线统一。
- 多租户表按需要增加 `tenant_id`。

## 3. 索引规则

- 索引只为明确查询场景服务。
- 不创建无效索引和重复索引。
- 联合索引按查询条件设计。
- 不在低区分度字段上滥建索引。

## 4. SQL 规则

- 必须使用参数化查询。
- 禁止字符串拼接 SQL。
- Mapper 只访问本域表。
- 跨域数据通过 API 获取。

## 5. 迁移规则

- 所有 DDL 变更必须使用 Flyway migration。
- 只新增 migration，不修改历史 migration。
- migration 按模块隔离。
- migration 命名使用 `V{version}__{description}.sql`。

## 6. 禁止事项

- 直接改生产库
- 在 Mapper 中跨域 join
- 表结构变更没有 migration
