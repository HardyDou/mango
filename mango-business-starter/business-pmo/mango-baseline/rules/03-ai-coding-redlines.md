# AI 编码红线

## 1. 适用范围

- 适用于 Mango 框架开发者。
- 适用于基于 Mango 的业务项目开发者。
- 适用于 Agent、人工开发者和人机协同开发。
- 适用于后端、前端、数据库、测试、配置、脚本和文档交付。

## 2. 禁止事项

- 禁止写伪代码。
- 禁止写 TODO、FIXME、未来再优化、临时方案。
- 禁止写 mock、fake、dummy、hardcode 数据作为最终交付。
- 禁止写空实现。
- 禁止用 `UnsupportedOperationException`、空方法、固定返回值或接口壳子冒充已实现。
- 禁止 catch 后吞异常。
- 禁止返回固定成功。
- 禁止绕过权限、租户、数据权限。
- 禁止未接数据库却伪装完成。
- 禁止未接真实接口却伪装完成。
- 禁止把应由数据库、配置、字典、枚举中心或真实接口提供的数据写死在 Controller、Service、前端页面或测试替身中。
- 禁止以“能编译、能启动、能打开页面”替代真实业务链路、真实数据、真实权限和真实交互验收。
- 禁止交付明显低质量页面，包括布局混乱、字段无业务分组、联动字段分散、按钮区域不统一、状态表达不清、加载/空/错误态缺失。

## 3. 必须满足

- 代码必须可编译。
- 单测必须可运行。
- 配置必须可启动。
- 接口必须有真实实现。
- 异常必须有明确处理。
- 日志必须有上下文。
- 数据必须来自数据库、配置或真实接口。
- 页面必须完成真实交互、真实数据回显和 UI 布局走查。
- 字典、菜单、权限、租户默认数据、状态映射、业务配置等持久化或可运营数据必须有明确数据来源和初始化方式。
- 不确定时必须先列出阻塞点。

## 4. 交付要求

- 交付报告必须说明实际验证命令和结果。
- 未连接真实数据库、真实接口或真实权限链路时，必须标记为阻塞或例外，禁止声明完成。
- 未完成真实 UI 走查、真实接口联调或真实数据来源确认时，必须标记为未验证，禁止声明页面或接口可用。
- 发现红线冲突时，必须先修复冲突；无法修复时，必须在交付台账中标记 `EXCEPTION` 并写明用户确认依据。
