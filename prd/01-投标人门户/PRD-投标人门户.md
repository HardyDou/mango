# PRD - 投标人门户系统

## 1. 产品概述

### 1.1 产品背景

投标人门户是电子保函系统的面向投标人的Web端门户，用于投标人在线完成投保申请、保函查询、订单管理等操作。

### 1.2 产品定位

面向投标人的一站式电子保函服务平台，实现：
- 线上投保 → 支付 → 出函全流程
- 多交易系统项目统一入口
- 保函全生命周期管理

### 1.3 目标用户

| 角色 | 说明 |
|------|------|
| 投标人 | 参与投标的企业 |
| 经办人 | 投标人方办理保函人员 |

## 3. 功能详细说明

### 3.1 首页（工作台）

#### 3.1.1 功能描述

用户登录后的主页面，展示关键信息和快捷入口。

#### 3.1.2 页面内容

| 区块 | 内容 |
|------|------|
| 顶部导航 | Logo、搜索框、消息图标、用户头像下拉 |
| 快捷入口 | 投保申请、我的保函、订单管理、发票管理 |
| 待办事项 | 待支付订单、待确认退保 |

- 点击消息图标 → 弹出消息列表抽屉

### 3.4 投保申请

#### 3.4.1 功能描述

核心业务流程，投标人提交投保申请。

#### 3.4.2 投保流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         投保申请流程                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   步骤1: 确认项目      步骤2: 填写信息      步骤3: 选择机构            │
│      ✅                   ➡                   ➡                        │
│                                                                         │
│   步骤6: 完成           步骤5: 支付          步骤4: 确认投保            │
│      ⬅                   ⬅                   ⬅                        │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 3.4.3 步骤详情

**步骤1：确认项目**
- 显示选择的项目信息
- 确认后进入下一步

**步骤2：填写投保信息**

| 表单项 | 说明 | 必填 |
|--------|------|------|
| 投标人名称 | 企业全称 | 是 |
| 统一社会信用代码 | 18位代码 | 是 |
| 联系人 | 经办人姓名 | 是 |
| 联系电话 | 手机号 | 是 |
| 邮箱 | 接收通知 | 否 |

**步骤3：选择机构**

| 机构 | 保费费率 | 预计保费 | 出函时效 |
|------|----------|----------|----------|
| XX银行 | 0.5% | ¥4000 | 2分钟 |
| XX担保 | 0.6% | ¥4800 | 5分钟 |
| XX保险 | 0.4% | ¥3200 | 3分钟 |

- 支持单选/多选（根据平台配置）
- 显示预计保费和出函时效

**步骤4：确认投保**

- 展示投保信息汇总
- 勾选投保须知
- 提交申请

**步骤5：支付**

| 支付方式 | 说明 |
|----------|------|
| 微信支付 | 扫码支付 |
| 支付宝 | 扫码支付 |
| 企业网银 | 对公转账 |

- 跳转支付页面
- 支持支付结果查询

**步骤6：完成**

- 投保成功提示
- 跳转到保函详情
