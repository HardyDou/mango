## 一、核心概念澄清

### 1.1 可入围机构（仅三类）

| 机构类型 | 资质要求 | 说明 |
|----------|----------|------|
| **商业银行** | 金融许可证 | 银行保函，认可度最高 |
| **融资担保公司** | 融资担保业务经营许可证 | 市场占比70%以上，行业主流 |
| **财产保险公司** | 保险业务许可证 | 保证保险（投标保证保险） |

### 1.2 入围规则

- **一地一准入**：省内/市单独特审核
- 资质审核：牌照、注册资本、评级、合规记录
- 额度备案：担保能力需在交易中心登记
- 技术对接：必须通过接口联调

### 1.3 技术服务商定位

- **不具备入围资格**
- 为金融机构提供**出函系统**
- 由金融机构以自身名义入围对接

### 1.4 出函方式（按出函主体分类）

| 方式 | 说明 |
|------|------|
| **银行自营保函** | 银行直接审核、直接出函 |
| **银行分离式保函** | 银行出函 + 融资担保公司操盘 |
| **融资担保公司保函** | 持牌融资担保公司直接出具，可直接入围交易中心 |
| **保证保险** | 财产保险公司出具投标保证保险单 |

### 1.5 分离式保函专项说明

**分离式保函 = 银行出函 + 融资担保公司负责业务全流程**

核心逻辑：
1. 银行给予**融资担保公司**专项授信额度
2. 担保公司推荐客户、审核风险
3. 银行使用担保公司额度，以**银行名义**出具保函
4. 保函效力 = **标准银行保函**
5. 投标人仅对接担保公司，无需对接银行

**重要结论**：非融资性担保公司不能出电子保函

## 三、支付模式

### 3.1 支付模式

| 模式 | 说明 |
|------|------|
| **交易中心代收** | 90%地区采用 |
| **金融机构直收** | 机构直接收取保费 |
| **线下对公转账** | 线下转账 |

### 3.2 保费计价模式

| 模式 | 说明 |
|------|------|
| **按保额比例计费** | 公式：保函金额 × 约定费率，投标保函费率：0.2%～1.5% |
| **固定金额计费** | 一口价：300/500/800/1000元 |
| **阶梯比例计费** | 金额越大，费率越低 |
| **按时间计费** | 公式：保额 × 年化费率 × 期限 |

## 四、全流程数据流

### 阶段1：项目与准入准备

```
1. 招标人发布项目，中心备案
2. 金融机构入围、额度备案、接口联调
```

### 阶段2：投标人申请

```
1. 投标人选择项目 → 电子保函 → 选择金融机构
2. 数据流1：中心 → 金融机构（申请推送）
```

### 阶段3：风控审核

```
1. 机构自动审核：工商、失信、司法、黑名单、额度
2. 数据流2：金融机构 → 中心（审核结果）
```

### 阶段4：支付

```
1. 投标人支付保费
2. 数据流3：中心 → 金融机构（支付结果）
```

### 阶段5：出函（密文/明文分支）

```
投标保函（密文函）：
  - 接收中心加密的标段/招标人信息
  - 生成密文保函（仅4个核心字段加密）
  - 签章、加密、组装报文

其他保函（明文函）：
  - 直接生成明文保函

数据流4：金融机构 → 中心（出函结果）
```

### 阶段6：生效与投标

```
1. 中心验签、核验
2. 投标保函：保持密文状态，标记生效
3. 其他保函：直接明文生效
4. 数据流5：中心 → 金融机构（生效通知）
```

### 阶段7：开标与终结

```
1. 项目开标
2. 投标保函：
   - 中心调用还原保函接口，推送解密Key
   - 系统自动解密 → 生成明文保函
   - 明文归档
3. 中心同步开标结果
4. 数据流6：中心 → 金融机构（开标结果）
5. 额度释放、保函失效、归档
```

## 五、保函状态

### 5.1 投保状态

| 状态 | 说明 |
|------|------|
| SUBMITTED | 已提交 |
| REVIEWING | 审核中 |
| APPROVED | 审核通过 |
| REJECTED | 审核拒绝 |

### 5.2 支付状态

| 状态 | 说明 |
|------|------|
| PENDING | 待支付 |
| PAID | 已支付 |

### 5.3 出函状态

| 状态 | 说明 |
|------|------|
| PENDING | 待出函 |
| ISSUING | 出函中 |
| ISSUED | 已出函 |
| FAILED | 出函失败 |

### 5.4 保函生效状态

| 状态 | 说明 |
|------|------|
| ACTIVE | 生效中 |
| VERIFIED | 验真通过 |
| EXPIRED | 已到期 |
| CANCELLED | 已注销 |

## 六、异常流程

### 6.1 退保流程

```
投标人申请 → 中心推送 → 机构审核 → 退费/注销 → 回执同步
```

### 6.2 理赔流程

```
招标人索赔 → 中心核实 → 机构赔付 → 资金划转 → 追偿
```

### 6.3 废标/流标/项目撤销

```
中心推送状态 → 机构自动释放额度 → 保函失效
```

## 七、授信与额度体系

### 7.1 授信模式

| 模式 | 说明 |
|------|------|
| **银行授信模式（主流）** | 银行给予担保公司/保险机构总额度，中心备案，逐笔占用/释放 |
| **资金监管模式** | 机构存入保证金到监管户，按冻结额度出函 |
| **主体信用模式** | 大型机构凭自身信用获得中心认可额度 |

### 7.2 额度生命周期

```
总额度 → 可用额度 → 冻结/占用 → 出函生效 → 到期/开标 → 释放额度
```

---

## 八、一句话总结

**项目发布 → 机构入围 → 投标人申请 → 中心推数据 → 机构风控 → 支付 → 加密出函（密文/明文）→ 中心生效 → 投标 → 开标解密 → 额度释放 → 业务终结**
