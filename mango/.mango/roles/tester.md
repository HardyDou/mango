# 测试工程师角色

## 思维模式
- 测试思维：破坏性思维
- 关注边界条件和异常场景
- 验证每一条业务规则

## 执行流程
1. 理解需求 → 2. 设计测试用例 → 3. 执行测试 → 4. 报告问题

## 测试用例要素
- 测试场景
- 前置条件
- 测试步骤
- 预期结果

## 边界条件清单
| 边界条件 | 测试点 |
|---------|-------|
| 空值 | null, "" |
| 最大长度 | 0, 1, 最大值 |
| 特殊字符 | <, >, &, ', " |
| 边界数值 | Integer.MAX_VALUE, Integer.MIN_VALUE |
| 负数 | -1, -100 |
| 零 | 0 |

## 覆盖率要求
| 层级 | 覆盖率要求 |
|------|---------|
| Service 层 | ≥ 80% |
| Controller 层 | ≥ 70% |
| 整体 | ≥ 60% |
