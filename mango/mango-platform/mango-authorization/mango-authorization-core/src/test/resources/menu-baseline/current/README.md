# Current Menu Baseline

This directory records the menu data baseline before migrating default menus from Flyway SQL to resource manifests.

## Source

- Source migrations: `mango-authorization-core/src/main/resources/db/migration/authorization/V1__init_authorization.sql` through `V106__button_display_rule.sql`
- Replay database: `mango_tmp_issue186_menu_baseline_20260619`
- Excluded migration: `V107__restore_workflow_manage_menu.sql`

`V107__restore_workflow_manage_menu.sql` is excluded because it was a local untracked fix for the workflow orphan menu and violates the new rule that default menu data must not be maintained by Flyway SQL.

## Files

- `authorization-menu.tsv`: stable fields from `authorization_menu`
- `frontend-menu-runtime-config.tsv`: stable fields from `frontend_menu_runtime_config`
- `authorization-menu-package-item.tsv`: stable package-to-menu bindings
- `authorization-role-menu.tsv`: stable role-to-menu bindings
- `orphan-menus.tsv`: orphan menu records after resource manifest migration; data rows must stay empty

Timestamp and audit actor fields are intentionally excluded because they are generated by migration execution time and are not stable behavioral baseline fields.

## Orphan Menu Gate

The pre-migration Flyway replay exposed two orphan workflow menus:

- `260401 / workflow:template / 流程模板` has missing parent `2604`
- `260402 / workflow:definition / 流程定义` has missing parent `2604`

The migrated baseline keeps `orphan-menus.tsv` header-only. `workflow:manage` and its children are now declared through the workflow resource manifest, not by adding menu data back into Flyway SQL.
