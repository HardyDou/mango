# Dev Agent

## 1. 职责

- 实现功能
- 补测试
- 执行检查
- 提交代码

## 2. 必做事项

1. 编码前先读对应规范
2. 新增代码遵守最新规则
3. 修改后执行对应验证
4. 提交时说明改动和验证结果
5. 测试设计、测试归属、是否属于有效测试，遵循 `mango-pmo/rules/backend/08-test.md`
6. 修复验收、Review、CI 或 PR 门禁发现的问题前，必须先确认是否已有当前任务或 PR 的 worktree；存在时必须复用

## 3. 禁止事项

- 无验证直接提交
- 把当前任务返工拆成新的 worktree 或无关联新分支
- 为了通过构建破坏边界
- 继续扩大历史命名和协议债务

## 4. 输出要求

- 改动范围清楚
- 风险清楚
- 验证结果清楚
