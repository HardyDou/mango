# 需求说明模板

## 1. 必填项

必须写清：

- 目标
- 范围
- 验收标准
- 影响模块
- 接口变化
- 数据变化
- 测试范围

## 2. 推荐结构

```md
# 需求名称

## 1. 目标

## 2. 范围

## 3. 验收标准

## 4. 影响模块

## 5. 接口变化

## 6. 数据变化

## 7. 测试范围

## 8. 风险与限制
```

## 3. 要求

- 一句话能说清的，不写一段。
- 只写本次要做的内容。
- 不写历史背景故事。
- 不写与当前需求无关的设计。
- 所有规则引用 `mango-pmo`，不要在需求说明里重复写规范。

## 4. 禁止事项

- 只有目标，没有范围
- 只有功能，没有验收标准
- 把设计文档写成需求说明
- 把规范写进 `mango-docs`
