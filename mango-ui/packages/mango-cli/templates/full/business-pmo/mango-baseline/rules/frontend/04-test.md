# 前端测试规范

## 1. 基本要求

- 有前端改动，必须有对应验证。
- 新增交互，必须覆盖主要流程。
- 修复缺陷，必须补回归验证。

## 2. 测试层次

- 单元测试：工具函数、组合式函数、独立逻辑。
- 组件测试：组件渲染、输入输出、事件。
- E2E 测试：页面主流程、路由、接口联动。
- 是否需要视觉截图，按改动影响面决定。

## 3. 测试范围

至少覆盖：

- 正常场景
- 空状态
- 错误场景
- 关键交互
- 权限或路由限制

## 4. 命名规则

- 测试文件使用 `*.spec.ts` 或 `*.test.ts`。
- 用例名称直接描述场景和结果。

## 5. 提交前检查

至少执行与改动范围对应的检查：

- `pnpm test`
- `pnpm build`
- `pnpm playwright test`

## 6. 禁止事项

- 页面改动没有任何验证
- 只测 happy path
- 发现回归后不补测试
