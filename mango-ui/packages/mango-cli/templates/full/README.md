## Frontend

The frontend consumes Mango npm packages. It does not depend on Mango workspace source paths.

```bash
npm --prefix frontend install --registry={{npmRegistry}}
npm --prefix frontend run build
VITE_ADMIN_PROXY_PATH=http://127.0.0.1:5555 npm --prefix frontend run dev
```

Use `npm run dev` for backend integration. `VITE_ADMIN_PROXY_PATH` must point to the running backend port; if the backend is started with a non-default `MANGO_BACKEND_PORT`, update the proxy target to the same port before starting the frontend. `npm run preview` serves built static assets and does not proxy `/api`.

## Backend

The backend consumes `io.mango:mango-admin-starter`.

Create the local database before first startup:

```sql
CREATE DATABASE IF NOT EXISTS `{{projectKebab}}` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
```

```bash
scripts/backend-dev.sh
curl http://127.0.0.1:5555/actuator/health
```

`scripts/backend-dev.sh` installs local backend modules before starting the app. Use this script after `mango module add`, otherwise Maven may try to resolve local business starters from the remote repository.

Default local account:

```text
username: admin
password: admin123
```

Maven credentials must stay in user-level Maven `settings.xml`. NPM credentials must stay in user-level npm config or CI secrets.
