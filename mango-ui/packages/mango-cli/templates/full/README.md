## Frontend

The frontend consumes Mango npm packages. It does not depend on Mango workspace source paths.

```bash
scripts/dev-workspace.sh start
```

Use `scripts/dev-workspace.sh start` or `mango start` for local backend integration. The Mango CLI starts the frontend with `VITE_ADMIN_PROXY_PATH` pointing to the configured backend port. For standalone frontend build verification, use `npm --prefix frontend install --registry={{npmRegistry}}` and `npm --prefix frontend run build`. `npm run preview` serves built static assets and does not proxy `/api`.

## Backend

The backend consumes `io.mango:mango-admin-starter`. `io.mango.platform.seed:mango-seed-starter` is an optional startup seed capability and is not required by the default generated app.

Development startup has one entry:

```bash
scripts/dev-workspace.sh start
curl http://127.0.0.1:5555/actuator/health
```

The real runner lives in the Mango CLI and reads `mango.dev.json`. The shell script is a compatibility wrapper. `mango.dev.json` defines backend/frontend apps, dependencies and health checks. `.mango/dev-workspace.env` keeps local ports, database settings and secrets.

Useful commands:

```bash
mango validate
mango doctor
mango plan
mango start
mango status
mango logs {{projectKebab}}-service
mango stop
```

The generated backend app uses the explicit Spring Boot Maven plugin goal from `mango.dev.json`, waits for backend readiness, and then starts the frontend with the matching proxy target when the default group is started.

Deployment packaging is separate from development startup. Use `package + java -jar` or Docker only in deployment runbooks, not for local development.

To inspect or initialize local backend settings:

```bash
scripts/dev-workspace.sh init
scripts/dev-workspace.sh print
scripts/dev-workspace.sh backend
scripts/dev-workspace.sh frontend
```

`scripts/backend-dev.sh` is kept only as a compatibility wrapper and delegates to `scripts/dev-workspace.sh backend`.

Official seed data is disabled by default and requires adding `io.mango.platform.seed:mango-seed-starter` to the backend app dependencies. Enable it only for an empty or prepared database, and provide the initial administrator password explicitly:

```bash
MANGO_SEED_ENABLED=true \
MANGO_SEED_ADMIN_PASSWORD='replace-with-a-strong-password' \
scripts/dev-workspace.sh backend
```

The seed runner is idempotent: repeated startup does not duplicate the default tenant, administrator, member, role, application binding or role menu grants. Existing administrator passwords are not overwritten. In `prod` or `production` profiles, weak default passwords are rejected during startup.

Maven credentials must stay in user-level Maven `settings.xml`. NPM credentials must stay in user-level npm config or CI secrets.
