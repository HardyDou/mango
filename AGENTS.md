# 电子保函系统 - 项目指南

## 项目概述

电子保函系统是一个 B2B 平台，连接：
- **上游**：交易系统（公共资源交易中心、招标平台）
- **下游**：金融机构（银行、保险公司、担保公司）

## 关键业务点

### 1. 业务类型

| 类型 | 说明 |
|------|------|
| 银行电子保函 | 银行直接开具 |
| 分离式银行保函 | 担保公司担保，银行出具 |
| 保证保险保单 | 保险公司出具 |
| 担保保函 | 担保公司出具 |

### 2. 使用场景

- 投标保证金保函（≤80万）
- 履约保证金保函（合同价5-10%）
- 工程质量保证金保函（合同价3%）
- 农民工工资支付保函

### 3. 核心流程

```
项目信息同步 → 投保申请 → 保费支付 → 保函开具 → 电子签章 → 保函验真
                                ↓
                         退保/注销/理赔
```

## 关键技术点

### 1. 支付规则

- **支付方式**：企业网银、微信、支付宝
- **规则**：每个上游平台独立配置（是否允许个人支付、可用支付方式）

### 2. 接口安全（重点）

#### 接口防篡改
| 签名算法 | 密钥 | 典型平台 |
|----------|------|----------|
| SM3 | appkey + appsecret | 数字泸州、眉山 |
| MD5 | appkey + appsecret | 蔓延 |
| SM2 | CA证书 | 部分标准规范 |

#### 业务数据加密
| 方案 | 密钥来源 | 典型平台 |
|------|----------|----------|
| 数字信封 (SM4 + SM2) | 投标人CA证书 | 数字泸州 |
| 平台密钥 (SM4) | 平台分配appSecret | 江西省 |

#### 关键差异
- **签名算法**：SM3 / MD5 / SM2
- **参与签名参数**：不同平台可能不同
- **解密策略**：数字信封 vs 平台密钥

### 3. 电子签章

- 金融机构使用CA证书电子签名
- OFD/PDF格式
- 时间戳保持不变

### 4. 信息加密

- **开标前**：投标人名称、保证金金额等敏感信息加密
- **开标后**：推送解密key，生成明文保函

## 上游对接要点

### 已分析的平台

1. **数字泸州** - 数字信封模式
2. **江西省** - 平台密钥模式
3. **眉山市4.0** - 平台密钥模式
4. **蔓延-见费模式** - MD5签名

### 设计要求

每个上游平台需独立配置：
- `platformCode` - 平台编码
- `signAlgorithm` - 签名算法
- `encryptAlgorithm` - 加密算法
- `signParams` - 参与签名的参数列表
- `secretKey` - appSecret
- `caCertPath` - CA证书路径（数字信封模式）
