## 3.1 测试用例编写

| 属性 | 内容 |
|------|------|
| **主角色** | QA Agent |
| **协助角色** | Dev Agent |
| **工作内容** | 编写完整测试用例（含正向/逆向）；覆盖边界条件（空值、最大长度、特殊字符、边界数值） |
| **产出物** | 测试用例文档 |

### 边界条件覆盖

| 边界条件 | 测试点 |
|---------|-------|
| 空值 | null, "" |
| 最大长度 | 0, 1, 最大值 |
| 特殊字符 | `<`, `>`, `&`, `'`, `"` |
| 边界数值 | Integer.MAX_VALUE, Integer.MIN_VALUE |
| 负数 | -1, -100 |
| 零 | 0 |

### 测试用例结构

```markdown
## 测试用例：XXX功能

### 正向用例
| 步骤 | 操作 | 预期结果 | 截图 |
|------|------|---------|------|
| 1 | 打开页面 | 显示列表 | list.png |
| 2 | 点击新增 | 弹出表单 | form.png |

### 逆向用例
| 步骤 | 操作 | 预期结果 | 截图 |
|------|------|---------|------|
| 1 | 提交空表单 | 提示必填 | error.png |
```

## 3.2 E2E 执行

| 属性 | 内容 |
|------|------|
| **主角色** | QA Agent |
| **协助角色** | - |
| **工作内容** | 执行完整 E2E 测试；每步截图留痕；包含正常流和异常流 |
| **产出物** | 测试报告 + 截图 |

### E2E 执行原则

- 每一步都截图留痕
- 包含正常流和异常流
- 覆盖边界条件

### 规范要求

详见：`@mango-pmo/rules/frontend/04-test.md`

| 层级 | 覆盖率 |

| Mapper 层 | ≥ 60% |
