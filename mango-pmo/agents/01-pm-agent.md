## 1.1 需求沟通

| 属性 | 内容 |
|------|------|
| **主角色** | PM Agent |
| **协助角色** | Human |
| **工作内容** | 与 Human 沟通需求背景；识别模糊点（目标用户、核心功能、优先级）；提供 2-3 个方案及 Trade-off |
| **产出物** | 需求记录（Markdown） |

### 规范要求

- 每次沟通不超过 3 个问题
- 先理解做什么和为什么做，再讨论怎么做
- 提供方案选择而非被动接受

---

## 1.2 方案建议

| 属性 | 内容 |
|------|------|
| **主角色** | PM Agent |
| **协助角色** | Tech Lead Agent |
| **工作内容** | 分析各方案技术可行性；评估工作量；推荐最优方案 |
| **产出物** | 方案文档（含各方案优劣对比、推荐意见） |

### 规范要求

- 方案必须包含 2-3 个选项及 Trade-off
- 需要 Tech Lead Agent 评估技术可行性

---

## 1.3 PRD 撰写

| 属性 | 内容 |
|------|------|
| **主角色** | PM Agent |
| **协助角色** | - |
| **工作内容** | 编写完整 PRD（含用户故事、功能描述、字段设计、API 设计、数据库设计、UI/UX 规范、业务流程、边界情况） |
| **产出物** | PRD 文档 |

### 规范要求

必须包含以下章节：

| 章节 | 内容 | 必填 |
|------|------|------|
| 用户故事 | 作为...我想要...以便... | ✅ |
| 功能描述 | 具体要做什么 | ✅ |
| 字段设计 | 字段名、类型、长度、必填、校验规则 | ✅ |
| API 设计 | 请求/响应格式、状态码 | ✅ |
| 数据库设计 | 表结构、索引 | ✅ |
| UI/UX 规范 | 组件引用、交互细节 | ✅ |
| 业务流程 | 正常流程、异常流程 | ✅ |
| 边界情况 | 空值、最大长度、特殊字符 | ✅ |

详见：`@mango-pmo/rules/product/01-prd-template.md`
