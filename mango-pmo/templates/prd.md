# &lt;需求名称&gt; PRD

> 适用规范：`mango-pmo/rules/product/01-prd-template.md`。
> 首先 `For AI`：让 AI 可理解、可开发、可验收。
> 其次 `For Human`：帮助人把需求写成 AI 能理解的无歧义输入，降低 AI 落地偏差。
> 本文只写业务需求和页面原型。禁止写接口、权限码、数据库、路由、component key、模块归属、初始化方式、测试命令和交付台账。

## 16. AI 输出自检

自检结果只能填写 `PASS`、`FAIL`、`BLOCKED` 或 `EXCEPTION`。任一 `FAIL/BLOCKED` 时，最终动作只能是 `STOP/ASK`；`EXCEPTION` 必须引用用户确认来源；全部 `PASS` 且进入详细设计判定为 `READY` 时，最终动作才可为 `NEXT`。

| 检查项 | 结果 | 说明 |
|---|---|---|
| 每个关键对象是否至少关联一个流程、页面或验收项 |  |  |
| 每个关键流程是否包含规则、状态变化、异常限制和验收标准 |  |  |
| 每条业务规则是否至少被一个流程、页面或验收项引用 |  |  |
| 每个页面按钮是否能追溯到关键流程或业务规则 |  |  |
| 每个验收项是否有明确用户操作和可见结果 |  |  |
| 是否覆盖正常态、空态、加载态、失败态、不可操作态 |  |  |
