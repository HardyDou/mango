# 安全规范

## 1. 配置安全

- 密钥、口令、地址、账号必须从配置读取。
- 禁止硬编码敏感信息。
- 敏感配置不提交到仓库。

## 2. 输入安全

- 所有外部输入必须校验。
- 字符串输入必须限制长度和格式。
- 高风险内容必须做过滤或转义。

## 3. 数据访问安全

- 所有 SQL 必须参数化。
- 禁止拼接 SQL。
- 权限相关查询必须显式控制范围。

## 4. 权限控制

- 受保护接口必须做权限校验。
- 权限码命名保持统一。
- 不允许只在前端做权限控制。

## 5. 敏感数据

- 敏感字段按需要脱敏返回。
- 日志禁止打印敏感内容。
- 密码只允许加密存储。

## 6. 传输安全

- 外部通信默认使用 HTTPS。
- 不使用已淘汰协议和弱加密配置。

## 7. 禁止事项

- 在代码里写明文口令
- 日志输出 token、密码、证件号
- 只做页面权限，不做接口权限
