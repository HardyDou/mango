# 表单 UI 布局规范

## 1. 适用范围

- 适用于 Mango 前端后台的新增、编辑、确认、审批、处理、详情和配置类表单。
- 适用于页面表单、弹窗表单、抽屉表单、详情信息展示和只读核对区。
- 本规范只约束信息组织和布局原则；组件用法仍遵守 Element Plus UI 规范、Vue 代码规范和组件开发规范。

## 2. 核心原则

- 表单不是接口字段容器，而是业务判断路径。
- 布局必须帮助用户按业务顺序完成识别、判断、录入、核对和提交。
- 禁止按接口字段、数据库字段、返回顺序或开发追加顺序机械排列。
- 禁止为了填满多列而破坏字段关联、阅读顺序或视觉重量。

## 3. 分组原则

- 表单必须按业务判断分组，每个分组必须能回答一个明确问题。
- 分组依据必须来自业务关系、判断关系、计算关系、依赖关系或核对关系。
- 禁止只因为字段类型相近就分组；例如不能把所有只读字段统一放一组、所有输入字段统一放一组。
- 只读信息和可编辑信息可以在同一组内，只要它们服务同一个业务判断。
- 技术字段、内部 ID、JSON、扩展参数、报文、规则 ID 和低频审计信息必须后置、折叠或放入独立技术信息分组。

常见分组示例：

- 业务识别：用于确认处理对象是谁。
- 核心判断：用于确认本次操作是否合理。
- 关联对象：用于确认上下游对象、主体、通道、组织或人员。
- 金额信息：用于对比原始金额、操作金额和计算结果。
- 账户信息：用于确认收款、付款、退款或结算账户。
- 凭证与说明：用于上传依据、填写原因和补充备注。
- 状态与结果：用于展示状态、处理结果和失败原因。
- 时间与审计：用于追溯谁在什么时候做了什么。

## 4. 组内排序

- 组内字段必须有业务先后顺序，后面的字段应能参考前面的字段。
- 典型顺序是：基础值 -> 操作值 -> 计算结果 -> 状态或结论。
- 有对比关系的字段必须相邻，不能拆到不同分组或页面角落。
- 有级联、互相影响、前置依赖或计算关系的字段必须相邻。
- 同一判断链上的字段优先放在同一行或相邻行。

示例：

```text
可退金额 -> 退款金额 -> 剩余金额 -> 退款类型
```

## 5. Label 与控件

- 后台表单默认使用左 label、右内容。
- 宽度足够时，不默认使用 label 在上、内容在下的结构。
- 字段独占整行不等于 label 必须在上方。
- 只有长文本、说明、富文本、文件上传、表格型控件、JSON、复杂编辑器或移动端窄屏才允许改为上下结构。
- label 必须使用业务名词，禁止使用实现词、接口字段名或数据库字段名。

## 6. 多列布局

- 多列布局必须按字段关联强度、内容长度、控件复杂度和视觉重量决定。
- 短字段适合左右并排，例如状态、类型、金额、数量、日期、开关、短编码。
- 长字段优先独占整行，例如名称、地址、开户行、URL、备注、原因、说明、文件上传、JSON、表格型配置、富文本和复杂选择器。
- 两个长字段禁止强行左右并排。
- 允许左短右长或长字段独占整行，目标是阅读顺畅，不是网格填满。
- 多列布局不要求从上到下机械填满每个格子。

## 7. 详情展示

- 详情页也必须按业务判断展示，不得直接展开数据库字段。
- 详情信息顺序优先为：识别信息 -> 状态金额 -> 通道、账户或关联对象 -> 凭证与结果 -> 时间与审计。
- 关键状态使用统一状态标签；金额右对齐或在同组内形成对比；账号和敏感信息必须脱敏。
- 扩展信息、技术信息和审计追踪不得压过主业务信息。

## 8. 操作区

- 表单主操作固定在底部右侧，顺序为取消、保存或确认。
- 主操作只能有一个 primary。
- 危险操作必须使用 danger，并有明确影响范围和二次确认。
- 提交中必须显示 loading，并禁止重复提交。
- 关闭未保存表单前必须提示确认。

## 10. 检查清单

- 是否先按业务判断分组，而不是按接口字段分组。
- 每个分组是否有明确业务依据。
- 组内字段是否符合业务先后顺序。
- 有对比、级联、依赖或计算关系的字段是否相邻。
- 只读字段和输入字段是否按业务判断混合组织，而不是机械拆开。
- 多列布局是否考虑字段长度和视觉重量。
- 长字段、复杂控件是否独占整行。
- label 是否默认左侧展示。
- 技术字段是否后置、折叠或独立分组。
- 操作区是否稳定、提交态是否完整、关闭是否有未保存确认。

## 11. 禁止事项

- 禁止把表单做成接口字段展示器。
- 禁止按数据库字段顺序直接生成页面。
- 禁止两个长字段强行左右并排。
- 禁止为了凑满两列拆散强关联字段。
- 禁止把同一判断链上的字段放到不同分组。
- 禁止默认把所有 label 放在控件上方导致表单高度膨胀。
- 禁止让内部 ID、规则 ID、JSON 和技术报文占据主业务区域。
