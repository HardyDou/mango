# 列表页 UI 规范

## 1. 页面结构

1. 列表页内容区不展示页面标题、面包屑、页签标题。
2. 列表页不展示“搜索条件”“数据列表”等卡片标题。
3. 标准结构为：搜索区、功能区、表格区、分页区。
4. 搜索区和表格区通过白底、轻阴影、圆角和间距区分。

## 2. 搜索区

1. 搜索条件使用统一表单布局。
2. 桌面端一行最多 4 个字段。
3. 默认最多展示两行常用条件。
4. 超出默认展示范围时，使用居中图标展开/收起。
5. 展开/收起图标使用 Element Plus Icons。
6. 查询、重置按钮位置保持稳定。
7. 时间范围使用 range 类型，不拆成两个无关联输入。
8. 搜索条件变化后分页回到第一页。

## 3. 功能区

1. 功能区只放业务操作。
2. 不默认放刷新、列设置、密度切换等工具按钮。
3. 功能区按钮统一使用 `plain` 风格。
4. 主操作使用 `primary plain`。
5. 危险操作使用 `danger plain`。
6. 批量操作未选择数据时禁用。
7. 删除类按钮即使禁用，也保留 `danger` 语义。

## 4. 表格区

1. 表格使用 `ElTable` 或项目统一表格组件。
2. 表格列宽、状态、操作列和分页保持稳定。
3. 操作列固定右侧。
4. 行内操作使用文本按钮。
5. 行内删除使用 `danger` 文本样式。
6. 空值统一显示 `-`。
7. 状态统一使用 `ElTag` 或公共状态组件。
8. 分页位于表格下方，分页数据由接口结果驱动。

## 5. 检查清单

1. 是否误放页面标题、面包屑或卡片标题。
2. 搜索区是否一行最多 4 个字段、默认最多两行。
3. 超出两行时是否使用居中图标展开。
4. 功能区按钮是否统一 `plain`。
5. 删除、批量删除是否使用 `danger`。
6. 表格操作列、状态列、分页是否稳定。
7. 是否覆盖 loading、empty、error 和无权限状态。
