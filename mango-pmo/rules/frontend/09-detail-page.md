# 详情页 UI 规范

## 1. 页面结构

1. 详情页是下级页面，必须有返回栏。
2. 返回栏只放返回图标和当前标题。
3. 返回栏不放面包屑、状态、页面操作按钮。
4. 页面级操作放到底部悬浮操作栏。
5. 底部操作栏与内容区同宽，按钮居中展示。

## 2. 业务组织

详情展示必须按业务判断路径组织，不能按接口字段、数据库字段顺序展开。

推荐顺序：

1. 识别信息：当前对象是谁。
2. 状态与金额：当前状态、关键金额、数量、余额、结果。
3. 通道、账户、关联对象：和业务判断相关的外部对象。
4. 凭证与结果：附件、依据、处理结果、失败原因。
5. 时间与审计：谁在什么时候做了什么。

## 3. 模块与字段

1. 详情页必须按业务模块分组。
2. 模块标题必须清晰，例如基本信息、金额信息、账户信息、凭证与结果、时间与审计。
3. 字段展示优先使用 `ElDescriptions`。
4. 后面的字段要能参考前面的字段。
5. 有对比、级联、依赖、计算关系的字段必须相邻。
6. 长文本字段单独占整行。
7. 关联数据较多时使用模块内表格。
8. 技术字段后置、折叠或放入独立技术信息分组。

## 4. 技术字段

以下字段不得放在主业务阅读路径前面：

- 内部 ID
- 规则 ID
- JSON
- 扩展参数
- 报文
- trace 信息
- 低频审计信息

## 5. 检查清单

1. 是否按业务判断路径组织。
2. 每个模块标题是否能说明字段归属。
3. 字段顺序是否符合业务先后。
4. 对比、级联、计算字段是否相邻。
5. 技术字段是否后置。
6. 状态是否放在概要模块或具体字段中，而不是返回栏。
7. 是否覆盖 loading、empty、error 和无权限状态。
