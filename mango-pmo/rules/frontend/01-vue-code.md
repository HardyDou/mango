# Vue 代码规范

## 1. 基本规则

- 使用 Vue 3。
- 使用 `script setup`。
- 使用 TypeScript。
- 不使用 `any`。
- 新增页面和组件优先复用现有能力。

## 2. 组件规则

- 组件名使用 `PascalCase`。
- 页面组件放 `views`。
- 只在当前页面使用的组件放页面目录。
- 可跨模块复用的组件放对应 `packages/*`。
- 不在 `apps` 和 `packages` 里维护两份相同组件。

## 3. 组件使用规则

- 优先直接使用 Element Plus 原生组件。
- 只有存在稳定复用价值时才封装组件。
- 公共组件禁止直接依赖基座 store、router、菜单、启动逻辑。
- 公共组件需要的数据和行为通过 `props`、`emits`、回调传入。

## 4. 类型规则

- 对象结构优先使用明确类型。
- API 类型与页面状态类型分开定义。
- 公共类型放公共位置。
- 页面私有类型不要放到公共包。
- 后端 `Long`、雪花 ID、数据库主键、业务主键等标识字段在前端统一使用字符串语义，公共类型使用 `ApiId`。
- 禁止对 ID 调用 `Number(id)`、`parseInt(id)` 后再用于接口请求、路由参数、表格 key、组件 model 或文件 token。
- 只有页码、数量、金额、排序等真实数值字段可以建模为 `number`。

## 5. 样式规则

- 优先使用现有设计变量和主题变量。
- 不直接覆盖 Element Plus 全局样式。
- 不为一个页面临时定义新的全局设计规则。
- 第三方组件样式必须显式引入。

## 6. Store 规则

- 基座 store 只服务基座。
- 公共包禁止直接依赖基座 store。
- 跨包状态通过契约、参数、事件传递。

## 7. 路由规则

- 路由按模块组织。
- 动态路由逻辑集中管理。
- 页面组件不直接写权限装配逻辑。

## 8. 禁止事项

- 为简单按钮、输入框做无意义二次封装
- 在公共组件里引用基座代码
- 同一能力在多个目录重复实现
- 直接把页面临时样式扩散成全局规则
