# 弹框与抽屉 UI 规范

## 1. 使用场景

1. 短表单和确认操作使用弹框。
2. 字段较多、需要上下文参照或分组编辑时使用抽屉或独立页面。
3. 复杂业务判断、长流程、多模块编辑不放进小弹框。

## 2. 弹框规则

1. 弹框标题必须明确表达动作和对象。
2. 弹框内容按业务分组，不按接口字段顺序堆叠。
3. 弹框 footer 操作位置保持一致。
4. 确认按钮必须有提交 loading。
5. 提交中禁止重复提交。
6. 关闭未保存内容前必须提醒确认。

## 3. 抽屉规则

1. 抽屉适合字段多、上下文强、需要分组编辑的场景。
2. 抽屉内仍需按业务模块分组。
3. 抽屉操作区保持稳定。
4. 不得把完整复杂页面硬塞进抽屉。

## 4. 删除确认

1. 删除、移除、清空等危险操作必须二次确认。
2. 删除确认必须说明影响范围。
3. 删除确认按钮使用 `danger`。
4. 删除确认按钮文案应明确为“确认删除”。
5. 禁止只写“确定”作为危险操作确认文案。

## 5. 检查清单

1. 弹框/抽屉是否适合当前任务复杂度。
2. 标题是否明确表达动作和对象。
3. 字段是否按业务分组。
4. 确认按钮是否有 loading。
5. 未保存关闭是否有确认。
6. 危险操作是否说明影响范围并使用 `danger`。
