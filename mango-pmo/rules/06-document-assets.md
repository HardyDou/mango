# 文档资产归档边界

## 1. 定位

- 本文件约束 Mango 规范、使用说明、设计文档、Sprint 计划和交付记录的归档边界。
- `mango-pmo` 是唯一长期规范源。
- `mango-docs` 不是规范源，只保存设计、计划、交付和历史材料。

## 2. 判定规则

一段内容按最强约束归档：

| 内容类型 | 判定标准 | 归档位置 |
|----------|----------|----------|
| 规范 | 长期有效，包含必须、禁止、流程门禁、架构边界、质量门禁、命名、权限、租户、事务、发布或测试约束 | `mango-pmo/rules/**` |
| 角色约束 | 定义 Agent 角色职责、必做事项、禁止事项和输出要求 | `mango-pmo/agents/**` |
| 使用说明 | 面向开发者操作某个模块或工具，包含命令、配置示例、常见错误、使用步骤 | 模块 README、开发者指南或工具 README |
| 设计文档 | 解释一次能力的背景、方案、模块边界、接口、数据和取舍 | `mango-docs/designs/**` |
| Sprint 计划 | 描述一次迭代的目标、范围、改动项、验证方式、完成标准和遗留问题 | `mango-docs/plans/**` |
| 交付记录 | 记录一次任务或迭代的验证结果、证据、例外和风险 | `mango-docs/evidence/**` 或 `mango-docs/plans/**` |
| E2E 脚本 | 可重复执行的端到端测试、回归脚本、fixture 和支持代码 | 对应应用、包或模块的正式测试目录 |
| 测试结果基线 | 某次正式验收的命令、环境、数据集、账号/租户标识、结果摘要、报告和证据路径 | `mango-docs/evidence/**` 或 `mango-docs/plans/**` |

如果同一段内容同时满足多个类型，按优先级归档：规范 > 角色约束 > 使用说明 > 设计文档 > Sprint 计划 > 交付记录。

## 3. 规范

- 长期规则只允许写在 `mango-pmo/rules/**`。
- 规则文件必须短、硬、可执行。
- 规则只能维护一份；其它文档只能链接规则源，不得复制规则正文。
- 新增规则必须接入 `mango-pmo/rules/index.json`，保证 preflight 可加载。
- 修改规则时必须说明影响范围和验证命令。

## 4. 使用说明

- 模块使用说明放在对应模块 README 或开发者指南。
- 工具使用说明放在对应工具 README。
- 使用说明可以包含示例命令、YAML 配置、注解用法、错误排查和操作步骤。
- 使用说明引用长期约束时，只能链接 `mango-pmo` 规则源。
- 禁止在使用说明中新增必须遵守的长期规则。

## 5. 设计与计划

- 设计文档只记录本次能力的背景、方案、边界、接口、数据、测试范围和风险。
- Sprint 计划只记录本次迭代的目标、范围、改动项、验证方式、完成标准和遗留问题。
- 设计文档和 Sprint 计划可以提出待治理问题，但不能把待治理建议写成长期规则。
- 设计或计划中出现长期规则时，必须迁移到 `mango-pmo/rules/**`，原文改为链接规则源。

## 6. 交付记录

- 交付记录只记录某次任务的事实、验证结果、截图、日志摘要、证据路径、例外和风险。
- `mango-docs/evidence/**` 只保存最终证据，不保存生成项目、依赖缓存、构建产物或运行日志。
- 可复用脚本不能长期放在 evidence 目录；必须沉淀到正式测试目录或工具目录。
- E2E 测试结果基线属于交付记录，必须能追溯到对应 E2E 脚本、测试命令、环境、数据集和报告路径。
- evidence 中允许保存当次执行脚本副本或报告摘要；长期维护入口必须放在正式测试目录。

## 7. 多数据源说明落位

- 模块默认数据源声明规则、覆盖优先级、禁止绕过租户/权限/事务边界等长期约束归档到 `mango-pmo/rules/backend/07-persistence.md` 或相关 PMO 规则。
- `@PersistenceDataSource`、`PersistenceDataSourceContext`、YAML 配置示例、常见错误排查归档到 `mango-infra-persistence-starter` 使用说明或开发者指南。
- 多数据源设计背景、方案取舍和历史交付记录保留在 `mango-docs/designs/**`、`mango-docs/plans/**` 或 `mango-docs/evidence/**`。

## 8. 禁止事项

- 禁止在 `mango-docs` 新增规范文件。
- 禁止在 `mango-docs` 维护规则入口。
- 禁止在设计文档、Sprint 计划、交付记录或 README 中复制长期规则正文。
- 禁止同一规则在 `mango-pmo`、`mango-docs`、README 和 Agent 入口多处维护。
