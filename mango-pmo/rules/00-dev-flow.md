## P - Plan（规划阶段）

### 1.1 需求沟通

| 属性 | 内容 |
|------|------|
| **主角色** | PM Agent |
| **协助角色** | Human |
| **工作内容** | 与 Human 沟通需求背景；识别模糊点（目标用户、核心功能、优先级）；提供 2-3 个方案及 Trade-off |
| **产出物** | 需求记录（Markdown） |
| **遵守规约** | PM Agent 文件 |

### 1.2 方案建议

| 属性 | 内容 |
|------|------|
| **主角色** | PM Agent |
| **协助角色** | Tech Lead Agent |
| **工作内容** | 分析各方案技术可行性；评估工作量；推荐最优方案 |
| **产出物** | 方案文档（含各方案优劣对比、推荐意见） |
| **遵守规约** | PM Agent 文件 |

### 1.3 PRD 撰写

| 属性 | 内容 |
|------|------|
| **主角色** | PM Agent |
| **协助角色** | - |
| **工作内容** | 编写完整 PRD（含用户故事、功能描述、字段设计、API 设计、数据库设计、UI/UX 规范、业务流程、边界情况） |
| **产出物** | PRD 文档 |
| **遵守规约** | `@mango-pmo/rules/product/01-prd-template.md` |

### 1.4 技术设计

| 属性 | 内容 |
|------|------|
| **主角色** | Tech Lead Agent |
| **协助角色** | PM Agent |
| **工作内容** | 系统架构设计（模块划分、依赖关系）；API 接口设计（请求/响应格式、错误码）；数据库设计（表结构、索引）；创建 Flyway Migration 文件 |
| **产出物** | 技术设计文档；Migration 文件 |
| **遵守规约** | `@mango-pmo/rules/backend/03-api.md` `@mango-pmo/rules/backend/04-db.md` `@mango-pmo/rules/backend/05-module.md` |

### 1.5 任务分解

| 属性 | 内容 |
|------|------|
| **主角色** | Tech Lead Agent |
| **协助角色** | Dev Agent |
| **工作内容** | 按后端 ≤ 50 行/任务、前端 ≤ 100 行/任务 分解；每个任务可独立交付；确定任务顺序和依赖 |
| **产出物** | Sprint 任务列表（TODO） |
| **遵守规约** | `@mango-pmo/rules/product/02-sprint.md` |

### 1.6 Human 确认

| 属性 | 内容 |
|------|------|
| **主角色** | Human |
| **协助角色** | PM Agent |
| **工作内容** | 确认 PRD 业务方向；确认技术设计方案 |
| **产出物** | 确认签字 |
| **触发条件** | PRD 和技术设计完成后必须 Human 确认才能继续 |

## A - Act（验收阶段）

### 4.1 PM 初步验收

| 属性 | 内容 |
|------|------|
| **主角色** | PM Agent |
| **协助角色** | - |
| **工作内容** | 确认功能范围；验证业务流程 |
| **产出物** | 初步验收意见 |

### 4.2 Human 随机验证

| 属性 | 内容 |
|------|------|
| **主角色** | Human |
| **协助角色** | QA Agent |

|------|------|

| **产出物** | 验收结论 |
| **触发条件** | 测试报告完成后必须 Human 最终验收 |

## 问题处理（贯穿全程）

```
发现问题
    │
    ▼
评估影响 -> 影响当前 Sprint？
    │
    ├── 是 -> 紧急处理 -> 更新 Sprint 计划
    └── 否 -> 纳入下一 Sprint 规划
```

## 上下文管理

| 条件 | AI 行为 |
|------|---------|
| < 50% | 正常工作 |
| 50-60% | 提示用户上下文消耗 |
| > 70% | 必须完成当前任务后总结 |
| > 80% | 强制暂停，拒绝继续 |
