# 交付契约与设计说明规则

## 1. 适用范围

适用于需要完整交付确认的正式任务，包括需求、设计、开发、测试、验收、发布、PR、按设计文档推进、按 Sprint 推进和按交付台账推进。

以下任务不强制建立交付契约和交付台账：

- 简单查询、状态确认、日志查看、切换目录、关闭进程。
- 只读定位文件、类、接口、配置、worktree 或报错片段。
- 只修改规范、流程、设计文档、交付记录、Agent 入口或 PMO 临时复盘，且不影响服务代码、接口、数据库、测试、前端页面或构建配置。

豁免任务仍必须说明目标、改动范围、验证方式和风险。

## 2. 目标

防止 Agent 将任务压缩成极简主链路、聚合页、演示页、临时实现或不可用交付。

## 3. 开工门禁

开始正式需求、设计、编码、测试、验收、发布或 PR 前，必须先明确交付契约。

交付契约必须说明：

- 目标
- 范围
- 不做什么
- 设计输入
- 交付物
- 验收方式
- E2E 覆盖判断
- 测试结果基线
- 风险与限制

## 4. 设计说明门禁

实质任务必须有明确设计说明。

- 已有设计文档时，设计说明必须引用该文档并抽取原子要求。
- 没有设计文档时，必须先为本次任务产出轻量设计说明。
- 设计说明必须写清模块边界、接口变化、数据变化、菜单/页面/权限变化、测试范围。
- 对外能力新增、修改、删除或重构时，设计说明必须写清 README、需求文档、设计文档、E2E 脚本和测试结果基线是否需要同步更新。
- 未经用户确认，禁止把完整任务改成 MVP、第一阶段、纵向切片或演示版本。

## 5. 交付台账门禁

开始编码或交付前，必须建立交付台账。

交付台账必须从用户要求、设计说明、设计文档和验收标准中抽取原子验收项，逐项记录：

- `ID`
- `来源`
- `要求`
- `设计决策`
- `代码交付物`
- `README/使用说明`
- `需求/设计文档`
- `E2E 脚本`
- `测试结果基线`
- `验收方式`
- `状态`
- `证据文件`

推荐使用模板：

```text
mango-pmo/templates/delivery-contract.md
```

## 6. 原子验收项规则

- 一个后台能力、业务能力、接口能力、数据能力、菜单能力、页面能力、权限能力或验收标准必须是一条独立台账项。
- 多个能力禁止合并成一个“管理页”“概览页”“主流程”“通用能力”台账项。
- 要求涉及菜单、页面、API、数据、权限、E2E 时，台账项必须逐类列出代码交付物、README 或使用说明、需求或设计文档、E2E 脚本、测试结果基线和验收证据。
- 新增、修改、删除或重构对外能力时，台账项必须关联 E2E 覆盖判断；不需要 E2E 脚本或结果基线时，必须标记 `EXCEPTION` 并说明用户确认依据或不适用原因。
- 无法落地的项必须标记为 `EXCEPTION`，并记录用户确认依据。

## 7. 禁止替代

未经用户明确确认，禁止用以下内容替代交付契约：

- 用聚合页替代多个独立管理页面。
- 用 MVP、第一阶段、纵向切片替代完整范围。
- 用主链路跑通替代完整后台、接口、数据和验收能力。
- 用前端临时菜单替代后端入库菜单。
- 用 Mock、虚拟、演示数据替代要求的沙箱或真实能力。
- 用单元测试、接口 200、页面打开或口头说明替代应有的 E2E 脚本和测试结果基线。
- 用“后续优化”“未来补齐”替代本次明确要求。

## 8. 验证门禁

交付前必须执行：

```bash
node mango-pmo/tools/delivery-contract-check.mjs \
  --design <设计说明或设计文档路径> \
  --ledger <交付台账路径> \
  --mode verify \
  --require-materials
```

验证模式下：

- 设计说明或设计文档必须存在。
- 台账必须存在。
- 必填列必须完整。
- 必需交付项必须存在。
- `--require-materials` 开启时，代码交付物、README/使用说明、需求/设计文档、E2E 脚本、测试结果基线、验收方式和证据文件必须逐项填写；不适用时必须在对应单元格或状态中写明 `EXCEPTION`。
- 状态必须为 `DONE` 或 `EXCEPTION`。
- `EXCEPTION` 必须有证据文件或用户确认说明。
- 交付范围内不得存在 `TODO`、`FIXME`、`mock`、`virtual`、`模拟`、`伪代码`、`未来优化`、`后续优化` 等未交付标记。

## 9. 交付报告

最终交付必须报告：

- 交付台账路径。
- 台账总项数。
- `DONE` 项数。
- `EXCEPTION` 项数。
- E2E 脚本路径或 EXCEPTION 依据。
- 测试结果基线路径或 EXCEPTION 依据。
- 未完成项。
- 台账检查命令与结果。

未完成项不为 0 时，禁止声明“完成”。
